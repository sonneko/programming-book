# この本について

この本は、0からプログラミング、ひいてはIT（情報技術）に関わる多岐にわたる分野を体系的に学ぶための一貫した、長く深いチュートリアルのようなものです。また、単なる教科書として完結するのではなく、世界中に溢れている素晴らしいドキュメントや書籍を探すための「索引」としての役割も果たします。

現代において、ITという分野はあまりにも深く、そして広大です。すべての知識を一人の人間が網羅することは不可能です。そして、すべての知識があなたに必要というわけでもありません。AIが普及した現代、単なる「暗記された知識」の価値は相対的に下がっています。わからないことがあれば、その場で調べれば良いのです。

### 「知的な悟り」を目指して

しかし、知識が全くない状態で何かを生み出すことはできません。なぜなら、「何を調べるべきか」さえわからないからです。特にIT分野では、表面的な操作手順だけを覚えても、トラブルが発生した際に立ち往生してしまいます。

そこで、あなたがこの本を通して学ぶべきなのは、ITの世界に共通して現れる**エッセンス（本質）**です。

ITのあらゆる技術の裏側には、共通のアナロジー（類推）が存在します。例えば、Webブラウザがページを表示する仕組みと、ゲームがキャラクターを描画する仕組みには共通のデータ構造が見て取れます。また、スマートフォンのアプリがデータを保存する方法と、巨大な銀行のシステムが残高を記録する方法も、根本的な論理（不変条件やデータ変換）は驚くほど似ています。

このような共通項に気づき、一つの分野で得た理解を別の分野に応用できるようになった状態。私はこれを**「知的な悟り」**と呼んでいます。この状態になれば、新しい技術が登場しても「ああ、あれの進化系か」と瞬時に理解し、裏で何が起きているかを高い精度で予測できるようになります。

この本は、まさにその「知的な悟り」を得るための最短かつ最小のステップを提示しています。

### この本の構成と対象読者

本書は以下のような構成になっています。

1.  **第1章〜第3章（基礎編）**: プログラミングの共通言語であるPythonとJavaScriptを学びます。どんな分野に進むにせよ、避けては通れない最重要パートです。
2.  **第4章〜第9章（実践・応用編）**: TypeScript、React、バックエンド開発、CI/CDなど、現代のエンジニアが現場で必要とする実践的なスキルを網羅します。
3.  **第10章以降（深淵編）**: 「技術を使う人」から「技術を作る人」へのステップアップを目指します。メモリ管理、CPUの動作原理、OSの仕組みなど、低レイヤーの現実に触れます。
4.  **付録**: 特定のAPIの使い方や、各種プログラミング言語の紹介、法律、ライセンスの扱いなど、必要に応じて参照する補足資料です。

もしあなたがプログラミング未経験なら、最初からじっくりと進めることをお勧めします。もし*Scratch*などのビジュアルプログラミングの経験があるなら、1章を軽く流して2章から始めても良いでしょう。

### 執筆の背景

私は中学2年の頃に独学でプログラミングを始めました。当時は右も左もわからず、断片的な情報を繋ぎ合わせるのに非常に苦労しました。幸運にも、好奇心という燃料だけで深淵まで辿り着くことができましたが、もっと効率的な「地図」があればよかったのに、と何度も思いました。

この本は、その時の私のような「好奇心旺盛な学生」や、あるいは「プログラミングを教えて！」と言ってくる友人たちのために、高校2年生の夏に書き始めたものです。
