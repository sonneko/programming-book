# 非同期関数

前章で Promise の仕組みを学びましたが、正直に言って `.then()` や `.catch()` を延々と繋げるのは、少し読みづらいと感じませんでしたか？ 処理が複雑になればなるほど、数珠つなぎのコードは「今、どこの階層にいるのか」を見失わせます。

そこで登場したのが、現代の JavaScript/TypeScript における非同期処理の決定版、**`async`** と **`await`** です。これを使うことで、非同期処理をまるで「普通の同期的なコード」のように、上から下へ流れるように書くことができます。

## 1. async/await の魔法

まずは、同じ処理を Promise 版と async/await 版で比較してみましょう。

```typescript
// Promise 版
function getUserData() {
  return fetch('/api/user')
    .then(res => res.json())
    .then(user => console.log(user))
    .catch(err => console.error(err));
}

// async/await 版
async function getUserData() {
  try {
    const res = await fetch('/api/user');
    const user = await res.json();
    console.log(user);
  } catch (err) {
    console.error(err);
  }
}
```

どうでしょうか？ `async/await` 版の方が、どこで何を取得して、次に何をするのかが直感的にわかりますよね。

- **`async` キーワード**: 関数の前に付けると、その関数は自動的に **「Promise を返す関数」** になります。
- **`await` キーワード**: Promise の前に付けると、**「その処理が終わるまで、そこで一旦停止して待つ」** という魔法が発動します。処理が終わると、Promise の中身（解決された値）をそのまま取り出して変数に代入できます。

## 2. try-catch による美しいエラーハンドリング

`async/await` のもう一つの大きなメリットは、エラーハンドリングです。
Promise の `.catch()` ではなく、プログラミングでおなじみの **`try...catch`** 構文がそのまま使えます。

```typescript
async function safeFetch() {
  try {
    const data = await doSomethingRisky();
    return data;
  } catch (error) {
    // ネットワークエラーも、JSONのパースエラーも、全部ここで捕まえられる
    console.log("エラーが起きたけど、冷静に対処します");
  }
}
```

## 低レイヤーの視点：ジェネレータとステートマシン

「関数が途中で止まる」なんて、コンピュータにとってそんなことが可能なのでしょうか？

実は、`async/await` はコンパイラ（TypeScript コンパイラや Babel）によって、内部的には **「ステートマシン（状態遷移図）」** のような形に書き換えられています。

1. `await` に出会うと、関数はその時点のローカル変数（状態）をメモリに保存して、一旦実行を終了（中断）し、制御をイベントループに戻します。
2. 非同期処理が終わると、保存しておいた状態をメモリから復元し、`await` の直後から実行を再開します。

これは CPU のレジスタを切り替える「コンテキストスイッチ」にも似た、非常に高度な制御を JavaScript エンジンが行ってくれているのです。

## 私の視点：直列の罠に気をつけろ！

`async/await` はあまりに便利すぎて、初心者が陥りやすい罠があります。それが **「不必要な直列実行」** です。

```typescript
// 悪い例：1つ目の取得が終わるまで、2つ目を始めない（時間がかかる）
const user = await fetchUser();
const posts = await fetchPosts();

// 良い例：両方同時にスタートさせて、両方終わるのを待つ
const [user, posts] = await Promise.all([fetchUser(), fetchPosts()]);
```

`await` は「待つ」コマンドです。前の処理を待つ必要がないのに `await` を連発すると、アプリの動作がどんどん遅くなってしまいます。
「これは待つ必要があるか？ それとも同時にできるか？」を常に自問自答できるようになると、パフォーマンスに強いエンジニアになれますよ。

### まとめ

- `async` 関数は、常に Promise を返す。
- `await` は Promise が完了するまで実行を一時停止し、中身を取り出す。
- `try-catch` を使うことで、非同期のエラーも同期的に処理できる。
- 裏側では「ステートマシン」として、実行の中断と再開が行われている。
- 独立した処理は `Promise.all` で並列化して、速度を稼ごう。
