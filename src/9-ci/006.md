# Gitコマンドチートシート

覚えておくと便利なGitコマンド集。

- `git status`: 現在の状態確認（頻繁に使う）。
- `git diff`: 変更内容の確認。
- `git branch`: ブランチ一覧。
- `git checkout -b <name>`: 新しいブランチを作成して移動。
- `git pull`: リモートの変更をローカルに取り込む。
- `git stash`: 変更を一時退避させる。

### 深い理解のために

- **技術的深み**: これらのコマンドは、Gitの「インデックス」「オブジェクトデータベース」「HEAD」を操作します。特に`status`はワークツリーとインデックスの差分を計算しています。
- **低レイヤーの現実**: `git stash`は、変更内容をコミットオブジェクトとしてデータベースに書き込み、ポインタを移動させています。メモリへの負荷は非常に低いです。
- **設計思想**: Gitはコンテンツアドレサブルなストレージであり、すべてのデータはSHAハッシュで識別される不変のオブジェクトです。この思想を理解するとコマンドの挙動が納得できるようになります。

これらを手に馴染ませることで、開発中のGit操作におけるストレスが激減します。
