# シェーダーの実践：頂点とフラグメント

レンダリングパイプラインの章で学んだ通り、私たちがプログラムできる主要なステージは二つあります。物体の「形」を決める **頂点シェーダー** と、その「色」を決める **フラグメントシェーダー** です。

これら二つがどのように連携して一枚の絵を作り上げるのか、具体的なコードのイメージとともに見ていきましょう。

### 1. 頂点シェーダー：空間を構築する

頂点シェーダーの使命はシンプルです。入力された「3D空間の点」を、「画面上のどこに表示するか（クリップ空間座標）」へ変換して出力することです。

```glsl
// 頂点シェーダーの例 (GLSL)
attribute vec3 position; // JavaScriptから渡される3D座標
uniform mat4 modelViewMatrix; // モデルとカメラの行列
uniform mat4 projectionMatrix; // 投影行列

void main() {
  // 3D座標に行列を掛けて、画面上の位置を計算する
  gl_Position = projectionMatrix * modelViewMatrix * vec4(position, 1.0);
}
```

ここで重要なのは、頂点シェーダーは「個々の点」のことしか知らないということです。隣の点がどこにいるかは分かりません。しかし、この「点の変換」を数万回繰り返すことで、複雑な3Dモデルが正しい位置に配置されます。

#### 頂点シェーダーでできること
- 物体を移動、回転、拡大させる。
- 風に揺れる草や、波打つ水面のように、頂点を動かしてアニメーションさせる。
- 次のステージ（フラグメントシェーダー）に渡すための情報（法線やUV座標）を準備する。

### 2. フラグメントシェーダー：世界に魂を吹き込む

頂点シェーダーが終わると、GPUは三角形の中にあるピクセルを特定し（ラスタライズ）、一つ一つのピクセルに対してフラグメントシェーダーを実行します。

```glsl
// フラグメントシェーダーの例 (GLSL)
precision mediump float;
uniform vec3 lightColor; // 光の色

void main() {
  // すべてのピクセルを、光の色で塗りつぶす
  gl_FragColor = vec4(lightColor, 1.0);
}
```

フラグメントシェーダーの本当の力は、**「補間」** にあります。例えば、頂点Aを赤、頂点Bを青に指定すると、その間にあるピクセルはGPUによって自動的に紫色のグラデーションになります。

#### フラグメントシェーダーでできること
- テクスチャ（画像）を貼り付ける。
- 光の反射、影、屈折などを計算する（ライティング）。
- 霧（フォグ）や火、煙などのエフェクトを表現する。
- 画面全体をぼかしたり、色調を整えたりする（ポストエフェクト）。

### 連携の鍵：varying変数

頂点シェーダーで計算した情報を、フラグメントシェーダーに伝えたいときはどうすればいいでしょうか？ そのために `varying` (現代のGLSLでは `out` と `in`) 変数を使います。

1. 頂点シェーダーで、各頂点の「法線（向き）」を `varying` に書き込む。
2. GPUが三角形の中のピクセルごとに法線をいい感じに混ぜ合わせる（補間）。
3. フラグメントシェーダーが、その補間された法線を受け取って光の反射を計算する。

この「頂点からピクセルへの情報の橋渡し」こそが、グラフィックスプログラミングの醍醐味です。

### 低レイヤーの現実：ドローコールと状態変更

「頂点シェーダーとフラグメントシェーダーを切り替える」という行為は、GPUにとって非常に重い作業です。

もし100個の物体があったとして、それぞれ違うシェーダーを使おうとすると、描画のたびにGPUの「脳」を書き換えることになり、パフォーマンスが劇的に落ちます。賢いエンジニアは、できるだけ同じシェーダーを使い回し、データをまとめてGPUに送り込むことで、この切り替え回数（ドローコール）を最小限に抑えます。

### まとめ

- 頂点シェーダーは「位置」を決め、フラグメントシェーダーは「色」を決める。
- 二つのシェーダーは `varying` 変数を通じて連携し、データは自動で補間される。
- グラフィックスの表現力はフラグメントシェーダーの計算にかかっている。
- パフォーマンス向上のためには、シェーダーの切り替えを減らす設計が重要。

さて、シェーダーの役割は分かりましたが、実際にコードを書くには「行列を掛ける」や「法線を計算する」といった操作を避けて通れません。次は、グラフィックスの真の土台である「3D数学」の世界へ、勇気を持って踏み込んでみましょう。
