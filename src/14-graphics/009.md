# 座標変換の旅：ローカルからスクリーンまで

3Dモデルのデータは、最初は単なる「頂点座標の集まり」に過ぎません。それらが最終的にあなたのスマホやPCのディスプレイ上に正しい色として現れるまでには、いくつかの異なる「空間」を渡り歩く必要があります。

この一連の旅を、**MVP変換** と呼びます。グラフィックスプログラマが最も頻繁に扱い、そして最もバグを生み出しやすいこのプロセスを、順を追って見ていきましょう。

### 1. ローカル空間 (Local Space)
旅の始まりは、モデル自身の中心を原点とした空間です。
例えば、キャラクターのモデルデータでは、足元や腹部が `(0, 0, 0)` に設定されています。モデルを作ったアーティストにとって最も扱いやすい座標系です。

### 2. ワールド空間 (World Space)
モデルを「ゲームの世界（ワールド）」に配置します。
ここで使われるのが **モデル行列** (*Model Matrix*) です。
- キャラクターをワールドの `(10, 0, 5)` に移動させる。
- 30度回転させて北を向かせる。
- 少し大きく（1.2倍）する。
これらの変形を適用すると、すべての頂点はワールド共通の座標を持つようになります。

### 3. ビュー空間 (View Space)
次に、「カメラ（自分自身の目）」を基準にした空間に変換します。
使われるのは **ビュー行列** (*View Matrix*) です。
3D空間では「カメラを動かす」ことは、数学的には「カメラ以外の世界中のすべてを、カメラの動きと逆方向に動かす」ことと同じです。あなたが右に1メートル動くとき、世界は左に1メートル動いている――この相対的な変換を行うことで、カメラが常に原点 `(0, 0, 0)` にあり、常に前を向いている状態を作り出します。

### 4. クリップ空間 (Clip Space)
ここが最も魔法めいたステップです。3Dの世界を、画面という2Dの平面に「投影」します。
使われるのは **プロジェクション行列** (*Projection Matrix*) です。
- **透視投影** (*Perspective*): 遠くのものを小さく、近くのものを大きく見せる。現実の世界の見え方に近いです。
- **平行投影** (*Orthographic*): 遠近感を無視して、同じ大きさで描く。建築図面や2Dゲーム風の描画に使われます。

この変換が終わると、すべての頂点は `x, y, z` がすべて `-1.0` から `1.0` の範囲に収まる小さな立方体の世界に閉じ込められます。この範囲からはみ出した部分は「クリップ（切り捨て）」され、描画されません。

### 5. スクリーン空間 (Screen Space)
最後に、ブラウザのウィンドウサイズ（例えば 1920x1080）に合わせて座標をスケーリングします。
`(-1.0, 1.0)` という抽象的な数字が、具体的な「左から何ピクセル目、上から何ピクセル目」というピクセル座標に変換されます。これでようやく、GPUはピクセルを塗りつぶす準備が整います。

### MVP行列：3つの行列を一つに

プログラミング上では、これら3つの行列をあらかじめ掛け合わせておきます。
`MVP = Projection * View * Model`

頂点シェーダーの中で `gl_Position = MVP * position;` と一行書くだけで、頂点は一瞬にして「ローカル空間」から「クリップ空間」へとワープします。

### 低レイヤーの現実：浮動小数点の誤差

座標変換を何度も繰り返すと、コンピュータの数値計算の限界（浮動小数点の精度誤差）によって、微妙なズレが生じることがあります。

非常に巨大なワールド（例えば宇宙空間を旅するゲームなど）では、原点から遠く離れるとキャラクターが震えたり、表示がガタついたりすることがあります。これは、大きな数字を扱うときに小数点以下の精度が落ちるためです。シニアエンジニアは、カメラの周囲だけを原点として計算し直す「相対座標」の手法などで、この物理的な限界を克服します。

### まとめ

- 座標変換は、モデルから画面までの「空間の橋渡し」。
- モデル、ビュー、プロジェクションの3つの行列が主役。
- プロジェクション変換によって、3Dから2D（クリップ空間）への投影が行われる。
- 最終的にはウィンドウサイズに合わせたスクリーン空間へ着地する。

これで、3Dモデルがどうやって画面に収まるのかが分かりました。しかし、空間の中での「向き」や「回転」には、行列だけでは解決できない深い問題が潜んでいます。次は、それを解決する「クォータニオン」という強力な道具について見ていきましょう。
