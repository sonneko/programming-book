# CPUとGPU：設計思想の衝突

現代のコンピュータにおける「頭脳」といえば、真っ先に思い浮かぶのは **CPU** (*Central Processing Unit*) です。しかし、グラフィックスの世界の主役は **GPU** (*Graphics Processing Unit*) です。これら二つのプロセッサは、どちらも「計算をする」という点では同じですが、その中身や設計思想は驚くほど正反対です。

この違いを理解することは、効率的なグラフィックスプログラムを書くための絶対的な基礎となります。

### CPU：少数精鋭の天才数学者

CPUは、コンピュータ全体の指揮官です。OSを動かし、Webブラウザを立ち上げ、ユーザーの入力を処理し、複雑な条件分岐が重なるロジックを解きます。

- **設計の目標**: 「一つの難しいタスクを、いかに速く、順番に終わらせるか（低レイテンシ）」。
- **中身**: 数個から数十個の非常に強力なコアを持っています。それぞれのコアは、巨大なキャッシュメモリや、複雑な命令を先読みして実行する「アウトオブオーダー実行」など、計算を止めないための高度な回路で埋め尽くされています。

CPUは「一人の天才」です。難しい数式を解くのも、過去の経験から次の行動を予想するのも得意です。しかし、どれほど優秀でも、一人で数百万個の単純作業を同時にこなすことはできません。

### GPU：数千人の単純作業員

一方のGPUは、グラフィックスを描画するために生まれました。画面には数百万人分（ピクセル）の「色を計算する」という作業が待っています。

- **設計の目標**: 「単純な計算を、いかに大量に、同時に終わらせるか（高スループット）」。
- **中身**: 数千個（！）もの小さなコアが並んでいます。一つ一つのコアはCPUに比べれば非常に単純で、予測回路も大きなキャッシュも持っていません。しかし、その分だけ多くの計算ユニットをチップ上に詰め込むことができます。

GPUは「数千人の作業員」です。一人一人は単純な足し算や掛け算しかできませんが、全員が同じ命令を同時に実行することで、膨大なデータの束を一気に処理します。

### SIMD：一斉に同じ動きをせよ

GPUの驚異的な並列性能を支えているのが、**SIMD** (*Single Instruction, Multiple Data*) という方式です。

CPUが「Aさんにこの計算をさせ、次にBさんにあの計算をさせる」という個別指導を行うのに対し、GPUは「全員、右を向け！そして目の前の数字に2を掛けろ！」と大声で号令を出します。全員が同じ命令に従うため、命令を送るための回路を共有でき、その分だけ計算機を増やせるのです。

この「全員同じ動き」という性質は、グラフィックスにおいては非常に理にかなっています。なぜなら、3Dモデルの1万個の頂点を動かすとき、普通はすべての頂点に同じ行列計算を適用するからです。

### ヘテロジニアス・コンピューティングの時代

現代のプログラミング、特に私がHydentで目指している世界では、これら二つの頭脳を使い分ける **ヘテロジニアス（異種混在）計算** が重要になります。

- **CPUの役割**: ゲームのルール判定、ストーリーの進行、ユーザーインターフェースの制御など、複雑で予測不可能なロジック。
- **GPUの役割**: 数万個のポリゴンの座標変換、数百万個のピクセルの色計算、物理シミュレーション、AIの行列演算など、単純で規則的な大量の計算。

エンジニアの仕事は、データを適切に仕分けし、GPUが得意な仕事はGPUに丸投げし、結果を再びCPUで受け取るという「データの交通整理」をすることに他なりません。

### 低レイヤーの現実：データの転送コスト

GPUは計算自体は爆速ですが、一つだけ大きな弱点があります。それは、CPU（メインメモリ）からGPU（専用ビデオメモリ）へデータを送るのに時間がかかるという点です。

これを **「PCI Expressのボトルネック」** と呼びます。
たとえGPU側の計算が0.001秒で終わったとしても、そのデータを送るのに0.01秒かかってしまっては意味がありません。シニアエンジニアは、いかにCPUとGPUの間の通信（ドローコールやデータ転送）を減らすか、という設計に心血を注ぎます。

### まとめ

- CPUは「低レイテンシ」を目指した少数精鋭の天才。複雑なロジック担当。
- GPUは「高スループット」を目指した数千人の作業員。単純な大量計算担当。
- GPUはSIMD方式で、全員が同じ命令を一斉に実行する。
- データの転送コストを意識した設計が、パフォーマンスの鍵を握る。

プロセッサの個性を知ることは、彼らに最高のパフォーマンスを発揮させるための第一歩です。次は、この「作業員たち」がどのように連携して一枚の絵を作り上げているのか、その工程表である「レンダリングパイプライン」を覗いてみましょう。
