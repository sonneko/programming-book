# ゲームループとECS

これまでの章では、主に「一度実行して終わる」プログラムや、「ユーザーの操作（イベント）を待つ」Webアプリの作り方を学んできました。しかし、グラフィックスを多用するリアルタイムなプログラム（ゲームやシミュレーター）では、全く異なるプログラム構造が使われます。

それが **メインループ**（またはゲームループ）という考え方です。

### メインループ：終わらない繰り返しの世界

リアルタイムグラフィックスの基本は、以下の処理を高速に（1秒間に60回〜144回以上）繰り返すことです。

1. **入力処理**: キーボード、マウス、ゲームパッドの状態を読み取る。
2. **状態更新**: 物体を動かす、衝突判定を行う、AIの計算をする。
3. **描画**: 更新された状態をもとに、GPUに描画命令を送る。
4. **同期**: 次のフレーム（画面の更新）まで少し待機してタイミングを合わせる。

Webブラウザでは `requestAnimationFrame` という関数を使ってこのループを実現します。これは「ブラウザが次の画面を書き換える準備ができたら、この関数を呼んでね」と予約する仕組みです。

### オブジェクト指向の限界

かつて、ゲーム開発ではオブジェクト指向（OOP）が主流でした。`Player` クラスや `Enemy` クラスを作り、それぞれに `update()` や `draw()` メソッドを持たせる方法です。しかし、この方法にはいくつかの問題が出てきました。

- **深い継承の罠**: 「空飛ぶ敵」と「魔法を使う敵」を作るとき、どちらの機能を継承すべきか？（菱形継承問題）
- **パフォーマンス**: 大量のオブジェクトがメモリ上にバラバラに配置されるため、CPUのキャッシュ効率が悪くなり、速度が低下する。

### 新しいパラダイム：ECS (Entity Component System)

これらの問題を解決するために登場したのが **ECS** という設計手法です。

1. **Entity (エンティティ)**: ただの「ID」です。それ自体は何のデータも持ちません。
2. **Component (コンポーネント)**: 「データのみ」を持つ構造体です。例えば `Position` (位置)、`Velocity` (速度)、`Health` (体力) などです。
3. **System (システム)**: 「処理のみ」を持つ関数です。例えば「位置と速度の両方のコンポーネントを持つエンティティを探し出し、位置を更新する」という処理を行います。

#### なぜECSが良いのか？

- **データ指向設計**: 同じ種類のコンポーネントをメモリ上に連続して配置できるため、CPUのキャッシュヒット率が劇的に向上し、数万個のオブジェクトを同時に動かせます。
- **柔軟な組み合わせ**: クラスの継承ではなく、エンティティに「どのコンポーネントを持たせるか」という組み合わせだけで、新しいキャラクターの性質を定義できます。

### メインループのあるプログラムの作り方

メインループを作る際に最も重要なのは、**デルタタイム** (*Delta Time*) の扱いです。
デルタタイムとは、前回のフレームから今回のフレームまでに経過した時間のことです。

```javascript
function gameLoop(currentTime) {
  const deltaTime = currentTime - lastTime;
  lastTime = currentTime;

  update(deltaTime); // 時間に応じて移動距離を変える
  draw();

  requestAnimationFrame(gameLoop);
}
```

「1フレームごとに1ピクセル動かす」という書き方をすると、性能の高いPCでは速く動き、低いPCでは遅く動くという不公平なソフトになってしまいます。「1秒間に100ピクセル動かす」という計算にデルタタイムを掛けることで、どんな環境でも同じ速度で動作する、正しいリアルタイム処理が実現できます。

### ゲーム開発は「総合芸術」

グラフィックス、数学、物理、メモリ管理、サウンド、そしてECSのような高度なアーキテクチャ。ゲーム開発は、プログラミングにおけるあらゆる技術の集大成です。

私たちが学んでいる低レイヤーの知識は、こうした複雑なシステムが「なぜそのように設計されているのか」を理解するための最強の武器になります。エンジンの裏側でメモリがどうレイアウトされ、GPUとどう通信しているか。それが見えるようになれば、あなたはただの「ライブラリを使う人」から「システムを創る人」へと進化できるはずです。

### まとめ

- リアルタイム処理の基本は「メインループ」。
- requestAnimationFrameで滑らかなループを実現する。
- 大規模な開発には、データ指向な「ECS」というパラダイムが有効。
- デルタタイムを使って、環境に依存しない動作を作る。

最後は、さらに一歩進んで、これらのシステムをいかに高速に、効率よく動かすかという「最適化」の世界を覗いてみましょう。
