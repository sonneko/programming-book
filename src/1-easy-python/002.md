# エラーが出た時は

このページを読むのは、エラーが出てからでも構いません。プログラムを書き進めていて、思い通りに動かない、真っ赤な文字が表示されて止まってしまった...そんな時に、このページのことを思い出してください。

プログラミングにおいて、エラーは「敵」ではありません。むしろ、コンピュータが「ここがわからないよ！」と教えてくれている「親切なヒント」です。エラーが出た時にあなたが取るべき行動を、優先順位が高い順にまとめてみました。

### 1. エラーメッセージをじっくり読む
エラーが出たことを示す文章や記号を**エラーメッセージ**と呼びます。多くの場合、英語で表示されるため、慣れないうちは拒絶反応が出るかもしれませんが、頑張って読みましょう。実は、解決策の9割がここに書かれています。

Pythonの場合、エラーメッセージの最後の行に注目してください。例えば `SyntaxError: invalid syntax`（文法が間違っています）や `NameError: name 'x' is not defined`（'x'という名前は定義されていません）といった具合です。また、エラーが発生した行番号も教えてくれます。まずは「どこで」「どんな」エラーが起きたのかを把握しましょう。

### 2. 自分のコードと「対話」する
コンピュータは驚くほど厳格です。人間なら「これくらいわかるでしょ？」という些細なミスも、コンピュータにとっては致命的です。以下のポイントをチェックしてみてください。

- **全角文字の混入**: プログラミングの基本は半角です。全角スペースや全角のカッコ、カンマが紛れ込んでいませんか？（特にコピペした時に起こりやすいです）
- **タイピングミス**: `print` を `prnt` と書いていませんか？
- **インデント（字下げ）**: Pythonはインデントに意味がある言語です。スペースの数がずれていませんか？

### 3. エラーメッセージを検索する
メッセージを読んでも原因がわからない場合は、エラーメッセージ全体（または特徴的な一部）をコピーして、Googleなどの検索エンジンに貼り付けてみましょう。

世界中の誰かが、あなたと同じエラーに遭遇し、その解決策をインターネット上に残してくれています。ただし、解決策をそのまま真似するのではなく、「なぜその修正で治るのか」を考えるクセをつけてください。

### 4. AI（LLM）に相談する
最近では、ChatGPTやGeminiなどの **LLM**（*Large Language Model*：大規模言語モデル）に聞くのが非常に効率的です。「以下のコードでこのエラーが出ました。原因と対策を教えてください」と、コードとエラーメッセージをセットで渡すと、的確なアドバイスをくれます。

ただし、注意点があります。AIは平気で「もっともらしい嘘（ハルシネーション）」をつくことがあります。AIが提示したコードを盲目的にコピペするのは避けましょう。AIはあくまであなたの「家庭教師」であり、プログラムを書くのはあなた自身です。

### 5. 公式ドキュメント・リファレンスにあたる
その技術の「正解」が書かれているのが、公式の**リファレンス**（*reference*）です。Pythonであれば [docs.python.org](https://docs.python.org/ja/3/) がそれにあたります。

リファレンスを読むのは、慣れるまでは非常に苦痛かもしれません。専門用語が並び、不親切に感じることもあるでしょう。しかし、ネット上の二次情報（誰かのブログ記事など）は古かったり間違っていたりすることがありますが、公式ドキュメントは常に最も信頼できる情報源です。ここを読みこなせるようになると、エンジニアとしてのレベルが一段階上がります。

### 6. コミュニティで質問する
どうしても解決しない場合は、詳しい人に聞きましょう。

- **Qiita**: 日本人エンジニア向けの記事共有・質問サイト。
- **Stack Overflow**: 世界最大のQ&Aサイト。英語が基本ですが、情報の宝庫です。
- **teratail**: 日本語でITに関する質問ができるサービス。

質問する際は、「何をしようとして」「どんなエラーが出て」「自分は何を試したのか」を具体的に伝えるのがマナーです。

### 7. ソースコードを読んでみる
これは上級者向けですが、使っているライブラリやフレームワークが **OSS**（*Open Source Software*：ソースコードが公開されているソフトウェア）であれば、その中身を直接読んでエラーの原因を探ることもできます。

---

### 「もしかしたら、私は間違っていないのかも？」
稀に、あなたが使っている言語やツール自体のバグである可能性もあります。しかし、あなたがプログラミングを始めたばかりの初心者であれば、その確率は限りなく0に近いと言っても過言ではありません。まずは自分を疑い、謙虚にコードを見直すことから始めましょう。

エラーは、あなたが成長するための階段です。一つエラーを解決するたびに、あなたは少しだけ賢くなっています。
