# for文

前回学んだ `while`文を使えば、どんな繰り返し処理も書くことができます。しかし、実際にプログラムを書いていると「特定の回数だけ繰り返す」とか「リストの要素を一つずつ取り出す」といったパターンが非常に多いことに気づきます。

このような「よくある繰り返し」を、より簡潔に、安全に書くための構文が **`for`文** です。

## 基本的な使い方：回数を指定する

Pythonの `for`文は、他の言語（C言語やJavaなど）とは少し使い勝手が違います。最も一般的なのは `range()` 関数と組み合わせる方法です。

```python
for index in range(10):
    print(f"現在は {index} 回目です")
```

このプログラムは、0から9までの数字を順番に `index` に代入しながら、10回実行されます。

`while`文で同じことを書くとこうなります：
```python
index = 0
while index < 10:
    print(f"現在は {index} 回目です")
    index += 1
```

`for`文の方が「何回繰り返すか」が一目で分かり、さらに `index += 1` のような更新処理を書き忘れて無限ループになる心配もありません。

## イテレータという考え方

ここで少し不思議なのが、`range(10)` の正体です。実はこれ、単なる数字ではなく、**「イテレータ（*Iterator*）」** と呼ばれる特別なオブジェクトを生成する関数です。

イテレータとは、一言で言えば **「次の値を順番に出してくれる『もの』」** です。

`for`文は、このイテレータに対して「次の値をちょうだい」と頼み続け、値がなくなるまで処理を繰り返します。

## 実践：データの集まりを処理する

`for`文が最も輝くのは、複数のデータが入った「リスト（配列）」を扱うときです。

```python
fruits = ["apple", "banana", "orange", "grape"]

print("お店の在庫一覧：")
for fruit in fruits:
    print(f"・{fruit}")
```

このコードでは、`range()` の代わりに `fruits` というリストを直接 `for`文に渡しています。Pythonはリストから要素を一つずつ取り出し、変数 `fruit` に入れてくれます。

「リストの長さを調べて、`range()` で回して、添字（インデックス）で取り出す……」といった面倒な手順は一切不要です。これを **「拡張for文」** や **「for-in句」** と呼んだりします。

## 低レイヤーの視点：メモリ消費を抑える工夫

`range(1000000)`（100万回）と書いたとき、コンピュータの中では何が起きているのでしょうか。

もし `range` が 100万個の数字が入ったリストを実際に作っていたら、メモリを大量に消費してしまいます。しかし、Pythonの `range` は **「遅延評価（*Lazy Evaluation*）」** という仕組みを使っています。

`range` は、100万個の数字をメモリに用意するのではなく、「今の数字は 0 です」「次は 1 です」という **計算ルールだけ** を覚えています。`for`文から「次をちょうだい」と言われた瞬間に、次の数字を計算して作り出すのです。

これを「オンデマンド（必要に応じて）生成」と呼び、これによってどれだけ大きな回数を指定しても、メモリをほとんど消費せずに実行できるのです。

## 私の視点：`enumerate` でインデックスも一緒に使う

`for fruit in fruits:` は便利ですが、たまに「今、何番目の要素を処理しているか（番号）」も知りたいときがあります。そんな時は `enumerate()` 関数を使うのが Python 流です。

```python
fruits = ["apple", "banana", "orange"]

for i, fruit in enumerate(fruits):
    print(f"{i + 1}番目のフルーツは {fruit} です")
```

これを使わずに、わざわざ `i = 0` というカウンター変数を外で作ってループ内で `i += 1` するのは、あまり「Pythonらしくない（Non-Pythonic）」とされます。便利な道具を使いこなして、スッキリとしたコードを目指しましょう。

### まとめ

- `for`文は「決まった回数」や「データの集まり」を処理するのに適している。
- `range(n)` は 0 から n-1 までの値を生成する。
- リストを直接渡すと、要素を一つずつ取り出せる。
- `range` は「遅延評価」によってメモリを節約している。
- インデックスが必要なときは `enumerate()` を使おう。

### 展望

さて、ここまで「データの集まり」としてサラッと「リスト」を紹介してきましたが、リストはプログラミングにおいて非常に奥が深く、重要なデータ構造です。次は、このリスト型の使い方をより詳しく、徹底的にマスターしていきましょう。
