# 四則演算の実行

`Hello World`を出力することができたので、次はコンピュータに「計算」を依頼してみましょう。コンピュータの語源は *Compute*（計算する）ですから、これこそが彼らの本業です。

以下のプログラムを、エディタに入力して実行してみてください。

```python
# 1足す2を出力する
print(1 + 2) # 3

# 1引く2を出力する
print(1 - 2) # -1

# 2掛ける3を出力する
print(2 * 3) # 6

# 3割る2の「切り下げ」を出力する
print(3 // 2) # 1

# 3割る2を小数として出力する
print(3 / 2) # 1.5

# 5割る3の余りを表示する
print(5 % 3) # 2

# 2の10乗（べき乗）を表示する
print(2 ** 10) # 1024
```

足し算（*addition*）、引き算（*subtraction*）、掛け算（*multiplication*）、割り算（*division*）、そして余り（*remainder*）が正しく出力されましたか？

## Pythonの演算子ルール

このプログラムから学べることはたくさんあります。

1. **記号の違い**: 掛け算は `*`（アスタリスク）、割り算は `/`（スラッシュ）を使います。数学で使う `×` や `÷` はキーボードにありませんし、プログラミング言語では一般的にこれらの代わりの記号が使われます。
2. **べき乗**: Pythonでは `**` を使うことで「〜の何乗」という計算が簡単にできます。`2 ** 10` は $2^{10}$ つまり 1024 です。
3. **優先順位**: 数学と同じく、掛け算・割り算は足し算・引き算よりも先に計算されます。括弧 `()` を使うことで計算の順序を明示することもできます。

## 二つの「割り算」を使い分ける

Pythonで特に面白いのが、割り算が二種類あることです。

```python
# 切り下げ除算（整数を返す）
print(3 // 2) # 1

# 通常の除算（小数を返す）
print(3 / 2) # 1.5
```

`/` を一つ使うと、結果は必ず「小数（*float*）」になります。一方、`//` を使うと、小数点以下を切り捨てた「整数（*integer*）」が返ってきます。

これは「10個のお菓子を3人で分けるとき、一人何個もらえるか（余りは無視）」という計算をしたいときに `10 // 3` と書けるので便利です。

## 低レイヤーの視点：CPUの中の計算機

私たちが `+` や `*` と書くとき、コンピュータの内部（CPU）では何が起きているのでしょうか。

CPUの中には **ALU（*Arithmetic Logic Unit*：算術論理演算ユニット）** という、計算専用の回路があります。
私たちがPythonで `1 + 2` と命令すると、Pythonの実行エンジンがそれを解釈し、最終的にCPUに対して「レジスタAの数字とレジスタBの数字を足して、結果をレジスタCに入れろ」という命令（機械語）を送ります。

掛け算や割り算は、足し算に比べると回路が複雑で、計算に時間がかかる（クロック数を多く消費する）傾向があります。昔のコンピュータでは割り算を避けるような工夫もされていましたが、現代のCPUは非常に強力なので、日常的なプログラミングで気にする必要はほとんどありません。

## コラム：コンピュータは小数が苦手？

コンピュータで小数を扱うときは、少し注意が必要です。試しに、Pythonで `0.1 + 0.2` を実行してみてください。

驚くことに、結果は `0.3` ではなく、 `0.30000000000000004` のようになります。

これはバグではありません。コンピュータがすべての数値を「0」と「1」の2進数で扱っているために起こる、**「丸め誤差」** という現象です。
10進数における `0.1`（1/10）は、2進数に直すと無限小数になってしまい、有限のメモリに収めるためにどこかで切り捨てが発生します。

これを標準化した規格を **IEEE 754（浮動小数点数）** と呼びます。

### まとめ

- 足し算・引き算・掛け算・割り算の記号を覚えよう。
- `/` は小数、`//` は整数の結果を返す。
- コンピュータの内部では ALU が計算を担当している。
- 小数の計算には「誤差」がつきもの。
