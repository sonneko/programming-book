# 再利用性・設計のキソ

```python
prices = [100, 200, 300]
total = 0

for price in prices:
    total = total + price

print(f"合計金額は {total} 円です")
```
（再掲）

このプログラムは、リストにあるデータ全てに対して同じ処理（この場合は足し算）をリストの長さにかかわらずに行うことができます。言い方を変えると、値段のリストなら何でもかんでもこのプログラムに「注入」することができ、期待通りに動作するということです。

```python
prices = [200, 300, 400, 1000]
```

別に上記のように`prices`変数の中身を変えても別に問題はないですよね。

それと以下の書き方を比べてみましょう。

```python
total = 100 + 200 + 300
print(f"合計金額は {total} 円です")
```

別に合計金額の計算は上記のように書いても良いです。ただし大きな問題があります。

さっきのようにこのプログラムを、別の値段リストに書き換えるにはどうすれば良いですか？

```python
total = 200 + 300 + 400 + 1000
print(f"合計金額は {total} 円です")
```

こうすればいいですね。このコードでは、本来変更したかった「値段リストの内容」が変更されています。
しかし、合計金額計算のロジック=*logic*すなわち計算処理も一緒に変更されてしまっています。

このように非常に小規模なプログラムでは何も問題にならないですが、大きなプログラムになると大問題になってきます。
