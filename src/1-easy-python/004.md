# 変数定義と使用

プログラミングを学ぶ上で、最も基本的で、かつ最も強力な概念の一つが「変数（*Variable*）」です。これを使えるようになると、プログラムは単なる計算機から、複雑なデータを処理する「ソフトウェア」へと進化します。

## 変数とは：データに名前をつける

変数とは、一言で言えば **「データに名前をつけて保存しておくための箱」** です。箱の中にデータ（数値や文字列など）を入れ、後でその名前を使って中身を取り出したり、計算に使ったりすることができます。

なぜ名前をつける必要があるのでしょうか？ それは、人間にとって「150」という数字よりも「price（価格）」という名前の方が、そのデータが何を意味しているのか圧倒的に理解しやすいからです。

前回の記事で紹介した、変数を使わない単純な足し算を、変数を使って書き直してみましょう。

```python
# 変数を使わない場合
print(1 + 2)

# 変数を使う場合
one = 1
two = 2
result = one + two
print(result)
```

「行数が増えて面倒になっただけじゃない？」と思うかもしれません。しかし、プログラムが複雑になればなるほど、変数の真価が発揮されます。

## 実践：レジの合計金額計算

より現実的な例として、レジで商品の合計金額を計算するプログラムを考えてみましょう。

```python
# 商品の価格（単価）を定義する
apple_price = 150 
banana_price = 100
orange_price = 120 

# 購入する個数を定義する
apple_count = 3
banana_count = 1
orange_count = 5

# 各商品の小計（単価 × 個数）を計算する
apple_total = apple_price * apple_count
banana_total = banana_price * banana_count
orange_total = orange_price * orange_count

# すべての小計を足して、合計金額を出す
total_price = apple_total + banana_total + orange_total

# 消費税（10%）を計算する
tax_rate = 0.1
total_with_tax = total_price * (1 + tax_rate)

# 結果を表示する
print(f"リンゴの小計: {apple_total}円")
print(f"バナナの小計: {banana_total}円")
print(f"オレンジの小計: {orange_total}円")
print(f"税抜き合計: {total_price}円")
print(f"税込合計（10%）: {int(total_with_tax)}円")
```

このように変数を使うことで、「どの数字が何の価格なのか」「どんな計算をしているのか」が、コードを読むだけで一目瞭然になります。これをプログラムの **「可読性（*Readability*）」** と呼びます。

## 変数の中身を書き換える（再代入）

「変数」という名前の通り、その中身は後から変える（変えられる）ことができます。これを **「再代入（*Reassignment*）」** と呼びます。

```python
# 最初の価格
wallet = 1000
print(f"お財布には{wallet}円あります")

# お菓子を買ったので減る
snack_price = 200
wallet = wallet - snack_price

print(f"お菓子を買った後の残高: {wallet}円")
```

ここで `wallet = wallet - snack_price` という書き方に注目してください。数学の等式としてはおかしいですが、プログラミングにおける `=` は「右側の計算結果を、左側の変数に代入する」という意味です。

「今の `wallet` の値から `snack_price` を引いて、その結果を新しい `wallet` の値として保存し直す」という動きになります。

## 低レイヤーの視点：メモリ上の住所

コンピュータの内部（ハードウェア）では、変数はどのように扱われているのでしょうか？

私たちが `apple_price = 150` と書いたとき、コンピュータは **主記憶装置（メモリ / RAM）** の中に、このデータを置くための小さなスペースを確保します。

メモリには「住所（アドレス）」が割り振られており、CPUはその住所を指定してデータを読み書きします。しかし、人間が「住所 0x7FFF1234 に 150 を書け」と指示するのは大変です。そこで、プログラミング言語が **「住所に名前（変数名）をつける」** という役割を担ってくれているのです。

変数は、人間とハードウェアの架け橋となる「抽象化」の第一歩と言えます。

## 変数名の付け方ルール（命名規則）

Pythonでは、変数名をつけるときに守るべきルールや習慣があります。

1. **英単語を使う**: 誰が見ても意味がわかるようにしましょう。
2. **スネークケース**: `total_price` のように、単語の間をアンダースコア `_` で繋ぎます。
3. **数字から始めない**: `1price` はNGですが、`price1` はOKです。
4. **予約語を避ける**: `if` や `for` など、Pythonが特別な意味で使っている言葉は変数名にできません。

### よくあるミス：スペルミス

再代入のときにスペルを1文字でも間違えると、Pythonは「新しい変数を作った」と判断してしまいます。

```python
price = 100
pirce = 200 # スペルミス！
print(price) # 100のまま。「あれ？？」
```

これは初心者が必ず一度はハマる道ですので、動かないときはまずスペルを疑ってみましょう。

### まとめ

- 変数は「データに名前をつけた箱」。
- 適切な名前をつけることで、プログラムの意図（可読性）が向上する。
- 変数はメモリ上の特定の場所に対応している。
- 再代入によって中身を更新できるが、スペルミスには注意。
