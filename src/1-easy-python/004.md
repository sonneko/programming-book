# 変数定義と使用

プログラミングを学ぶ上で、最も基本的で、かつ最も強力な概念の一つが「変数（*Variable*）」です。勘違いしてほしくないのは、「変数」という名前だが、変わらないことが大半であるということです。

## 変数とは：データに名前をつける

変数は、「データに名前をつけて保存しておくための箱」というように考えることができます。箱の中にデータを入れ、後でその名前を使って中身を取り出したり、計算に使ったり、違うデータを入れ直することができます。

## 実践：レジの合計金額計算

より現実的な例として、レジで商品の合計金額を計算するプログラムを考えてみましょう。

```python
# 商品の価格（単価）を定義する
apple_price = 150 
banana_price = 100
orange_price = 120 

# 購入する個数を定義する
apple_count = 3
banana_count = 1
orange_count = 5

# 各商品の小計（単価 × 個数）を計算する
apple_total = apple_price * apple_count
banana_total = banana_price * banana_count
orange_total = orange_price * orange_count

# すべての小計を足して、合計金額を出す
total_price = apple_total + banana_total + orange_total

# 消費税（10%）を計算する
tax_rate = 0.1
total_with_tax = total_price * (1 + tax_rate)

# 結果を表示する
print(f"リンゴの小計: {apple_total}円")
print(f"バナナの小計: {banana_total}円")
print(f"オレンジの小計: {orange_total}円")
print(f"税抜き合計: {total_price}円")
print(f"税込合計（10%）: {int(total_with_tax)}円")
```

このように変数を使うことで、「どの数字が何の価格なのか」「どんな計算をしているのか」が、コードを読むだけで一目瞭然になります。これをプログラムの 「可読性（*Readability*）」 と呼びます。

## 変数の中身を書き換える（再代入）

「変数」という名前の通り、その中身は後から変える（変えられる）ことができます。これを「再代入（*Reassignment*）」と呼びます。

```python
# 最初の価格
wallet = 1000
print(f"お財布には{wallet}円あります")

# お菓子を買ったので減る
snack_price = 200
wallet = wallet - snack_price

print(f"お菓子を買った後の残高: {wallet}円")
```

ここで `wallet = wallet - snack_price` という書き方に注目してください。数学の等式としてはおかしいですが、プログラミングにおける `=` は「右側の計算結果を、左側の変数に代入する」という意味です。

「今の `wallet` の値から `snack_price` を引いて、その結果を新しい `wallet` の値として保存し直す」という動きになります。

## 変数名の付け方ルール（命名規則）

プログラミングでは、変数名をつけるときに守るべきルールや習慣がそれぞれの言語に対してあります。

別に1,2は守らなくても良いではないかと思うかもしれませんが、一般的にはこのルールですので、このルールを守らなければ共同作業はできません。一人しか見ないコードだったとしても、それなりに慣れてる人はこのルールに従います。

どんな言語でも1のルールは当てはまりますが、例外的に、共通テストの情報で出題されるプログラミングの問題では、ローマ字を使う習慣があります。

1. **英単語を使う**: 日本語ではなくローマ字で書く習慣があります。おそらく可読性やグローバル化による影響が理由です。
2. **スネークケース**: `total_price` のように、単語の間をアンダースコア `_` で繋ぎます。
3. **数字から始めない**: `1price` はNGですが、`price1` はOKです。どちらにしろ数字を使った変数名はあまり良くないことが多いです。
4. **予約語を避ける**: `if` や `for` など、Pythonが特別な意味で使っている言葉は変数名にできません。

### よくある間違い

再代入のときにスペルを1文字でも間違えると、Pythonは「新しい変数を作った」と判断してしまいます。

```python
price = 100
pirce = 200 # スペルミス！
print(price) # 100のまま。「あれ？？」
```

これは初心者が必ず一度はハマる道ですので、動かないときはまずスペルを疑ってみましょう。但し、このような問題はVSCodeなどのコードエディタ/IDE=`Integrated Development Environment`などを用いることによって解決されます。
