# if文

プログラムは、ただ上から下へ流れるだけではありません。状況に応じて「右へ行くか、左へ行くか」を決める必要があります。これを **「条件分岐」** と呼び、そのための最も基本的な道具が **`if`文** です。

私が初めて条件分岐を学んだとき、「これでようやく、コンピュータに『意志』を持たせることができる！」とワクワクしたのを覚えています。

## 基本的な使い方

`if`文の基本構造は以下の通りです。

```python
if 条件式:
    # 条件が満たされた（True）の時の処理
    print("条件を満たしました")
```

Pythonの特徴は、条件を満たした時の処理を **「インデント（行頭の空白）」** で表現することです。通常、半角スペース4つを使います。

## if, elif, else：複数の分岐

「もし〜ならA、そうでなくて〜ならB、それ以外ならC」という風に、複数の条件を組み合わせることもできます。

```python
score = 85

if score >= 90:
    print("大変素晴らしいです！")
elif score >= 70:
    print("合格です。")
elif score >= 40:
    print("もう少し頑張りましょう。")
else:
    print("不合格です。")
```

- `if`: 最初の条件。
- `elif`: "else if" の略。前の条件が外れた場合に、次の条件をチェックします。
- `else`: すべての条件に当てはまらなかった場合の「最終手段」です。

## 比較演算子：条件を作る道具

条件式には、2つの値を比べる **「比較演算子」** を使います。

| 記号 | 意味 | 例 |
|---|---|---|
| `==` | 等しい | `x == 10` |
| `!=` | 等しくない | `x != 0` |
| `<` | より小さい | `x < 5` |
| `>` | より大きい | `x > 10` |
| `<=` | 以下 | `x <= 8` |
| `>=` | 以上 | `x >= 0` |

ここで注意が必要なのは、**「等しい」を表すのが `==`（イコール2つ）** であることです。プログラミングにおいて `=` 1つは「代入」を意味するため、区別するために2つ重ねるのが一般的です。

## 論理演算子：複雑な条件を組み合わせる

「AかつB」「AまたはB」といった複雑な条件は、**論理演算子（`and`, `or`, `not`）** を使って作ることができます。

```python
age = 18
has_license = True

if age >= 18 and has_license:
    print("車を運転できます。")

if age < 18 or not has_license:
    print("車を運転できません。")
```

- `and`: 両方の条件が正しい時に `True`。
- `or`: どちらか片方でも正しければ `True`。
- `not`: 判定をひっくり返します（`True` なら `False`、`False` なら `True`）。

## 低レイヤーの視点：分岐の裏側と「分岐予測」

コンピュータの内部（CPU）にとって、実は `if`文は少し厄介な存在です。

CPUは、次に実行する命令をあらかじめ読み込んで準備しておく「パイプライン」という仕組みを持っています。しかし、`if`文が現れると、条件が決まるまで「次にどっちの命令を実行すればいいか」が分かりません。これが原因で計算の手が止まってしまうことがあります。

これを防ぐために、現代のCPUには **「分岐予測（*Branch Prediction*）」** という機能が備わっています。「この `if`文は、たぶん前回と同じく `True` になるだろう！」と予測して、先に計算を進めておくのです。

## 私の視点：ネストを深くしすぎない

`if`文の中にさらに `if`文を書くことを「ネスト（入れ子）」と言います。

```python
# 悪い例：ネストが深くて読みづらい
if user_exists:
    if user_active:
        if user_has_permission:
            do_something()
```

ネストが深くなると、今自分がどの条件の中にいるのか分からなくなり、バグの温床になります。そんな時は、条件を反転させて先に `return` してしまう **「ガード節」** というテクニックを使うのがお勧めです。

```python
# 良い例：ガード節でスッキリ
if not user_exists:
    return
if not user_active:
    return
if not user_has_permission:
    return

do_something()
```

### まとめ

- `if`, `elif`, `else` でプログラムの流れを枝分かれさせる。
- 比較演算子（`==`, `!=`, `<`, `>`）で条件を作る。
- 論理演算子（`and`, `or`, `not`）で条件を組み合わせる。
- CPUは内部で「分岐予測」を行っている。
- ネストを浅く保つことが、読みやすいコードへの近道。

### 展望

次は、この「条件」をさらに活用して、同じ処理を何度も繰り返す「ループ（繰り返し構文）」について学んでいきましょう。コンピュータの真骨頂は、飽きることのない単純作業の繰り返しにあります。
