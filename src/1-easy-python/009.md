# while文

「同じメッセージを100回表示して」「リストの中身を全部合計して」……そんな単純な繰り返し作業こそ、コンピュータが最も得意とする分野です。人間やLLMがやれば飽きてミスをしますが、コンピュータは文句一つ言わずに、正確に実行し続けます。

この「繰り返し」を実現するための第一の道具が`while`文です。

## `while`文の基本構造

`while`文は、指定した条件が `True`（真）である限り、中の処理を何度も実行します。逆にいうと、`Flase`になった瞬間ループを停止します。

```python
while 条件式:
    # 条件が正しい間、ここが繰り返される
    繰り返したい処理
```

最もシンプルな、そして最も危険な例が「無限ループ」です。

```python
while True:
    print("止まらないよ！")
```

`True` は常に `True` なので、このプログラムは強制終了（Ctrl+C など）しない限り、ずっと動き続けます。

## 回数を指定して繰り返す

「100回繰り返したい」という場合に、現在の回数を数えるための「カウンター変数」を用意することが昔はありました。今は悪手ですが、ループの理解には打って付けです。

```python
index = 0 # 今何回目かを数える箱

while index < 100:
    print(f"現在 {index} 回目です。")
    index = index + 1 # 1つ増やす（インクリメント）

print("終わりました！")
```

このプログラムの動きを脳内でシミュレーションしてみましょう。
1. `index` は最初 `0` です。
2. `0 < 100` は `True` なので、中身を実行します。
3. `index` が `1` になります。
4. また `1 < 100` をチェックします……。
5. `index` が `100` になった瞬間、`100 < 100` は `False` になるので、ループを抜けます。

## 0から数える

上記の例で `index` を `0` から始めたことに違和感を覚えたかもしれません。
「1回目、2回目……」と数えるなら `1` から始めたほうが自然に思えます。

しかし、一般的なでは、**「数字は0から始める」** のが鉄則です。
- 100個のデータがあるとき、番号は `0` から `99` までになります。
- 100個必要なときは、条件式を `index < 100` と書きます。

後でわかりますが、こっちの方が自然で扱いやすいです。

## `break` と `continue`

ループをさらに細かく制御するためのキーワードも紹介しておきます。

- `break`: 進行中のループをその場で強制終了します。
- `continue`: 今回の処理をスキップして、すぐに次の回のチェックに戻ります。

```python
# 奇数だけを表示する（偶数のときはスキップする）
i = 0
while i < 10:
    i += 1
    if i % 2 == 0:
        continue
    print(i)
```

これらを使いこなせると、より複雑なロジックをスッキリ記述できるようになります。ただし、可読性を下げるというデメリットも存在します。現在のトレンドでは、そもそも繰り返しは使わないのがベストです。これについては後で解説します。

### まとめ

- `while`文は条件が `True` の間、処理を繰り返す。
- 数字は `0` から数え始めるのがスタンダードで自然。

### 展望

`while`文は非常に自由度が高いですが、ある意味繰り返し的でない **`for`文** という構文があります。次は、Pythonで最もよく使われる繰り返し構文である `for`文について学んでいきましょう。
