# while文

「同じメッセージを100回表示して」「リストの中身を全部合計して」……そんな単純な繰り返し作業こそ、コンピュータが最も得意とする分野です。人間がやれば飽きてミスをしますが、コンピュータは文句一つ言わずに、正確に実行し続けます。

この「繰り返し」を実現するための第一の道具が **`while`文** です。

## `while`文の基本構造

`while`文は、指定した条件が **`True`（真）である限り**、中の処理を何度も実行します。

```python
while 条件式:
    # 条件が正しい間、ここが繰り返される
    繰り返したい処理
```

最もシンプルな、そして最も危険な例が「無限ループ」です。

```python
while True:
    print("止まらないよ！")
```

`True` は常に `True` なので、このプログラムは強制終了（Ctrl+C など）しない限り、永遠に動き続けます。

## 回数を指定して繰り返す

「100回繰り返したい」という場合は、現在の回数を数えるための **「カウンター変数」** を用意するのが一般的です。

```python
index = 0 # 今何回目かを数える箱

while index < 100:
    print(f"現在 {index} 回目です。")
    index = index + 1 # 1つ増やす（インクリメント）

print("終わりました！")
```

このプログラムの動きを脳内でシミュレーションしてみましょう。
1. `index` は最初 `0` です。
2. `0 < 100` は `True` なので、中身を実行します。
3. `index` が `1` になります。
4. また `1 < 100` をチェックします……。
5. `index` が `100` になった瞬間、`100 < 100` は `False` になるので、ループを抜けます。

ここで重要なのが、**`index = index + 1` を忘れないこと**です。これを忘れると、`index` は永遠に `0` のままなので、先ほどの無限ループになってしまいます。

## 低レイヤーの視点：ジャンプ命令とフラグレジスタ

CPUのレベルで見ると、`while`文はどのように動いているのでしょうか？

CPUには「次に実行する命令のアドレス」を指す **プログラムカウンタ（*PC*）** というレジスタがあります。通常、命令が一つ終わるたびに PC は次の命令へ進みます。

`while`文の終わりに来たとき、CPUは **「ジャンプ命令（*JMP*）」** を実行します。これにより、PC の値を強制的にループの先頭のアドレスに書き戻すのです。

また、条件分岐（`index < 100`）の判定には、CPU内部の **「フラグレジスタ」** が使われます。直前の比較演算の結果、値がマイナスになったか、ゼロになったかといった情報を記録し、そのフラグを見て「ジャンプするかしないか」を判断しているのです。

## プログラミングの習慣：0から数える

上記の例で `index` を `0` から始めたことに違和感を覚えたかもしれません。
「1回目、2回目……」と数えるなら `1` から始めたほうが自然に思えます。

しかし、プログラミングの世界（特にC言語の伝統を引き継ぐ言語）では、**「数字は0から始める」** のが鉄則です。
- 100個のデータがあるとき、番号は `0` から `99` までになります。
- 100個必要なときは、条件式を `index < 100` と書きます。

これに慣れておくと、次に学ぶ「リスト（配列）」の扱いや、他のプログラミング言語（JavaScriptやCなど）を学ぶときに非常にスムーズになります。最初は「おまじない」だと思って、0から数える癖をつけておきましょう。

## 私の視点：`break` と `continue`

ループをさらに細かく制御するためのキーワードも紹介しておきます。

- **`break`**: 進行中のループをその場で強制終了します。
- **`continue`**: 今回の処理をスキップして、すぐに次の回のチェックに戻ります。

```python
# 奇数だけを表示する（偶数のときはスキップする）
i = 0
while i < 10:
    i += 1
    if i % 2 == 0:
        continue
    print(i)
```

これらを使いこなせると、より複雑なロジックをスッキリ記述できるようになります。

### まとめ

- `while`文は条件が `True` の間、処理を繰り返す。
- カウンター変数を使って回数を制御するのが基本。
- `index = index + 1`（更新処理）を忘れると無限ループになる。
- CPUは内部で「ジャンプ命令」を使って繰り返しを実現している。
- 数字は `0` から数え始めるのがエンジニアのスタンダード。
