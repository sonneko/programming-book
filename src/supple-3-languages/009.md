# システムプログラミング言語

「システムプログラミング言語」とは、OS（オペレーティングシステム）そのものや、Web ブラウザ、ゲームエンジン、データベースといった、**「ソフトウェアの土台」** を作るための言語です。

これらの言語の共通点は、メモリや CPU といったハードウェアを直接、かつ極限まで効率的に操れることにあります。Python や JavaScript のような「楽さ」はありませんが、その代わり、コンピュータの性能を 100% 引き出すことができます。いわば、オートマ車に対する「マニュアル車」のような存在です。

ここでは、その代表格である 2 つの言語を紹介します。

## 1. C / C++ (シー / シープラスプラス)
50 年近くにわたって、プログラミングの世界を支配し続けている絶対的な王者です。

### 特徴：コンピュータの「生」の姿に触れる
C 言語には、メモリを直接指定するための **「ポインタ」** という概念があります。これは初心者が必ず挫折する難所ですが、同時に「コンピュータがどう動いているか」を理解するための最強の武器でもあります。ポインタを操ることは、コンピュータの脳内（メモリ）を直接指差し、命令することを意味します。

```c
// C言語の Hello World
#include <stdio.h>

int main() {
    printf("Hello World\n");
    return 0;
}
```

C++ は、C 言語にオブジェクト指向などの高度な機能を追加したものです。現在でも、Windows, Mac, iPhone アプリの裏側、そしてほぼすべての AAA タイトルのゲームは C++ で書かれています。Unreal Engine という強力なゲームエンジンの開発言語も C++ です。

## 2. Rust (ラスト)
11 章で詳しく紹介する、今最も熱い視線を浴びている新進気鋭の言語です。

### 特徴：安全性と速度の、歴史的な両立
これまでのシステムプログラミング言語（C/C++）の最大の悩みは、「一歩間違えるとメモリ関連の重大なバグ（やセキュリティホール）が簡単に発生してしまう」ことでした。プログラマの不注意で、他人の秘密のデータを読み取れてしまうようなミスが、世界中の深刻な脆弱性の原因となってきました。

Rust は、**「所有権（Ownership）」** という画期的な仕組みを導入することで、C/C++ 並みの超高速な実行速度を保ちつつ、メモリの安全性を「コンパイル時に」100% 保証することに成功しました。ミスがあればコンパイラが「それは危ないよ！」と容赦なく叱ってくれます。

```rust
// Rust の Hello World
fn main() {
    println!("Hello World");
}
```

Google や Microsoft, Amazon といった名だたる企業が、自社の重要なシステムを C++ から Rust へと書き換え始めています。

## なぜこれらの言語を学ぶべきか？
「自分は Web アプリやスマホアプリが作りたいだけだから、こんな難しい言語は関係ない」と思うかもしれません。

しかし、あなたが使っているブラウザの V8 エンジンは C++ で書かれています。あなたが使っている Python のインタプリタそのものも C 言語で書かれています。低レイヤーの言語を知ることは、あなたが普段使っている「魔法」のタネ明かしを知ることに他なりません。

また、システムプログラミング言語を学ぶことで、**「計算リソースの有限性」** を強く意識するようになります。メモリを一バイト単位で節約し、一秒でも早く処理を終わらせる。そんな「極限のエンジニアリング」を経験することで、あなたの書く Python や JavaScript のコードも、より洗練されたものに変わっていくはずです。

## 学習の難易度と達成感
正直に言いましょう。システムプログラミング言語の学習難易度は非常に高いです。
- コンパイルエラーが呪文のように見える。
- セグメンテーション違反（メモリ不正アクセス）でプログラムが即座に落ちる。
- CPU のキャッシュや、スタックとヒープといった知識が要求される。

しかし、その苦労を乗り越えて「ハードウェアを手懐けた」という感覚を持てるようになったとき、あなたのエンジニアとしての視座は、他の誰よりも高くなっているはずです。自分の書いたコードが、OS も介さず直接 CPU のレジスタを動かしている——その全能感は、一度味わうと病みつきになります。

## まとめ
システムプログラミング言語は、コンピュータという荒馬を乗りこなすための手綱です。
最初は振り落とされることもあるでしょう。しかし、それを御することができるようになったとき、あなたは IT 世界の「構造そのもの」を作り変える力を手にします。

まずは 10 章で C 言語の洗礼を受け、11 章で Rust という現代の英知に触れてみてください。コンピュータの深淵が、あなたを待っています。そこから帰還したとき、あなたはもう以前のあなたではないはずです。
