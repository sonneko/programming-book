# スクリプト系言語

「スクリプト系言語」という言葉に厳密な定義はありませんが、一般的には **「コンパイルという面倒な手順を踏まずに、書いたらすぐに実行できる、人間にとって読みやすい言語」** のことを指します。

Python や JavaScript もこの仲間です。これらの言語は、開発のスピードが極めて速く、書きやすいため、初心者からプロのエンジニアまで幅広く愛されています。

ここでは、Python と JavaScript 以外で、エンジニアなら知っておくべき 2 つのスクリプト系言語を紹介します。

## 1. Ruby (ルビー)
まつもとゆきひろ（Matz）氏という日本人が開発した、世界中で使われている言語です。

### 特徴：プログラミングを楽しむための言語
Ruby の最大の哲学は **「Enjoy Programming」** です。コンピュータのためではなく、人間が楽しく、直感的に書けるように設計されています。「エンジニアがストレスなく、自分の思いをコードに乗せられるように」という Matz 氏の優しさが、言語の至る所から感じられます。

```ruby
# Ruby の FizzBuzz 例
1.upto(100) do |i|
  if i % 15 == 0
    puts "FizzBuzz"
  elsif i % 3 == 0
    puts "Fizz"
  elsif i % 5 == 0
    puts "Buzz"
  else
    puts i
  end
end
```
`1.upto(100)`（1 から 100 まで上がれ）という書き方、英語の文章みたいで面白いですよね。Ruby を書いていると、まるでコンピュータと「お喋り」をしているような感覚になります。

### 主な用途：Web 開発
**Ruby on Rails** という非常に強力な Web フレームワーク（アプリの雛形）が登場したことで、Ruby はスタートアップ企業などの Web 開発で爆発的に普及しました。「クックパッド」や「GitHub」も、初期の段階では Ruby で作られていました。開発効率が極めて高いため、少人数で素早くサービスを立ち上げるのに最適です。

## 2. PHP (ピー・エイチ・ピー)
Web サイトの裏側を支える、Web 専用のスクリプト言語です。

### 特徴：Web のための実用主義
PHP は最初から「HTML に直接プログラムを埋め込む」ことを目的として作られました。そのため、Web サーバーへの導入が非常に簡単で、多くのレンタルサーバー（XServer など）で標準サポートされています。

```php
<?php
// PHP の Hello World 例
echo "Hello World";
?>
```

### 主な用途：WordPress と Web システム
世界中の Web サイトの 4 割以上を占めると言われる **WordPress** は、PHP で書かれています。そのため、Web 制作の現場では PHP を避けて通ることはできません。また、Laravel というモダンなフレームワークの登場により、大規模な Web サービスの開発でも再び注目を集めています。

## スクリプト言語の裏側：インタプリタ
スクリプト言語は、なぜコンパイルが不要なのでしょうか？
それは、**「インタプリタ（通訳者）」** というプログラムが、あなたの書いたコードを一行ずつ読み取り、その場で CPU が理解できる形に変換して実行しているからです。

- **メリット**: 書いてすぐに動かせる。エラーの箇所が分かりやすい。
- **デメリット**: コンパイルする言語（C や Rust）に比べると、実行速度が少し遅い。

しかし、現代のコンピュータは十分に高速なため、大抵の Web アプリやツールであれば、スクリプト言語の速度で全く問題ありません。むしろ、人間の「考える時間」や「書く時間」を短縮できるメリットの方が、多くの場面で上回ります。

## どのスクリプト言語を選ぶべき？
- **AI やデータ分析をしたい**: Python 一択です。
- **Web サイトに動きをつけたい**: JavaScript が必須です。
- **Web 制作（WordPress など）を仕事にしたい**: PHP が強いです。
- **楽しく、直感的にコードを書きたい**: Ruby がお勧めです。

## まとめ
スクリプト系言語は、あなたのアイデアを最も手軽に形にしてくれる「魔法の筆」です。まずは一つ（例えば Python）を極め、その後に他の言語を覗いてみてください。文法の違いに「お、この言語はこんな工夫がされているのか！」と気づけるようになれば、プログラミングがどんどん楽しくなってくるはずです。

「コンピュータをどう動かすか」よりも「何を解決したいか」に集中できるのが、スクリプト言語の最大の魅力です。ぜひ、自分に合った「筆」を見つけてみてください。
