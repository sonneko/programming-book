# 各言語の共通点

世の中には、Python, JavaScript, C, Java, Rust, Go... 数えきれないほどのプログラミング言語が存在します。新しい言語が登場するたびに「また覚え直しなの？」と絶望的な気持ちになるかもしれませんが、安心してください。

実は、あらゆるプログラミング言語には、驚くほど多くの **「共通点」** があります。

本書の冒頭で「Python を学ぶのではなく、Python で（プログラミングを）学ぶ」と述べたのは、このためです。一つの言語を深く理解すれば、他の言語を習得するスピードは 2 倍、3 倍と加速します。なぜなら、彼らはみな同じ「コンピュータ」という融通の利かない機械を操るために生まれた「方言」に過ぎないからです。

## 1. データの保存：変数と型
どんな言語であっても、データを一時的に記憶しておくための「箱（変数）」が必要です。そして、その箱の中に何を入れるか（整数、浮動小数点数、文字列、真偽値など）という「型」の概念も共通しています。

- **静的型付け言語**（C, Java, Rust など）: 箱の中身が何かをあらかじめ厳密に決めておく。
- **動的型付け言語**（Python, JavaScript など）: 中身を見てから、その都度決める。

これらは、どちらが良い・悪いという話ではなく、「安全性を重視するか（静的）」「柔軟性を重視するか（動的）」という設計思想の違いに過ぎません。

## 2. 処理の流れ：制御構文
プログラムを上から下へ、条件によって枝分かれさせたり、同じことを繰り返させたりする仕組みも共通です。

- **順次**: 上から順に実行する。
- **分岐**: `if` 文。
- **反復**: `for` 文や `while` 文。

この 3 つの組み合わせ（構造化プログラミング）があれば、理論上、どんなに複雑なプログラムも書くことができます。

## 3. 処理の再利用：関数とスコープ
同じ処理を何度も書かなくて済むように、一連の流れに名前をつけてまとめるのが「関数」です。そして、その関数の中で定義した変数がどこまで有効か、という「スコープ」の考え方も、ほぼすべての言語で共通しています。

## 4. データの集合体：配列とオブジェクト
一つのデータだけでなく、複数のデータをまとめて扱うための仕組みも必ずあります。
- **配列（リスト）**: 順番に並んだデータの列。
- **オブジェクト（連想配列、辞書）**: 「キー（名前）」と「値」のペアの集まり。

10 章の「低レベルプログラミング」を学ぶと、これらが内部的には「メモリ上の連続した領域」であったり、「特定のアルゴリズムに基づいたハッシュテーブル」であったりすることが分かります。

## 5. エラーとの対峙：例外処理
プログラムは必ずいつかエラー（例外）を起こします。そのエラーを検知し、安全に処理を続行したり、適切に終了させたりするための `try...catch`（Python では `try...except`）のような仕組みも、多くの言語が共通して備えています。

## なぜ共通点が多いのか？：ハードウェアという制約
なぜ、これほどまでに似ているのでしょうか。それは、すべてのプログラミング言語が最終的に行き着く先が **CPU（中央演算処理装置）** と **メモリ** というハードウェアだからです。

CPU は、「メモリのここにある値を、レジスタに読み込んで、足し算をして、またメモリに戻す」といった単純な命令しか実行できません。どれほど高級な言語であっても、最終的にはこの「計算機の現実」に従わなければなりません。だからこそ、言語は違えど、その根底にある概念（アナロジー）は共通してくるのです。

## 「知的な悟り」への道
新しい言語に出会ったとき、「この言語の for 文はどう書くんだろう？」「変数の定義はどうやるんだろう？」という視点で眺めてみてください。すると、新しい知識をゼロから覚えるのではなく、すでに持っている知識（Python など）を「変換」しているだけであることに気づくはずです。

この感覚こそが、私が「知的な悟り」と呼んでいる状態の入り口です。言語の壁を超えて、その裏にある **ロジック（論理）** と **データフロー（データの流れ）** が透けて見えるようになる。そこまで来れば、あなたはもうどんな新しい技術が登場しても、恐れることはありません。

## まとめ
プログラミング言語は道具です。ハンマー（Python）の使い方を覚えたら、次はドライバー（JavaScript）を手に取ってみましょう。形は違えど、「何かを作る」という目的は同じです。

大切なのは文法を丸暗記することではなく、その文法が解決しようとしている「問題の本質」を掴むことです。さあ、次のページからは、様々な言語の具体的な顔ぶれを見ていくことにしましょう。
