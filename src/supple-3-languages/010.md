# 関数型言語

「プログラミングは、命令のリストである」。多くの人がそう思っています。
しかし、その常識を根本から覆すのが **「関数型言語（Functional Programming Languages）」** です。

関数型言語の世界では、プログラミングは「命令」ではなく **「データの変換」** です。数学の関数と同じように、ある入力を受け取って、新しい結果を返す。その「写像（マッピング）」を組み合わせていくことで、複雑なプログラムを構築します。

12 章で詳しく学びますが、ここではその入り口となる 2 つの言語を紹介します。

## 1. Haskell (ハスケル)
「純粋関数型言語」の代名詞であり、多くのプログラマにとっての「畏怖の対象」でもある言語です。

### 特徴：数学的な純粋さ
Haskell には、変数の値を書き換える（代入する）という概念がありません。また、ループ（for 文）もありません。
「えっ、どうやってプログラムを書くの？」と思うでしょう。答えは **「再帰」** と **「高階関数」** です。

```haskell
-- Haskell の FizzBuzz
fizzbuzz n
  | n `mod` 15 == 0 = "FizzBuzz"
  | n `mod` 3  == 0 = "Fizz"
  | n `mod` 5  == 0 = "Buzz"
  | otherwise       = show n

main = mapM_ (putStrLn . fizzbuzz) [1..100]
```
この美しく、簡潔な記述。命令を並べるのではなく、データの変換ルールを定義していることが分かります。

## 2. Lisp / Clojure (リスプ / クロージャ)
1958 年に誕生した、現役で使われている言語としては 2 番目に古い歴史を持つ言語の一族です。

### 特徴：コードとデータが同じ形をしている
Lisp の最大の特徴は、大量の「括弧 `()` 」です。

```clojure
;; Clojure (Lisp系) の Hello World
(println "Hello World")

;; FizzBuzz の一部
(if (= (mod i 15) 0) "FizzBuzz" ...)
```
プログラムそのものが「リスト」というデータ形式で表現されているため、**「プログラムを書き換えるプログラム」** を作るのが驚くほど簡単です。この強力な機能は「マクロ」と呼ばれ、AI（人工知能）研究の初期から愛されてきました。

## 関数型言語を学ぶメリット
なぜ、わざわざ難しい関数型言語を学ぶ必要があるのでしょうか？

1. **バグの削減**: 「状態（変数の中身）」が変わらないため、バグの入り込む余地が劇的に減ります。
2. **並列処理に強い**: データの書き換えが起きないため、複数の CPU で同時に計算を行っても、データの不整合が起きません。
3. **抽象的な思考力**: 関数型言語を学ぶと、他の言語（Python や JS）を書くときにも、「この処理はもっと簡潔に変換として書けるのではないか？」という視点を持てるようになります。

## 現代の言語への影響
実は、あなたが今使っている言語も、関数型言語の影響を強く受けています。
- JavaScript の `map`, `filter`, `reduce`
- Python の `リスト内包表記`
- Java や C# の `ラムダ式`
- Swift や Rust の `パターンマッチング`

これらはすべて、関数型言語から「輸入」された便利な機能です。つまり、関数型言語を学ぶことは、現代プログラミングの「源流」を辿る旅でもあるのです。

## まとめ
関数型言語は、最初は「異世界の言語」のように感じるかもしれません。しかし、その「知的な衝撃」こそが、あなたのエンジニアとしての思考を一段階上のレベルへと引き上げてくれます。

12 章でその深淵に触れるのを楽しみにしてください。きっと、昨日までのプログラミングとは全く違う景色が見えてくるはずです。
