# cssセレクター

CSS の文法を学んだとき、「セレクターは『どこを』装飾するかを指定するもの」だとお話ししました。しかし、実際の Web サイト制作では、単に `h1` や `p` といったタグ名だけで指定することは稀です。

特定のボタンだけを赤くしたり、リストの 3 番目だけを太字にしたり……。そんな「狙った要素をピンポイントで捕まえる」ための高度なテクニックをマスターしましょう。

## 1. 三種の神器：要素、クラス、ID

もっとも基本的なセレクターは、以下の 3 つです。

### 要素セレクター
タグ名そのものを指定します。
```css
p { color: gray; } /* すべての p タグに適用 */
```

### クラスセレクター
HTML で `class="name"` と名付けた要素に適用します。先頭に `.`（ドット）を付けます。
**これが最も多用されます。** なぜなら、同じクラス名を複数の要素に付けられるので、スタイルを使い回せるからです。
```css
.highlight { background: yellow; }
```

### ID セレクター
HTML で `id="name"` と名付けた要素に適用します。先頭に `#`（ハッシュ）を付けます。
1 ページに 1 つしか存在してはいけないというルールがあり、優先順位が非常に高いのが特徴です。
```css
#main-header { font-size: 32px; }
```

## 2. 関係性で捕まえる：子孫と子

要素の構造（木構造）を利用して指定する方法です。

- **子孫セレクター (`A B`)**: A の中にある B すべてを指定。
  ```css
  div p { color: red; } /* div の中にある p タグすべて（孫やひ孫も含む） */
  ```
- **子セレクター (`A > B`)**: A の直下にある B だけを指定。
  ```css
  ul > li { list-style: none; } /* ul のすぐ下にある li だけ */
  ```

## 3. 状態や順番で捕まえる：擬似クラス

「ユーザーがマウスを乗せたとき」や「奇数番目の要素」など、HTML の構造だけでは表現できない「状態」を指定するのが **擬似クラス (Pseudo-classes)** です。

```css
/* マウスが乗ったとき */
button:hover {
  opacity: 0.8;
  cursor: pointer;
}

/* リンクを訪問済みのとき */
a:visited {
  color: purple;
}

/* 3 番目の要素だけ */
li:nth-child(3) {
  color: orange;
}

/* 奇数番目だけ */
li:nth-child(odd) {
  background: #f0f0f0;
}
```

## 低レイヤーの視点：セレクターの「マッチング」コスト

## 私の視点：ID セレクターを封印してみる

私が初心者の頃は ID を使いがちでしたが、実は現代のプロの開発では **「ID セレクターを CSS で使わない」** というルールを設けることが多いです。

理由は、ID の優先順位（詳細度）が高すぎて、後からスタイルを上書きしたくなったときに `!important` などの無理やりな手段を使わざるを得なくなるからです。
基本はすべて「クラス」で設計し、ID は JavaScript で要素を特定するためだけに使う。そんな設計を意識すると、CSS の管理がグッと楽になりますよ。

### まとめ

- 基本は `要素`, `.クラス`, `#ID` の 3 種類。
- 関係性（`A B`, `A > B`）を使いこなすと HTML がスッキリする。
- `:hover` などの擬似クラスで、動きのあるデザインを実現できる。
- ブラウザはセレクターを「右から左」に解析する。
- 優先順位を制御しやすくするため、基本は「クラス」を使おう。

### 展望

セレクターでターゲットを決めたら、次は具体的に「どう飾るか」というプロパティの知識を深めましょう。特に、すべてのレイアウトの基本となる「ボックスモデル」を攻略します。
