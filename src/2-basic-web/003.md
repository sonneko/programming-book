# cssの文法

HTML が「骨組み（構造）」だとしたら、**CSS (Cascading Style Sheets)** は「化粧（装飾）」です。CSS を使うことで、文字の色を変えたり、要素を横に並べたり、アニメーションを加えたり、私たちが「Web デザイン」と呼ぶあらゆる視覚効果を実現できます。

私が初めて CSS で背景色を変えたとき、自分のコードが「デザイン」に変わる瞬間にゾクゾクしたのを覚えています。

## 基本的な書き方：セレクター、プロパティ、値

CSS の基本形は、以下の 3 つの要素で構成されます。

```css
h1 {
  color: blue;
  font-size: 24px;
}
```

1. **セレクター (Selector)**: 「どこを」装飾するか（ここでは `h1` タグ）。
2. **プロパティ (Property)**: 「何を」変えるか（ここでは `color` や `font-size`）。
3. **値 (Value)**: 「どう」するか（ここでは `blue` や `24px`）。

プロパティと値の間には `:`（コロン）を置き、最後には必ず `;`（セミコロン）を付けます。この `;` を忘れると、次の行のスタイルが読み込まれず、原因不明の「スタイルが適用されない」問題に悩まされることになるので注意しましょう。

## CSS をどこに書くか？（3つの方法）

CSS を HTML に適用する方法は主に 3 つありますが、現場では **「外部ファイル方式」** が圧倒的に使われます。

1. **外部ファイル方式（推奨）**:
   `.css` という拡張子のファイルを別途作り、HTML の `head` 内で読み込みます。
   ```html
   <link rel="stylesheet" href="style.css">
   ```
   HTML（構造）と CSS（デザイン）を完全に切り分けることで、管理が非常に楽になります。

2. **内部スタイルシート方式**:
   HTML の `head` 内に `<style>` タグを書いて、その中に直接記述します。特定の 1 ページだけに適用したいときに使われます。

3. **インラインスタイル方式**:
   HTML タグに直接 `style="..."` 属性を書きます。
   ```html
   <p style="color: red;">この文だけ赤くする</p>
   ```
   緊急の修正などには便利ですが、多用すると HTML が読みづらくなる（「密結合」になる）ため、極力避けるべきとされています。

## カスケーディング：スタイルの優先順位

CSS の名前にある **Cascading（カスケーディング：滝のように流れる）** とは、複数のスタイルが衝突したときに、どのルールを優先するかを決める仕組みのことです。

例えば、ある要素に対して 3 つの場所から色が指定されていたら、どうなるでしょうか？

```css
p { color: blue; } /* ルールA */
.red-text { color: red; } /* ルールB */
#main-paragraph { color: green; } /* ルールC */
```

優先順位は、**「詳細度（Specificity）」** という点数計算で決まります。
- **ID セレクター (`#`)**: 100点（最強）
- **クラス セレクター (`.`)**: 10点
- **要素 セレクター (`p`)**: 1点

## 低レイヤーの視点：スタイルの再計算（Recalc Style）

ブラウザの内部では、CSS はどのように処理されているのでしょうか。

ブラウザは HTML を解析して **DOM** を作るのと同時に、CSS を解析して **CSSOM** (*CSS Object Model*) という木構造を作ります。
そして、この 2 つを合体させて **「レンダーツリー（*Render Tree*）」** を構築します。

CSS を 1 行書き換えるたびに、ブラウザは「この変更は画面のどの範囲に影響するか？」を計算し直します。
例えば、文字の色を変えるだけなら高速ですが、要素の幅（`width`）を変えると、その周囲の要素すべての位置がずれるため、再計算（*Layout/Reflow*）に大きな負荷がかかります。
パフォーマンスの良い Web サイトを作るには、ブラウザが「楽に計算できる」ような CSS を書く視点が欠かせません。

## 私の視点：CSS は「論理」である

よく「デザインはセンスだ」と言われますが、コーディングにおける CSS は、むしろ「数学」や「論理」に近いです。
「なぜこの要素がここに配置されているのか」「なぜこの色が優先されているのか」は、すべてルールによって説明がつきます。

感覚で書くのではなく、CSS の仕組み（ボックスモデル、詳細度、継承など）を理解して書くようになると、どんな複雑なデザインもパズルのように組み立てられるようになりますよ。

### まとめ

- CSS は「セレクター { プロパティ: 値; }」の形式で書く。
- HTML とは別の `.css` ファイルに書くのが現代のスタンダード。
- 「カスケーディング」によって、優先順位が自動的に決まる。
- ブラウザ内部では DOM と CSSOM が合体して画面が作られる。
