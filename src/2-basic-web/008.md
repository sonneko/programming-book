# アニメーション

Web サイトに「動き」を加えることは、単に見栄えを良くするだけではありません。ボタンを押したときに少し凹む、メニューがスッと出てくる……といった適切なフィードバックは、ユーザーに「今、何が起きているか」を直感的に伝え、体験 (UX) を劇的に向上させます。

私が初めて CSS で要素を動かしたとき、ただのテキストだった Web ページが「生き物」のように感じられたのを覚えています。

## 1. 滑らかに変化させる：Transition

もっとも簡単で使いやすいのが、ある状態から別の状態へ「時間をかけて変化させる」 **`transition`** プロパティです。

```css
.button {
  background-color: blue;
  color: white;
  padding: 10px 20px;
  /* 背景色の変化を 0.3秒かけて 滑らかにする */
  transition: background-color 0.3s ease-in-out;
}

.button:hover {
  background-color: darkblue;
}
```

この指定がないと、マウスを乗せた瞬間に色がパッと切り替わりますが、`transition` を入れることで「ふわっ」と色が変わるようになります。

- **`ease-in-out`**: 「ゆっくり始まって、最後もゆっくり終わる」という加速・減速の指定です。これにより、動きがより自然（人間的）に見えます。

## 2. 複雑な動きを作る：Keyframes

「ページを開いたときに要素が回る」「ずっと浮き沈みしている」といった、より複雑なアニメーションには **`@keyframes`** を使います。

```css
/* アニメーションの内容（コマ割り）を定義 */
@keyframes float {
  0% { transform: translateY(0); }
  50% { transform: translateY(-20px); }
  100% { transform: translateY(0); }
}

.floating-box {
  /* 定義したアニメーションを適用 */
  animation: float 2s infinite ease-in-out;
}
```

- **`infinite`**: アニメーションを無限に繰り返します。
- **`alternate`**: 行って、戻って……という交互の動きにします。

## 低レイヤーの視点：パフォーマンスと GPU

アニメーションを多用すると、ブラウザが「カクつく」ことがあります。
プロのフロントエンドエンジニアは、アニメーションさせるプロパティを厳選します。

### 悪い例：`width`, `height`, `top`, `left`
これらを変えると、その要素だけでなく、周りの要素すべての位置を再計算（**Layout/Reflow**）する必要があります。これは CPU に多大な負荷をかけ、カクつきの原因になります。

### 良い例：`transform` (移動、回転、拡大), `opacity` (不透明度)
これらは「コンポジット・レイヤー」という仕組みを使って、CPU ではなく **GPU（グラフィックス処理ユニット）** で処理されます。

GPU は画像の処理に特化した計算機なので、複雑な移動や回転もヌルヌルと高速にこなしてくれます。
「アニメーションさせるなら transform と opacity」。これを守るだけで、あなたのサイトの品質は一気に「プロ級」になります。

## 私の視点：「やりすぎ」は毒になる

アニメーションを覚えると、あちこちを動かしたくなるものです。でも、注意してください。
あちこちで要素がふわふわしたり、ビカビカ光ったりするサイトは、ユーザーにとって「情報の邪魔」でしかありません。

「ここがクリックできるよ」「ここが注目すべき情報だよ」という **ユーザーの視線誘導** のために、スパイスとして控えめに使う。それが、洗練された Web サイトを作るためのデザインの極意です。

### まとめ

- `transition` は、マウスホバーなどの「状態の変化」を滑らかにする。
- `@keyframes` は、自由な動き（コマ割り）を作れる。
- アニメーションは CPU 負荷が高いため、GPU を活用できる `transform` や `opacity` を使うのが鉄則。
- 動きは「ユーザーの使いやすさ（UX）」のために使うべきで、過度な演出は避ける。

### 展望

アニメーションで視覚的な楽しさを覚えたら、次は Web サイトの「土台」となるレイアウトの仕組み――「ブロック要素」と「インライン要素」の深い話に戻りましょう。なぜ要素が横に並んだり縦に並んだりするのか、その根本的な理由を解き明かします。
