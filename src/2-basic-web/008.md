# アニメーション

Webサイトに動きを加えることで、ユーザー体験 (UX) を向上させることができます。

### Transition（遷移）

最も簡単なアニメーションは、状態が変わったときに滑らかに変化させる `transition` です。

```css
button {
  background: blue;
  transition: background 0.3s ease-in-out;
}

button:hover {
  background: red;
}
```

「マウスが乗ったときに、背景色を0.3秒かけて変化させる」という指定です。

### Keyframes（コマ割り）

より複雑な動きを作りたいときは、`@keyframes` を使います。

```css
@keyframes slidein {
  from { transform: translateX(-100%); }
  to { transform: translateX(0); }
}

.box {
  animation: slidein 1s infinite alternate;
}
```

### 低レイヤの視点：パフォーマンスとGPU

アニメーションを多用するとサイトが重くなることがあります。シニアエンジニアは「何を使って動かすか」を重視します。

-   **悪い例**: `left`, `top`, `width` などを動かす。
-   **良い例**: `transform` (translate, scale, rotate) や `opacity` を動かす。

ブラウザには「コンポジット・レイヤー」という概念があります。`transform` などの特定のプロパティは、CPUではなく **GPU（グラフィックス処理ユニット）** で処理されるため、画面のカクつきが起こりにくく、バッテリー消費も抑えられるのです。

「ただ動けばいい」のではなく、「コンピュータの資源を効率よく使って動かす」という視点を持ちましょう。
