# ブロック要素とインライン要素

CSSでレイアウトを組む際に必ず直面するのが、「要素がどう並ぶか」というルールです。

### 2つの基本性質

1.  **ブロック要素 (`display: block`)**
    -   画面の横幅いっぱいを占領する。
    -   前後に改行が入る。
    -   例: `<div>`, `<h1>`, `<p>`, `<ul>`
2.  **インライン要素 (`display: inline`)**
    -   中身の文章に必要な分だけ幅を取る。
    -   横に並ぶ。
    -   幅 (`width`) や高さ (`height`) を指定できない。
    -   例: `<span>`, `<a>`, `<strong>`

### 性質は変えられる

実はこれらの性質は、CSSの `display` プロパティで自由に変更できます。

```css
span {
  display: block; /* spanなのにブロック要素のように振る舞う */
}
```

最近では、これらの中間のような性質を持つ `inline-block` や、次節で解説する `flex` など、より高度な並び方の指定が主流になっています。

### なぜこの区別があるのか？

元々、HTMLは「文書（論文のようなテキスト）」を共有するために作られました。文章の中に埋め込まれる「語句（インライン）」と、文章の区切りとなる「段落（ブロック）」という区別は、文書作成における自然な形だったのです。

現代のWeb制作では、この「文書のための性質」を `display` プロパティで「アプリケーションのためのレイアウト」に上書きしていく作業が中心になります。
