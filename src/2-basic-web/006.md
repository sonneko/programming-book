# よく使うcssプロパティ

CSS のプロパティは膨大な数があり、すべてを丸暗記するのは不可能です。プロのエンジニアも、必要に応じてリファレンスを引きながら書いています。

しかし、**「これだけは理解していないとレイアウトが組めない」** というコアなプロパティがいくつか存在します。特に、すべての要素に共通する「ボックスモデル」の概念は、CSS 攻略の最重要ポイントです。

## 1. CSS の神髄：ボックスモデル

CSS の世界では、すべての要素（見出し、画像、段落など）は **「四角い箱（ボックス）」** として扱われます。この箱は、以下の 4 つの層で構成されています。

1. **Content (コンテンツ)**: テキストや画像が実際に表示されるエリア。
2. **Padding (パディング)**: コンテンツと枠線の間の **「内側の余白」**。
3. **Border (ボーダー)**: 箱を囲む **「枠線」**。
4. **Margin (マージン)**: 枠線の外側の **「外側の余白」**。他の要素との距離を保つのに使います。

```css
.box {
  width: 300px;
  padding: 20px;
  border: 5px solid black;
  margin: 10px;
}
```

ここで初心者が必ずハマるのが、**「箱の合計幅」** の計算です。
デフォルトでは、`width` は 1 の「コンテンツ」の幅だけを指します。つまり、上の例だと全体の幅は `300 + 20*2 + 5*2 = 350px` になります。
これを直感的に（`width` を全体の合計幅にする）するために、最近は以下の設定を「おまじない」として最初に入れるのが主流です。
```css
* { box-sizing: border-box; }
```

## 2. 文字と色の装飾

見た目を整えるための、基本のプロパティです。

- **`color`**: 文字の色。`red` などの名前や、`#ff0000` などの 16 進数、`rgb(255, 0, 0)` で指定します。
- **`font-size`**: 文字の大きさ。`px` 以外に `rem`（後述）などが使われます。
- **`font-weight`**: 文字の太さ。`bold` など。
- **`text-align`**: 行の中での文字の揃え方（`left`, `center`, `right`）。
- **`background-color`**: 背景の色。

## 3. 単位の使い分け：px, %, rem

CSS で数値を指定するときの「単位」選びは、モダンな Web 制作では非常に重要です。

- **`px` (ピクセル)**: 画面の最小単位に基づく固定値。分かりやすいですが、スマホで見ると小さすぎるといった問題が起きやすいです。
- **`%`**: 親要素の大きさを基準とした相対値。横幅の指定などによく使われます。
- **`rem` (レム)**: ページ全体の基本のフォントサイズ（通常 16px）を `1` とした相対値。**「アクセシビリティ」の観点から、フォントサイズにはこれを使うのが推奨されます。**（ユーザーがブラウザ設定で文字を大きくしたときに追随できるため）

## 低レイヤーの視点：ピクセルと解像度

私たちが CSS で `1px` と指定したとき、それはスマホの液晶の「1ドット」と同じだと思っていませんか？

実は、iPhone などの高精細なディスプレイ（Retina ディスプレイ）では、**1つの「論理ピクセル」を 2〜3 個の「物理ピクセル」で描画しています。**

これを **デバイスピクセル比 (DPR)** と呼びます。もし画像を表示するとき、CSS で指定したサイズぴったりの画像を用意すると、スマホではボケて見えてしまいます。そのため、実際には 2 倍や 3 倍の大きさの画像を用意して、CSS でギュッと縮めて表示するのが、綺麗なサイトを作るコツなのです。

## 私の視点：マージンの「相殺」に気をつけろ

CSS を書いていて、「上下のマージンが思ったより狭いな？」と思ったら、それは **「マージンの相殺 (Margin Collapse)」** が起きているかもしれません。
縦に並んだ 2 つの要素にそれぞれ `margin: 20px` を設定したとき、合計は 40px にならず、重なり合って 20px になってしまいます。

これは CSS の仕様の中でもかなりトリッキーな部分です。「なぜ？」と悩むより、「そういう仕様なんだ」と受け入れて、`padding` で調整するようにすると、ストレスが減りますよ。

### まとめ

- すべての要素は `Padding`, `Border`, `Margin` を持つ「ボックス」である。
- `box-sizing: border-box;` は最初に入れておくと計算が楽。
- 単位は固定（`px`）だけでなく、相対（`%`, `rem`）を使い分けよう。
- スマホでは 1px が複数のドットで描画される（DPR）。
- 上下マージンの相殺など、CSS 特有のクセに注意。
