# GitHubへの登録とGitのインストール

Git の仕組みを学んだら、次はその力を 100% 引き出すための環境を整えましょう。
自分の PC に Git をインストールし、世界最大のコード共有プラットフォームである **GitHub** と連携させることで、あなたの開発ライフは一気に「プロ」の領域へと突入します。

## 1. 開発のハブ：GitHub とは？

Git はあくまで「自分の PC 上で履歴を管理するツール」です。一方の GitHub は、その履歴を **「インターネット上に保管し、公開・共有するためのサービス」** です。

単なるバックアップ先ではありません。
- 世界中の開発者が書いたコード（オープンソース）を読んだり、使ったりできる。
- 自分のコードを他人にレビュー（チェック）してもらう。
- バグの報告（Issue）や、機能の提案（Pull Request）を行う。

GitHub を使うことは、世界中のエンジニアが集まる巨大なコミュニティに参加することと同義です。

## 2. インストールと初期設定の儀式

まずは自分の PC で Git が動くようにしましょう。

- **Windows**: [Git for Windows](https://gitforwindows.org/) をインストールします。
- **Mac**: ターミナルで `git --version` と打つと、インストールを促されるか、既に入っていることが確認できます。

インストール後、**「誰がこの変更をしたか」** という署名を登録します。これは履歴に残る非常に重要な情報です。

```bash
git config --global user.name "あなたの名前"
git config --global user.email "あなたのメールアドレス"
```

## 3. 安全な通信の鍵：SSH 設定

PC から GitHub にコードをアップロード（Push）するとき、毎回パスワードを入力するのは面倒ですし、セキュリティ上もあまり良くありません。そこで **SSH (Secure Shell)** という仕組みを使います。

1. 自分の PC で「鍵のペア（秘密鍵と公開鍵）」を作ります。
2. 「公開鍵」だけを GitHub に登録します。
3. 通信の際、GitHub が「君、本物の鍵（秘密鍵）持ってる？」と確認し、持っていればアクセスを許可します。

## 低レイヤーの視点：ハッシュ値による身分証明

Git で設定する `user.email` は、なぜ重要なのでしょうか？
Git のコミット ID（例: `abc1234...`）は、**「変更内容 + 親コミットの ID + 著者情報 + 日時」** をすべて混ぜ合わせてハッシュ計算した結果です。

著者の名前やメールアドレスが 1 文字でも変われば、生成されるコミット ID も完全に変わってしまいます。
つまり、Git において著者の情報は「ただのメモ」ではなく、その履歴の正当性を証明するための **「データの一部」** なのです。

## 私の視点：GitHub はあなたの「履歴書」になる

これからのエンジニアにとって、GitHub のプロフィールページは、どんな履歴書よりも雄弁にあなたの技術力を語ります。

「まだ初心者だから恥ずかしい」なんて思う必要はありません。
- どんな技術に興味を持ち、どんな学習を重ねてきたか。
- 失敗したコードをどう直してきたか。
- 他人のコードに対してどう接しているか。

それらすべてが GitHub には記録されます。今日から Git の練習を始めるその一歩が、数年後のあなたを素晴らしいチームへと導く「道しるべ」になるはずですよ。

### まとめ

- Git はローカル用、GitHub はクラウド（共有）用。
- インストール後は `git config` で署名を忘れずに。
- SSH 設定を行うことで、安全かつ快適に GitHub と通信できる。
- 著者情報はコミットハッシュを構成する重要なデータの一部。
- GitHub はエンジニアとしての成長を記録する最高のプラットフォーム。

### 展望

環境が整ったら、次は Git の具体的なコマンドを叩いてみましょう。`add`, `commit`, `push`……。この 3 つの呪文を覚えるだけで、あなたのコードは世界とつながります。
