# GitHubへの登録とGitのインストール

GitをPCに導入し、GitHubというホスティングサービスを活用することで、コードのバックアップとチーム連携が可能になります。

### インストール手順

1. **Gitのインストール**: 公式サイトまたはパッケージマネージャ (`brew install git`, `sudo apt install git` など) からインストールします。
2. **GitHubアカウント作成**: GitHub.comにサインアップします。
3. **SSH設定**: ローカルのPCとGitHubをSSH接続することで、安全な通信が可能になります。

### 詰まりポイントと解決策

- **SSH鍵のパスフレーズ忘れ**: 鍵生成時に設定したパスフレーズを忘れると、次回からSSH接続できなくなる。安全な場所で管理すること。
- **Gitユーザー設定ミス**: `git config` で名前やメールアドレスを間違えてコミットすると、GitHub上で正しいアカウントと紐付かない。`git config --list` で確認。
- **リモートURLの混同**: HTTPSとSSHでリモートURLが異なる。どちらを使っているか確認し、SSHを使う場合は適切な秘密鍵が登録されているか確認する。

### なぜGitHubを使うのか
Gitは分散型なのでローカルだけでも使えますが、GitHubのようなプラットフォームがなければ、チーム開発での「共有リポジトリ」のハブが存在しません。さらにGitHubは、プルリクエスト、Issue、GitHub Actionsなどのエコシステムを備えており、Gitを単なる履歴管理から「開発コラボレーションツール」に進化させています。

### 設計思想：設定階層

Gitの設定には `system` (システム全体)、`global` (ユーザーごと)、`local` (リポジトリごと) の階層があり、後者が前者を上書きする仕組みで柔軟性を確保しています。

インストール後は、必ず以下の設定を済ませておきましょう。
```bash
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
```
ここで指定した名前とメールアドレスが、コミット履歴に残ります。身分証明書のようなものなので、正確に設定してください。
