# Gitとは

プログラミングを学んでいると、必ず「**Git (ギット)**」という言葉に出会います。
Git は、一言で言えば **「ソースコードのタイムマシン」** です。コードの変更履歴をすべて記録し、いつでも過去の状態に戻したり、誰がいつどこを変えたかを確認したりできるようにするためのツールです。

私が初めて Git を使ったとき、複雑なコマンドに圧倒されましたが、一度「過去の自分に助けられた」経験をしてからは、Git なしの開発なんて考えられなくなりました。

## 1. バージョン管理の進化：集中型から分散型へ

Git が登場する前、世界では **Subversion (SVN)** のような「集中型」のバージョン管理システムが主流でした。

- **集中型**: サーバーに「正解」のデータが一つあり、全員がそこにアクセスして編集します。サーバーが止まると何もできず、過去の履歴を見るのも遅いという欠点がありました。
- **分散型 (Git)**: 各開発者の手元の PC に、**リポジトリ（履歴の全データ）の完全なコピー** を持ちます。

この「全員が全履歴を持っている」という設計のおかげで、Git はオフラインでも作業でき、変更の記録（コミット）が驚くほど高速です。また、サーバーが壊れても誰かの PC から復旧できるという、極めて高い堅牢性を持っています。

## 2. なぜ Git が「必須」なのか？

現代の開発現場で Git が使われないことはまずありません。その理由は 3 つあります。

1. **履歴管理（タイムマシン）**: 「昨日まで動いていたのに、今日直したら動かなくなった！」というとき、1 秒で昨日の状態に戻せます。
2. **ブランチ機能（パラレルワールド）**: 本番用のコードとは別に、「新機能開発用」「バグ修正用」といった並行世界を作れます。実験的なコードを書いて失敗しても、本番には影響を与えません。
3. **チーム開発（統合）**: 複数人で同じファイルを同時に編集しても、Git が賢く「マージ（合体）」してくれます。

## 3. 初心者がハマるポイントと解決策

- **初期化忘れ**: `git init` をせずに `git add` を叩いても「Not a git repository」と怒られます。まずはそこが Git の管理下にあるか確認しましょう。
- **`.gitignore` の設定**: 自分のパスワードが入った設定ファイルや、巨大なビルド成果物（`node_modules` など）をうっかりコミットしてしまう事故が多発します。最初に「無視するリスト」を作ることが重要です。

## 低レイヤーの視点：Git は「コンテンツ指向」のデータベース

Git の裏側は、実は非常にシンプルなデータ構造でできています。Git はファイルを名前で管理するのではなく、**「中身（コンテンツ）」** で管理します。

1. ファイルの中身を **ハッシュ関数 (SHA-1)** にかけ、一意の ID（ハッシュ値）を生成します。
2. そのハッシュ値を名前とした「Blob オブジェクト」として保存します。

もし、ファイル名を `A.js` から `B.js` に変えても、中身が同じなら Git は同じハッシュ値を使い回します。
「同じ内容なら一つにまとめる」。この **不変性 (Immutability)** を前提とした設計こそが、Git が数万件のファイルを高速かつ効率的に扱える秘密なのです。

## 私の視点：ツールに頼ってもいい

「Git はコマンド（黒い画面）で使いこなしてこそ一人前だ」という意見もありますが、私はそうは思いません。
VS Code の Git 機能や、**Sourcetree**、**Fork** といった GUI ツールを使えば、変更の履歴が美しい「グラフ」として視覚的に理解できます。

大切なのはコマンドを丸暗記することではなく、**「今、自分のコードがどのブランチにあり、どんな状態で記録されているのか」** という構造を正しく把握することです。道具は何でも構いません。Git という最強のタイムマシンを味方につけて、恐れずにコードを書き換えていきましょう。

### まとめ

- Git は分散型のバージョン管理システム。
- 「履歴管理」「ブランチ」「チーム開発」の 3 本柱で開発を支える。
- 全開発者が履歴のコピーを持つため、高速で壊れにくい。
- 裏側では「ハッシュ値」を使った高度なコンテンツ管理が行われている。
- 最初は GUI ツールを使って、構造を視覚的に理解するのもお勧め。

### 展望

Git の概念を理解したら、次は実際に自分の PC にインストールし、世界中の開発者のハブとなっている **GitHub** に登録してみましょう。いよいよ、あなたのコードを世界へ発信する準備が整います。
