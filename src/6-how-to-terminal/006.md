# 演習

ターミナルとGitの操作に慣れるための演習です。単にコマンドを叩くのではなく、「今何が起きているのか」を観察してください。

### 課題

1. **ディレクトリ操作**: 任意の場所に新しいディレクトリを作成し、その中にテキストファイルを作成して移動・コピー・削除を行ってください。`ls -la` でパーミッションの変化も確認しましょう。
2. **Git操作**: 新しいGitリポジトリを作成し、ファイルを追加・コミットし、`git log` で履歴を確認してください。さらに `git checkout` や `git branch` でブランチの移動も試してください。
3. **コマンド練習**: `history` コマンドで最近使ったコマンドを確認し、`alias` を設定してみてください。例えば `alias gs='git status'` など。

### ヒント：低レイヤーとデバッグ

- **`rm -rf`**: 危険な操作ですが、OSがどうファイルを消去するかを理解するきっかけになります。
- **Gitの理解**: `git status` がどのように「ステージングエリア」と「作業ディレクトリ」を比較しているのか（`git hash-object` などを使えば裏側を覗けます）に思いを馳せてください。
- **エラー処理**: Gitの操作で詰まったら `git status` だけでなく、`.git/` ディレクトリの中身（設定ファイルやオブジェクトデータベース）を覗いてみるのも一つの手です。

### 詰まりポイントと解決策

- **演習後のクリーンアップ**: 練習で作成した不要なリポジトリやディレクトリを放置しがち。ディスク容量の節約と、迷子防止のために不要になったものは削除しよう。
- **削除コマンドの破壊力**: 演習中に `rm -rf` で本当に重要なファイルを消してしまう悲劇がよくあります。「自分のPCのどこを操作しているか」という感覚を養う訓練だと思って、常に慎重に。

これらの操作を繰り返すことで、ターミナルとGitはあなたの開発における最も信頼できるパートナーになります。
