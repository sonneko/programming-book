# JSXと関数コンポーネント

Reactは、現在のフロントエンド開発において最も普及しているライブラリの一つです。その本質は「UIを状態の関数として表現する」ことにあります。

### JSXの正体
JSXは、JavaScriptの中にHTMLのような記法を書くための構文拡張です。一見するとテンプレートエンジンのようですが、JSXは単なる「見た目」の記述ではありません。JSXはビルド時に `React.createElement` という関数呼び出しに変換され、最終的にはJavaScriptのオブジェクト（仮想DOM）を生成します。

### 関数コンポーネントと不変性
関数コンポーネントは、props（入力データ）を受け取り、UI（出力データ）を返す純粋な関数です。Reactの理念として「propsを直接変更してはならない」という不変性の原則があります。データが変更されたら、古いオブジェクトを修正するのではなく、新しい状態を持つ新しいオブジェクトを生成します。

この計算モデルにより、Reactは「どのデータが変更されたか」を効率的に検知し、必要最小限のDOM操作を行うことができます。
