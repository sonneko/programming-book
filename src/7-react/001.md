# JSXと関数コンポーネント

React を学び始めると、最初に驚くのが JavaScript の中に HTML を直接書いているような不思議なコードです。これを **JSX (JavaScript XML)** と呼びます。

「JavaScript と HTML は分けるべきじゃないの？」と思うかもしれません。しかし、React の世界では、見た目（HTML）とロジック（JavaScript）は切り離せない「一つの部品」として扱います。ここでは、JSX の正体と、React の最小単位である「関数コンポーネント」の仕組みを解き明かしましょう。

## 1. JSX の正体：ただの JavaScript オブジェクト

JSX は、ブラウザがそのまま理解できる言葉ではありません。ビルド時に、Babel などのツールによって「普通の JavaScript」に変換（トランスパイル）されます。

```jsx
// あなたが書く JSX
const element = <h1 className="title">Hello</h1>;

// 変換後の JavaScript (React 17以前のイメージ)
const element = React.createElement('h1', { className: 'title' }, 'Hello');
```

この `createElement` が返すのは、実はただの **「JavaScript オブジェクト」** です。
「タグ名は h1 で、属性はこれで……」という情報が詰まった、メモリ上の軽いデータに過ぎません。React はこのオブジェクトを大量に作り、それらを組み合わせて **仮想 DOM ツリー** を構築します。

## 2. 関数コンポーネント：UI を返す「純粋な関数」

React におけるコンポーネントは、もっともシンプルな形では「JSX を返すただの JavaScript 関数」です。

```tsx
function WelcomeMessage({ name }: { name: string }) {
  return (
    <div className="welcome">
      <h1>ようこそ、{name}さん！</h1>
      <p>React の世界へ踏み出しましょう。</p>
    </div>
  );
}
```

コンポーネントを定義する際の鉄則は、**「大文字から始めること」** です。
小文字（`div` など）はブラウザ標準のタグ、大文字（`WelcomeMessage`）はあなたが作ったコンポーネント、として React は区別します。

## 3. 不変性（Immutability）という規律

関数コンポーネントを扱う上で最も大切な思想が **「不変性」** です。
コンポーネントに渡されるデータ（props）は、読み取り専用です。関数の中で勝手に書き換えてはいけません。

もしデータが変わったら、元のデータを書き換えるのではなく、**「新しいデータを使って、もう一度関数を実行する」**。これが React のレンダリングの仕組みです。
この「古いものは捨てて、新しいものを作る」という割り切った設計が、複雑な Web アプリの整合性を保つ鍵になっています。

## 低レイヤーの視点：メモリ管理とガベージコレクション

「レンダリングのたびに新しいオブジェクト（仮想 DOM）を大量に作って、メモリは大丈夫なの？」と心配になるかもしれません。

現代の JavaScript エンジン（V8 など）の **ガベージコレクション (GC)** は、短命なオブジェクト（作られてすぐ使われなくなるもの）の回収が非常に得意です。

仮想 DOM は非常に軽量なオブジェクトであり、計算が終わればすぐに GC によって回収されます。本物の DOM（ブラウザの重いリソース）をいじるコストに比べれば、メモリ上でオブジェクトを数千個作り直すコストの方が圧倒的に低いのです。
「物理的な重さ（DOM）」を「論理的な軽さ（オブジェクト）」で包み込む。これが React の魔法の正体です。

## 私の視点：JSX は「最強のテンプレート」

PHP や Ruby on Rails などのテンプレートエンジンを触ったことがある人なら、JSX の自由度に驚くはずです。
JSX は文字列ではなく JavaScript そのものなので、`map` でループを回したり、三項演算子で条件分岐したりといったことが、JavaScript の知識だけで完結します。

新しい「独自ルール」を覚える必要がなく、JavaScript の腕が上がれば上がるほど、より洗練された UI が書けるようになる。この「言語との一体感」こそが、私が React を手放せない理由です。

### まとめ

- JSX は HTML ではなく、JavaScript のオブジェクト（`createElement`）への書き換え。
- コンポーネントは「props を受け取り、JSX を返す関数」。
- コンポーネント名は必ず大文字で始める。
- 「データが変われば、新しいデータで関数を再実行する」のが React 流。
- 仮想 DOM は軽量なので、頻繁に作り直しても現代のブラウザなら問題ない。

### 展望

JSX の書き方がわかったら、次はコンポーネント同士でデータをやり取りする「props」と、要素を包み込む「children」の強力な使いこなし術を学びましょう。
