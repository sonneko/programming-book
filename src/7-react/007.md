# SPA（Single Page Application）の構造と価値

SPAは、ページ全体を再読み込みすることなく、必要な箇所だけを動的に書き換えることで、ネイティブアプリのような操作感を実現するWebアプリケーションの形態です。

### ブラウザのレンダリングパイプラインとSPA
従来のサーバーサイドレンダリング（MPA）では、ページ遷移のたびにHTML全体を取得し、ブラウザのパースとレンダリングがゼロからやり直されていました。SPAでは、一度読み込んだJS環境の中でデータのみを非同期に取得し、Reactが仮想DOMを用いて必要なDOM要素のみを効率的に更新します。これにより、ページ遷移のコスト（レイアウト計算、スタイル計算の再実行）が劇的に削減されます。

### クライアントサイドルーティングの技術
SPAにおいては、ブラウザのURL更新をフックにJS側で表示するコンポーネントを切り替える「クライアントサイドルーティング」が行われます。これはブラウザのHistory APIを巧みに利用しており、ページ遷移なしでURLを操作し、かつ戻るボタンなどのネイティブ機能との整合性を保つという高度なUI設計がなされています。Reactは、このSPAの複雑な状態（ルーティング、画面遷移）をコンポーネント階層の一部として宣言的に管理するための理想的なランタイムを提供しています。
