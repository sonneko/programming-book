# 特別な関数たち

Reactには、レンダリングの最適化や特定の処理を行うための特別なAPIが存在します。

- **`memo`**: コンポーネントの再レンダリングをpropsが変更されたときのみに制限する。
- **`useMemo`**: 計算結果をメモ化し、依存関係が変わらない限り再計算をスキップする。
- **`useCallback`**: 関数をメモ化し、不要な再作成を抑制する。

### 技術的深みとパフォーマンスの現実

- **参照の等価性 (Reference Equality)**: Reactはpropsの比較にシャロー比較（`===`）を用います。`useCallback`などで関数をメモ化しないと、親のレンダリングのたびに新しい関数インスタンスが生成され、子コンポーネントの最適化が無効化されます。
- **最適化のコスト**: メモ化にはメモリ使用量が増加するコストと、依存配列の比較計算コストが存在します。
- **アンチパターン**: あらゆる箇所にメモ化を適用するのは避けるべきです。パフォーマンスボトルネックを React DevTools の Profiler で特定し、再レンダリングによる影響が大きい計算や、重いコンポーネントに対して戦略的に適用するのがベストプラクティスです。

不必要なメモ化はコードの複雑性を増し、デバッグを困難にします。「まずはシンプルに書く、問題があれば最適化する」というエンジニアリング原則が特に重要です。
