# ReactRouterとは

SPA（シングルページアプリケーション）では、実際のページ遷移（リロード）は行いません。しかし、ユーザーにとっては「/about にアクセスしたら会社概要が見える」「/blog/1 にアクセスしたら記事が読める」という、従来の URL と画面が 1 対 1 で対応した体験が必要です。

この **「URL の状態」と「表示するコンポーネント」を同期させる** ためのデファクトスタンダード（事実上の標準）ツールが、**React Router** です。

## 1. 宣言的なルーティング

React Router の最大の特徴は、ルーティングの設定もまた「React コンポーネント」として記述できる点にあります。

```tsx
import { BrowserRouter, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <nav>
        <Link to="/">ホーム</Link>
        <Link to="/about">アバウト</Link>
      </nav>

      <Routes>
        {/* URL に応じて表示するコンポーネントを切り替える */}
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}
```

「もし URL が `/about` なら `<About />` を出してね」というルールを、JSX の中に宣言的に書くことができます。

## 2. リンクと遷移の制御

SPA では `<a>` タグを使ってはいけません。なぜなら、`<a>` タグはブラウザに「リロードして新しいページを取りに行け」と命じてしまうからです。

代わりに React Router が提供する **`<Link>`** コンポーネントを使います。
`<Link>` はクリックされたとき、リロードを阻止し、こっそりと URL だけを書き換えて、React に「画面を切り替えて！」と伝えます。

## 低レイヤーの視点：ブラウザ履歴とメモリルーティング

React Router には、いくつかの「ルーター」が用意されています。

- **BrowserRouter**: 最も一般的です。ブラウザの **History API** を使い、`/about` のような本物の URL を作ります。ただし、サーバー側で「どんな URL が来ても `index.html` を返す」という設定が必要です。
- **HashRouter**: URL の後ろに `/#/about` のようにハッシュを付けます。サーバー設定がいらないため、古いサーバーや GitHub Pages などで便利です。
- **MemoryRouter**: URL をブラウザのバーに表示せず、**メモリの中だけで** ルーティングを管理します。

なぜメモリ上のルーティングが必要なのでしょうか？ それは、画面（DOM）が存在しない環境、たとえば **Node.js でのテスト実行** や、スマホアプリ（React Native）などでルーティングをシミュレートするためです。
「URL という文字列」を「コンポーネントの状態」に変換する。React Router はこの写像（マッピング）を、環境を問わずに行えるよう抽象化してくれているのです。

## 私の視点：ルーティングは「アプリの地図」

私は、新しいプロジェクトに参加したとき、まず最初に React Router の設定ファイル（`App.tsx` など）を読みます。
そこには、そのアプリにどんな画面があり、どんなデータが必要なのかという **「地図」** がすべて書かれているからです。

複雑なアプリでは、`/dashboard` の中にさらに `/dashboard/settings` がある、といった入れ子構造が登場します。React Router の `Outlet` 機能を使いこなせると、共通のナビゲーションバーを残したまま中身だけを切り替える、といった高度な画面レイアウトが驚くほど簡単に実装できるようになりますよ。

### まとめ

- React Router は URL とコンポーネントを同期させるライブラリ。
- `<BrowserRouter>` で全体を包み、`<Route>` で条件を記述する。
- 遷移には `<a>` ではなく `<Link>` を使う（リロードを防ぐため）。
- 裏側では History API を操作して、ページ遷移を「擬態」している。
- ルーティング設計は、アプリの機能美を決定づける「地図」の作成である。

### 展望

React Router で SPA の基本を学んだら、次はさらに一歩進んで、サーバー側での処理や最適化を極限まで突き詰めた最強のフレームワーク、**Next.js** の世界へ。
