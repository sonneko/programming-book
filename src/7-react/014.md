# 演習

Reactの基礎を実践的に確認するための演習です。

### 課題

1. **カウンター**: `useState` を使って、ボタンを押すと数字が増減するカウンターを作成してください。
2. **リスト表示**: 配列データを受け取り、それをリスト (`<ul>`, `<li>`) として描画するコンポーネントを作成してください。
3. **副作用**: `useEffect` を使って、コンポーネントがマウントされた際にAPIからデータを取得する処理を書いてください。

### 実践的なアドバイスとベストプラクティス

- **ステート管理**: 複数のコンポーネントで共有すべきステートは、親コンポーネントへ持ち上げる（Lifting State Up）ことで、データの不整合を防ぎます。
- **テスティング**: コンポーネントのテストには、VitestやReact Testing Library（RTL）を採用しましょう。DOMのレンダリング結果を検証することで、実装の詳細に依存しない堅牢なテストが書けます。
- **ツール活用**: React DevTools をインストールして、コンポーネントの state や props の変化を可視化・追跡しましょう。
- **設計の原則**: コンポーネントは常に純粋関数のようにシンプルに保ち、複雑なロジックはカスタムフックへ抽出し、責務の分離を徹底しましょう。

この演習を通じて、Reactの宣言的UIの恩恵と、ステート管理の重要性を体感してください。
