# 公式ドキュメントの読み方

Reactの公式ドキュメントは、非常に質が高く、最新のベストプラクティスが網羅されています。

### 読み方のコツ

- **「Learn React」**: 基礎を体系的に学ぶためのチュートリアル。まずはここを熟読する。
- **「Reference」**: 特定のフックやAPIの仕様を確認する際に利用。
- **「Examples」**: 実践的なコード例を探す際に参照。

### 技術的探求の深化

- **RFC (Request for Comments)**: Reactの次期機能はRFCで議論されます。GitHubの `reactjs/rfcs` を追うことで、Reactの未来の設計思想を理解できます。
- **ソースコードを読む**: `facebook/react` のソースコードは、特にHooksの内部実装やReconciler（ファイバーアーキテクチャ）の理解に直結します。
- **コミュニティ**: GitHub Issuesやコミュニティの議論（Discussions）は、ドキュメントでは語られない具体的なユースケースやバグの回避策を知る宝庫です。

ドキュメントだけでなく、設計思想の裏側にある議論を追うことで、一歩先のReactエンジニアへと成長できます。
