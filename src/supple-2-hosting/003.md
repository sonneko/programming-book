# GoogleCloud

Google が自社の膨大なサービス（検索、YouTube、Gmail など）を支えるために構築した、世界最高峰のインフラ。そのインフラを私たち一般の開発者が利用できるようにしたのが **Google Cloud（旧 GCP: Google Cloud Platform）** です。

正直に言いましょう。Google Cloud は、これまで紹介してきたレンタルサーバーや VPS とは「次元が違う」サービスです。機能が多すぎて最初は圧倒されるかもしれませんが、モダンなエンジニアを目指すなら、遅かれ早かれここ（または AWS）を通ることになります。

## クラウドの本質：オンデマンドの抽象化リソース
Google Cloud のようなクラウドサービスの本質は、あらゆるリソース（CPU、メモリ、ディスク、ネットワーク、データベース、さらには AI モデルまで）を、必要なときに、必要な分だけ、API 経由で即座に呼び出せる「抽象化されたインフラ」であることです。

物理的なサーバーを借りるという感覚よりも、巨大な「コンピュータのプール」から一部の能力を切り出して使う、という感覚に近いです。

## 初心者がまず覚えるべき 3 つのサービス
Google Cloud には数百のサービスがありますが、まずはこの 3 つを知っておけば十分です。

1. **Compute Engine (GCE)**: 仮想マシン（VM）。カゴヤの VPS と同じように、Linux OS を自由に扱えます。
2. **Cloud Run**: 近年最も注目されているサービス。あなたの書いたコード（コンテナ）をアップロードするだけで、アクセスがあるときだけ自動でサーバーが立ち上がり、アクセスがなければゼロ（無料）になります。これを **サーバーレス** と呼びます。
3. **Cloud Storage (GCS)**: 無制限のデータを保存できるストレージ。Google Drive API の章で学んだことの、より大規模・システム向けの版です。

## Google Cloud が選ばれる理由
1. **圧倒的なネットワーク速度**: Google は世界中に自前の海底ケーブルを張り巡らせています。そのため、Google Cloud 内のデータ転送は非常に高速で安定しています。
2. **データ分析と AI への強み**: BigQuery という超高速なデータ分析基盤や、Gemini API を含む Vertex AI など、Google の得意分野がそのまま利用できます。
3. **モダンな開発フローとの親和性**: GitHub と連携して、コードをプッシュするだけで自動的に Web サービスが更新される仕組みが簡単に作れます。

## Google Cloud を使う上での「覚悟」
Google Cloud は非常に強力ですが、初心者にはいくつかの壁があります。

- **料金体系の複雑さ**: 「月額 ◯◯ 円」ではなく、すべてが「使った分だけ」の従量課金です。設定を間違えて巨大なサーバーを立ち上げっぱなしにすると、翌月に驚くような請求が来ることがあります（必ず **予算アラート** を設定しましょう）。
- **用語の難しさ**: IAM、VPC、Subnet、Load Balancer... ネットワークやセキュリティの専門用語が次々と出てきます。
- **UI の変化が激しい**: 管理画面（コンソール）の見た目や機能名が頻繁に変わります。「ネットの記事と画面が違う！」というのは日常茶飯事です。

## 無料枠を使い倒そう
幸いなことに、Google Cloud には強力な **無料枠（Always Free）** があります。
特定のリージョン（場所）にある小さな仮想マシンや、Cloud Run の一定回数までの利用は、一生無料（条件あり）で使い続けることができます。また、新規登録時には 300 ドル分（約 4〜5 万円分）の無料クレジットが付与されることが多いため、これを使って各種サービスを実験してみるのが最も賢い学習方法です。

## まとめ
Google Cloud は、単なる「置き場所」ではなく、あなたのプログラムを世界規模で動かすための「巨大なツールボックス」です。

まずは、[Google Cloud Console](https://console.cloud.google.com/) にアクセスし、プロジェクトを作成することから始めてください。10 章で学ぶ「コンパイル」や「OS の仕組み」を知っていれば、クラウドが提供する抽象化の凄さが、より深く理解できるはずです。
