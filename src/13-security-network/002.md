# ルーティングとネットワークの構築

データが地球の裏側にあるサーバーまで、わずか数百ミリ秒で届くのはなぜでしょうか。そこには、網の目のように張り巡らされたネットワークと、それを賢く制御する「ルーティング（経路制御）」という仕組みがあります。本節では、ルーティングテーブルや動的ルーティングなど、ネットワークがいかにして作られ、維持されているのかを多角的に解説します。

### ネットワークは「自律システム」の集合体である

そもそも「インターネット」という単一のネットワークが存在するわけではありません。インターネットとは、ISP（インターネットサービスプロバイダ）や巨大なテック企業、大学、政府機関などの「AS（自律システム / Autonomous System）」と呼ばれる独立したネットワークが、互いにつながり合った「ネットワークのネットワーク」です。

この「事務的・構造的側面」を理解することが、ネットワークエンジニアリングの第一歩です。各ASには「AS番号」という固有の識別番号が割り当てられ、隣接するAS同士でBGP（Border Gateway Protocol）というプロトコルを使って、「私のネットワークにはこのIPアドレス群（プレフィックス）がありますよ」「あのネットワークへ行くなら、私を経由するのが最短ですよ」という経路情報を交換し合っています。この情報のやり取りが、世界規模の地図（グローバルルーティングテーブル）を作り上げているのです。

### ルーティングテーブル：目的地への論理的な道しるべ

個々のルーター（ネットワークの交差点にある機器）は、「ルーティングテーブル」というリストを持っています。ここには、「この宛先IPアドレスのパケットは、次はこの出力ポートから出せ」という「論理的側面」の対応表が記されています。

ルーティングテーブルの構成要素は主に以下の通りです。
- **宛先ネットワーク**: 目的地のネットワークアドレスとサブネットマスク（例: 192.168.1.0/24）。
- **ネクストホップ**: 次にパケットを渡すべき隣のルーターのIPアドレス。
- **インターフェース**: 物理的にどのLANポートから出すか（例: eth0, Gi0/1）。
- **メトリック**: その経路の「コスト」。距離、帯域幅、遅延などに基づいて算出されます。

パケットがルーターに到着すると、ルーターは宛先IPアドレスをチェックし、ルーティングテーブルの中で最もビットの一致数が多いエントリを探します。これを「ロンゲストプレフィックスマッチ（最長一致の原則）」と呼びます。もし一致するものがなければ、「デフォルトゲートウェイ（0.0.0.0/0）」、つまり「何でも屋」のルーターへと送られます。

### 動的ルーティングの魔法：自己修復するネットワーク

手動でルーティングテーブルを書く（静的ルーティング）のは、小さなオフィス内なら可能ですが、インターネット全体では不可能です。そこで登場するのが「動的ルーティング」です。

ルーター同士が「ルーティングプロトコル（内部ネットワークならOSPFやRIP、外部接続ならBGP）」を用いて情報をリアルタイムで交換します。ネットワークの障害（ケーブル断線など）や混雑状況に応じて、数秒から数十秒の速さで経路を書き換えます。これが「動的ルーティング」の醍醐味です。もし、太平洋を横断する海底ケーブルが地震で切断されても、動的ルーティングのおかげで、自動的に衛星通信や別の国（例えば欧州経由）を回るルートが選択され、通信が維持されるのです。

### 低レイヤーの現実：ASICによる高速フォワーディング

「低レイヤー側面」から見ると、ルーターは極限のパフォーマンスを求められるデバイスです。コアネットワークのルーターでは、1秒間にテラビット単位、パケット数にして数億個を捌かなければなりません。

汎用的なCPUでソフトウェア的にルーティングテーブルを検索していては、到底この速度には追いつきません。そのため、最新のルーターでは「ASIC（特定用途向け集積回路）」や「FPGA」といった専用のハードウェア回路を用いて、ハードウェアレベルでテーブル検索と転送（フォワーディング）を行っています。「TCAM（Ternary Content Addressable Memory）」という特殊なメモリを使うことで、1クロックでテーブルの全文検索を実現しているのです。

また、パケットの「TTL（Time To Live）」という仕組みも重要です。これは「論理的な寿命」で、ルーターを1つ通過するたびに1ずつ減らされます。もしルーティング設定のミスでパケットがループ（循環）してしまっても、TTLが0になればパケットは自動的に破棄され、ネットワークがパンク（輻輳）するのを防いでいます。

### セキュリティとルーティングの脆弱性：BGPハイジャック

ルーティングには深刻な「セキュリティー的側面」の課題があります。特にBGPは、1980年代の設計思想を引きずっており、「相手の言うことを善意で信じる」という前提があります。これを悪用したのが「BGPハイジャック」です。

攻撃者が「GoogleのIPアドレスは私のASにあります！」という嘘の情報をBGPで流すと、世界中のトラフィックが攻撃者のルーターへと吸い込まれてしまいます。これにより、通信の盗聴、改ざん、あるいは大規模な通信障害が発生します。過去には実際に、ある国のISPの設定ミスや意図的な攻撃により、YouTubeやGoogleへのアクセスが数時間にわたって遮断された事例があります。

現在では、ルート情報にデジタル署名を付与し、その情報が正当な所有者から出たものかを検証する「RPKI（Resource Public Key Infrastructure）」などの「暗号学的側面」からの対策が進められていますが、完全な普及には至っていません。

### 事務的な側面：ピアリングと接続契約

ネットワークの構築には、物理的なインフラだけでなく、組織間の「事務的側面」の合意も重要です。「ピアリング（Peering）」と呼ばれる、ISP同士が相互にトラフィックを交換する契約や、IX（インターネットエクスチェンジ）という接続拠点の運用が、インターネットの接続コストを下げ、高速な通信を支えています。

### まとめ：繋がることの奇跡と設計の執念

私たちがブラウザにURLを打ち込むとき、パケットは数十台のルーターを経由し、その都度ルーティングテーブルが参照され、ミリ秒単位で「次の道」が決定されています。
- **事務的**: AS間の接続契約とグローバルな管理。
- **論理的**: ルーティングテーブルと最長一致のアルゴリズム。
- **低レイヤー**: 専用チップによるハードウェア転送の極致。
- **セキュリティー**: BGPハイジャックの恐怖とRPKIによる信頼の再構築。

これらを知ることで、単なる「ネットが繋がっている」という状態が、いかに高度な技術と、24時間365日の運用の結晶であるかが見えてくるはずです。

*コラム：tracerouteコマンドでネットワークを視覚化する*
ターミナルで `traceroute google.com`（Windowsなら `tracert`）と打ってみてください。あなたのパケットが自宅のルーターを抜け、プロバイダのネットワークを通り、海の向こうのデータセンターへ辿り着くまでの「ホップ（経由地点）」がリアルタイムで表示されます。ネットワークが「目に見えない一本の線」ではなく、世界中の「点の連鎖」であることを実感できるはずです。
