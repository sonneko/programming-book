# DOM操作：ブラウザ上の要素を操る

DOM（Document Object Model）は、HTMLやXMLドキュメントをプログラムが操作可能な「ツリー構造」として表現したものです。JavaScriptからDOMを操作することで、Webページを動的に更新し、ユーザーインタラクションを可能にします。

## DOMツリーの取得と探索

DOMツリー内の要素を取得するには、以下のメソッドが基本です。

```javascript
// IDによる取得
const element = document.getElementById('my-id');

// CSSセレクタによる取得
const listItems = document.querySelectorAll('ul > li.item');
```

取得した要素を起点に、DOMツリーを上下左右に探索することも可能です。
- `parentNode` / `parentElement`: 親要素へ
- `children`: 子要素の配列的オブジェクトへ
- `nextElementSibling` / `previousElementSibling`: 兄弟要素へ

## DOM操作の基本

Webアプリケーションにおいて、DOMを直接的に書き換えることは非常に頻繁に行われます。

### 要素の生成と追加
```javascript
const newDiv = document.createElement('div');
newDiv.textContent = '新しい要素';
document.body.appendChild(newDiv); // ページ末尾に追加
```

### 属性とスタイルの変更
```javascript
element.style.backgroundColor = 'blue';
element.classList.add('highlight');
element.setAttribute('data-id', '123');
```

## イベント処理：インタラクティブなページへ

Webページが動的に見えるのは、ユーザーの行動（クリック、キー入力など）をJavaScriptが検知しているからです。これには `addEventListener` を使用します。

```javascript
button.addEventListener('click', (event) => {
  console.log('ボタンがクリックされました！');
});
```

ここでの重要な概念が「イベントバブリング（伝播）」です。イベントはターゲット要素から親要素へ順に伝わっていきます。これを理解することで、親要素でまとめてイベントを管理する「イベント委譲（Event Delegation）」という手法が利用でき、メモリ消費を抑えることができます。

## パフォーマンスとDOM操作の限界

DOM操作はブラウザのレンダリングエンジンに対して「リフロー（再レイアウト）」や「リペイント（再描画）」という高負荷な処理を引き起こします。

- **リフロー**: 要素の寸法や位置が変わった際に発生するレイアウト計算。
- **リペイント**: 色や背景など、見た目が変わった際に発生する再描画。

頻繁なDOM直接操作は、ページのガタつきやパフォーマンス低下の原因になります。これが、現代のフロントエンド開発においてReactやVue.jsといった「仮想DOM」を採用するライブラリが標準となった理由です。仮想DOMは、メモリ上でDOMツリーの変更を計算し、必要な最小限の更新だけを実DOMに反映させることで、高パフォーマンスを実現しています。

DOM操作の仕組みを理解した上で、複雑な画面構築にはフレームワークの力を借りる、というのが現代のフロントエンド開発におけるベストプラクティスです。
