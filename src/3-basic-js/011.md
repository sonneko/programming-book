# シンボル (Symbol)：一意性の保証とメタプログラミング

シンボル（Symbol）は、ES6で導入されたプリミティブデータ型です。他のプリミティブ型（数値、文字列、ブール値など）とは異なり、シンボルは「一意」であり「変更不可能」な値を作成するために使用されます。

## なぜシンボルが必要なのか

オブジェクトのプロパティキーとして文字列を使用する場合、大規模なコードベースやライブラリとの統合において、予期せぬ衝突が発生するリスクがあります。異なるライブラリが同じキー名でプロパティを設定しようとすると、後から読み込まれた側が前の値を上書きしてしまいます。

シンボルはこれを解決します。`Symbol()` を呼び出すたびに生成される値は、たとえ説明（description）が同じであっても、決して他のシンボルと等しくなることはありません。

```javascript
const key1 = Symbol('id');
const key2 = Symbol('id');

console.log(key1 === key2); // false
```

## シンボルの種類と活用

### 1. ローカルシンボルとグローバルシンボル
通常 `Symbol()` で作成するシンボルはローカルな一意の値です。もし、アプリケーション全体で同じシンボルを共有したい場合は、グローバルシンボルレジストリを利用します。

```javascript
const sharedKey = Symbol.for('shared');
const sameSharedKey = Symbol.for('shared');

console.log(sharedKey === sameSharedKey); // true
```

### 2. Well-known Symbols (言語組み込みのシンボル)
JavaScriptエンジンは、特定の組み込み機能の挙動を変更するために、いくつかの特別なシンボル（Well-known Symbols）を使用しています。

- **`Symbol.iterator`**: オブジェクトが `for...of` ループでどのようにイテレートされるかを定義します。
- **`Symbol.toStringTag`**: `Object.prototype.toString.call()` が返す文字列をカスタマイズします。

これらをオブジェクトに実装することで、自作のオブジェクトに対してJavaScriptの言語機能を適用させることができます（メタプログラミング）。

```javascript
const myCollection = {
  data: [1, 2, 3],
  [Symbol.iterator]: function*() {
    yield* this.data;
  }
};

for (const value of myCollection) {
  console.log(value); // 1, 2, 3
}
```

## オブジェクトでの利用と注意点

シンボルをキーにしたプロパティは、通常のプロパティとは少し異なる振る舞いをします。

- **`JSON.stringify()`**: シンボルキーのプロパティはシリアライズ（JSON化）時に無視されます。
- **オブジェクト列挙**: `for...in` ループや `Object.keys()` では列挙されません。これはシンボルプロパティが「隠蔽（プライベートなメタデータ）」として適している理由です。
- **プロパティの取得**: シンボルキーを確認したい場合は `Object.getOwnPropertySymbols(obj)` を使用する必要があります。

シンボルは、日常的な開発で頻繁に使うわけではありませんが、複雑な状態管理、ライブラリ開発、あるいはJavaScriptの高度な言語機能をカスタマイズしたい場合に必須となる強力なツールです。
