# 型の明示について

JavaScriptは「動的型付け言語」であり、変数の型は実行時に決定されます。これは柔軟性を提供しますが、大規模開発においては型に関するバグの温床になりがちです。

### 動的型付けの現実

JavaScriptでは、同じ変数に数値を入れた後に文字列を入れることも可能です。

```javascript
let data = 100;
data = 'hello'; // 問題なく実行される
```

この柔軟性はプロトタイプ作成には役立ちますが、複雑なシステムでは「このデータは何型なのか？」という推論が困難になります。

### 型の安全性を高めるアプローチ

JavaScriptだけで型の安全性を担保するには、JSDocによる注釈付けが一般的です。

```javascript
/**
 * @param {number} a
 * @param {number} b
 * @returns {number}
 */
function add(a, b) {
  return a + b;
}
```

しかし、これでも実行時の強制力はありません。より堅牢な開発を目指すのであれば、TypeScriptを導入し、静的型付けによってコンパイル時に型の整合性を検証することを強く推奨します。
