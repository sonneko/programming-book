# 基本の制御構造：プログラムの流れを操る

プログラムの流れを制御することは、開発の基礎です。JavaScriptでは、条件分岐とループ処理を用いて処理の実行順序を柔軟に制御し、動的なロジックを実装します。

### 条件分岐の最適化

最も基本的な条件分岐は `if` 文です。特定の条件が真（true）である場合にのみ、ブロック内のコードが実行されます。

```javascript
const age = 20;

if (age >= 18) {
  console.log('成人です');
} else {
  console.log('未成年です');
}
```

`if-else` の入れ子が深くなるとコードの可読性が低下します。これを避けるためには「早期リターン（Early Return）」が有効です。条件を満たさない場合に即座に関数を終了させることで、ネストを浅く保つことができます。

また、変数の代入に条件分岐を用いる場合は、三項演算子（`condition ? exprIfTrue : exprIfFalse`）を使うと簡潔です。

さらに、多くの値に対する分岐が必要な場合は `switch` 文が使えます。ただし、JavaScriptの `switch` 文は `break` 文を書き忘れると後続の `case` も実行される（フォールスルー）仕様があるため、注意が必要です。現代では、オブジェクトリテラルや `Map` を使った条件テーブルによる制御の方が、堅牢で好まれる傾向があります。

### ループ処理の多様性

同じ処理を繰り返す必要がある場合、ループ処理を使います。

- **`for`文**: 回数が明確な場合に最適です。初期化、条件チェック、イテレーションの更新が一行にまとめられています。
- **`while`文**: 条件が満たされている間、繰り返し処理を行います。回数が不明な場合や、特定の条件まで処理を続ける場合に便利ですが、終了条件を誤ると無限ループに陥るリスクがあります。
- **`for...of` 文**: 配列や文字列など、イテラブル（反復可能）なオブジェクトの値を順番に取り出すのに適しています。
- **`for...in` 文**: オブジェクトのプロパティを列挙するために使用します。配列に対して使用すると、要素のインデックスを文字列として列挙するため、配列の反復には `for...of` や `forEach` を使うのが一般的です。

### 制御構造のベストプラクティス

ループの中で、特定の条件下で処理をスキップしたい場合は `continue` を、ループ自体を終了したい場合は `break` を使用します。

プログラムの可読性を高めるためには、複雑な条件式を `if` 文の中に直接書くのではなく、意味のある名前の変数や関数（例: `const isEligible = age >= 18 && hasPermission;`）に抽出することをお勧めします。

また、`map`、`filter`、`reduce` といった配列のイテレータメソッド（関数型プログラミング）を積極的に使うことで、ループ処理を「手続き的」ではなく「宣言的」に記述できます。これにより、ループ変数の管理ミスを防ぎ、コードの意図をより明確に表現できます。

制御構造はプログラミングの骨格です。ただ動くコードを書くのではなく、後から読み返したときにロジックが瞬時に理解できる、清潔なコード構造を目指しましょう。
