# オブジェクト型：データの構造化とプロトタイプ継承

JavaScriptにおけるオブジェクトは、複数の値や振る舞いを一つの単位としてまとめることができる、最も強力で中心的なデータ構造です。JavaScriptのほぼすべて（関数や配列も含めて）がオブジェクトの特性を持っています。

## オブジェクトの定義と操作

オブジェクトはキー（プロパティ名）と値のペアを保持する辞書構造です。

```javascript
const user = {
  name: 'Alice',
  age: 30,
  greet() {
    console.log(`Hello, my name is ${this.name}`);
  }
};
```

### プロパティへのアクセス
プロパティには、「ドット記法（`user.name`）」と「ブラケット記法（`user['name']`）」があります。ブラケット記法は、プロパティ名が変数である場合や、特殊文字を含む場合に使用します。

### モダンなオブジェクト操作
ES6以降、オブジェクトを効率的に扱うための構文が追加されました。

- **プロパティ短縮記法**: `const obj = { name, age };`
- **分割代入 (Destructuring)**: `const { name, age } = user;`
- **スプレッド演算子**: `{ ...user, age: 31 }` で一部を更新した新しいオブジェクトを作成。

## プロトタイプチェーン：JavaScriptの継承モデル

JavaScriptは「クラスベース」ではなく「プロトタイプベース」の継承を採用しています。各オブジェクトは内部的に `[[Prototype]]` という隠れたプロパティを持っており、そこから親オブジェクトのプロパティやメソッドを継承します。

```javascript
const animal = {
  eat() { console.log('Eating...'); }
};

const dog = Object.create(animal);
dog.bark = function() { console.log('Woof!'); };

dog.eat(); // プロトタイプ(animal)からメソッドを継承している
```

ES6で導入された `class` 構文は、このプロトタイプチェーンの仕組みに「糖衣構文（シンタックスシュガー）」を被せたものであり、他のクラスベース言語から来た開発者にとって理解しやすく設計されています。

## オブジェクトの安全性と制御

JavaScriptのオブジェクトはデフォルトで可変（Mutable）です。これは予期せぬ場所でのデータ変更を引き起こす原因となります。安全性を高めるために以下の手法が使われます。

### 不変性 (Immutability) の制御
`Object.freeze()` はオブジェクトを完全に不変にします。
```javascript
const config = Object.freeze({ theme: 'dark' });
// config.theme = 'light'; // 厳格モードではエラー、そうでなければ無視される
```

他にも `Object.seal()`（既存プロパティの変更は可能だが追加・削除不可）などがあり、状況に応じて使い分けることが重要です。

## 結論

オブジェクトは単なるデータの入れ物ではなく、状態を管理し、振る舞いを定義するための基盤です。特にモダンなJS開発においては、オブジェクトのスプレッド演算子や分割代入を駆使して、不変性を意識した設計を行うことが「予測可能でクリーンなコード」への近道です。
