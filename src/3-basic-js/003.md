# プリミティブ型：データの本質

JavaScriptのデータ型は、大きく「プリミティブ型」と「参照型（オブジェクト型）」の2つに分類されます。この分類を理解することは、メモリの使われ方や変数の振る舞いを正しく把握するために不可欠です。

### 主要なプリミティブ型

JavaScriptには、以下の7つのプリミティブ型が存在します。

- **String**: 文字列。`'hello'` や `"world"`、ES6で導入されたテンプレートリテラル `` `hello ${name}` `` などがあります。文字列はイミュータブルです。
- **Number**: 数値。JavaScriptは、すべての数値を64ビット浮動小数点数（IEEE 754形式）として扱います。そのため、`0.1 + 0.2 !== 0.3` のような浮動小数点演算特有の誤差が発生することがあります。
- **Boolean**: 真偽値。`true` または `false`。条件分岐の基礎です。
- **Null**: 「意図的な値の欠如」を示します。`typeof null` が `'object'` を返すのは、JavaScriptの言語仕様上の歴史的なバグです。
- **Undefined**: 「未定義」を示します。変数が宣言されたが値が代入されていない場合や、関数の引数が不足している場合、関数が何も返さない（戻り値がvoidである）場合に発生します。
- **Symbol**: ES6で導入された、一意な値です。オブジェクトのプロパティキーとして使用することで、プロパティ名の衝突を避けることができます。
- **BigInt**: 非常に大きな整数を扱うための型です。通常のNumber型では `Number.MAX_SAFE_INTEGER`（2の53乗 - 1）を超えると精度が保証されませんが、BigIntを使うことで任意精度の整数演算が可能です。

### プリミティブ型の特性：イミュータビリティとボクシング

プリミティブ型は「イミュータブル（不変）」です。つまり、一度作成された値自体を変更することはできません。変数に新しい値を代入したときは、その変数が新しい値を指すようになるだけで、元の値が変化したわけではありません。

```javascript
let name = 'Alice';
name = 'Bob'; // 'Alice'という文字列自体が'Bob'になったわけではない
```

しかし、プリミティブ型には「メソッド」が存在するように見えます（例: `'hello'.toUpperCase()`）。これは、JavaScriptが一時的にプリミティブ値を対応するラッパーオブジェクト（String型なら`String`オブジェクト）に自動的に変換（ボクシング）し、メソッド実行後に破棄しているからです。開発者はこの挙動を意識することなく、プリミティブ型に対してオブジェクトのようなメソッド呼び出しを行うことができます。

### nullとundefinedの使い分け

どちらも「値がない」ことを表しますが、その文脈は異なります。

- `undefined`: 「システムによって意図せず値が与えられていない」ことを示します。変数定義後に値が設定されていない状態です。
- `null`: 「開発者が意図的に値がないことを示す」ために使用します。

堅牢なプログラムを書くためには、不要な `undefined` の発生を避け、意図した値を明示的に管理することが重要です。プリミティブ型の特性を深く理解し、そのイミュータビリティを活かした副作用の少ない設計を目指しましょう。
