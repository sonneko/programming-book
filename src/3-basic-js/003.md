# プリミティブ型

JavaScriptのデータ型は、大きく「プリミティブ型」と「参照型」の2つに分類されます。プリミティブ型は、値そのものがメモリ上に保持され、変更不可能な（immutableな）データです。

### 主要なプリミティブ型

JavaScriptには、以下のプリミティブ型が存在します。

- **String**: 文字列。`'hello'` や `"world"` のように記述します。
- **Number**: 数値。整数や浮動小数点数を扱います（例: `42`, `3.14`）。
- **Boolean**: 真偽値。`true` または `false` のいずれかをとります。
- **Null**: 「値がない」ことを意図的に示す型。
- **Undefined**: 「未定義」であることを示す型。変数が宣言されただけで値が代入されていない場合に自動的に割り当てられます。
- **Symbol**: 一意な識別子として使用される値（ES6で導入）。
- **BigInt**: 非常に大きな整数を扱うための型（比較的新しい型）。

### プリミティブ型の特性：イミュータビリティ

プリミティブ型は「イミュータブル（不変）」です。つまり、一度作成された値自体を変更することはできません。変数に新しい値を代入したときは、その変数が新しい値を指すようになるだけで、元の値が変化したわけではありません。

```javascript
let name = 'Alice';
name = 'Bob'; // 'Alice'という文字列自体が'Bob'になったわけではない
```

この特性により、予期せぬデータの変更（副作用）を防ぐことができ、プログラムの安定性を高めることができます。
