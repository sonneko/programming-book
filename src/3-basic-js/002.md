# 変数定義

JavaScriptにおける変数定義は、データの状態を保持し、操作するための最も基本的な手段です。時代とともに、`var`、`let`、`const`と変遷してきました。

### 変数定義の変遷

- `var`: 古いJavaScriptで使われていた変数定義。関数スコープを持ち、再定義や再代入が可能で、意図しない挙動（ホイスティングによる混乱）を引き起こしやすいという問題がありました。現代のJavaScriptでは使用を避けるべきです。
- `let`: ES6（ECMAScript 2015）で導入されました。ブロックスコープ（`{}`内）を持ち、再代入が可能ですが、再定義はできません。通常、値が変化する変数にはこちらを使用します。
- `const`: ES6で導入されました。ブロックスコープを持ち、再代入も再定義もできません。値が不変な変数には、積極的にこれを使用します。

### constとletの使い分け

現代のJavaScript開発では、**デフォルトで `const` を使い、どうしても再代入が必要な場合のみ `let` を使う**というのが最も推奨されるスタイルです。これにより、意図しない変数の書き換えを防ぎ、コードの安全性を高めることができます。

```javascript
// 推奨されるスタイル
const PI = 3.14159;
let count = 0;

count++; // 再代入が必要なのでlet
```

注意点として、`const` で宣言したオブジェクトや配列のプロパティは変更可能です（参照先自体を変えることはできません）。

### スコープの理解

ブロックスコープとは、`if`文や`for`文、関数などで囲まれた `{}` の内側を指します。`let` や `const` はこのスコープ内でのみ有効であり、スコープ外からはアクセスできません。これにより、変数名の衝突を防ぐことができます。
