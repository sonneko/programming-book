# switch文：制御の分岐と最適化

`switch` 文は、一つの式の値に基づいて、複数のケース（分岐）を扱うための制御構造です。`if-else if` を連ねるよりも、特定の値を比較する場合にはコードの見通しが良くなる場合があります。

### 基本的な構文

```javascript
const fruit = 'apple';

switch (fruit) {
  case 'apple':
    console.log('りんごです');
    break;
  case 'banana':
    console.log('バナナです');
    break;
  default:
    console.log('その他の果物です');
}
```

### switch文の挙動と注意点：フォールスルー

`switch` 文の最も重要な特徴であり、かつバグの温床となるのが「フォールスルー（Fall-through）」です。各 `case` の後に `break` 文を記述しないと、そのケースの処理が終わった後、続くすべてのケースの処理が（条件を無視して）実行されてしまいます。

```javascript
const command = 'start';

switch (command) {
  case 'start':
    console.log('Starting...');
    // breakがないため、次のcaseも実行される
  case 'stop':
    console.log('Stopping...');
    break;
}
```

この挙動は、複数のケースで同じ処理を共有したい場合に便利ですが、予期せぬ挙動を防ぐためには、意図的にフォールスルーを利用する場合を除き、すべての `case` の最後に `break`（または `return`）を記述することが不可欠です。

### 比較の仕様

`switch` 文の式と各 `case` の値は、厳密等価演算子（`===`）を用いて比較されます。したがって、型が異なる場合は一致とみなされません（例：`'1'` は `1` とは一致しません）。この厳密な型チェックはバグを防ぐのに役立ちますが、予期しない型の不一致には注意が必要です。

### switch文の代替案：オブジェクトとMap

現代のJavaScript開発では、複雑な `switch` 文よりも、オブジェクトリテラルや `Map` を使ったマッピングの方が、コードが簡潔で拡張性が高いとみなされることが多いです。これを「ディスパッチテーブル（Dispatch Table）」と呼びます。

```javascript
// オブジェクトリテラルによるマッピング
const fruitMessages = {
  apple: 'りんごです',
  banana: 'バナナです',
  default: 'その他の果物です'
};

const fruit = 'apple';
console.log(fruitMessages[fruit] || fruitMessages.default);
```

このように、マッピングを使うことで、`switch` 文特有の `break` の記述忘れを防ぎ、コードをより宣言的に書くことができます。`switch` 文は、分岐条件がシンプルで、処理が少ない場合に限定して使用し、分岐が複雑になる場合はマッピングや他のロジックへの切り出しを検討しましょう。
