# コレクション型：データの管理と構造化

JavaScriptにおいて、複数のデータをまとめて扱うためのデータ構造を「コレクション」と呼びます。主に配列（Array）とオブジェクト（Object）が使われますが、ES6以降は `Map` や `Set` といった特殊なコレクションも標準で利用可能になり、用途に応じた使い分けが重要になっています。

### 配列 (Array)

配列は、順序付けられたデータの集合です。内部的には特殊なオブジェクトであり、インデックス（0から始まる番号）がキーとして機能します。

```javascript
const numbers = [1, 2, 3, 4, 5];
console.log(numbers[0]); // 1
```

配列の真の力は、組み込みのイテレータメソッドにあります。`map()` は全要素に変形を加えた新しい配列を作成し、`filter()` は条件に合う要素だけを抽出します。また、`reduce()` を使うと、配列全体を単一の数値やオブジェクトに集約できます。これらを使うことで、forループを直接書くよりも宣言的でミスの少ないコードが書けます。

### オブジェクト (Object)

オブジェクトは、キーと値のペアを保持する集合です。順序はなく、キーを使って値にアクセスします。オブジェクトは、データの構造化（構造体）としてだけでなく、動的なキーと値の保持にも使われます。

```javascript
const user = {
  name: 'Alice',
  age: 25
};
console.log(user.name); // Alice
```

オブジェクトの操作には、スプレッド構文（`...`）が非常に強力です。新しいオブジェクトを作成する際、既存のプロパティをコピーしながら新しいプロパティを追加したり、特定のプロパティを上書きしたりする処理が容易に行えます。また、`Object.freeze()` を使うと、オブジェクトの変更を防ぎ、イミュータブルな状態を強制できます。

### Map と Set：ES6以降の選択肢

- **Map**: キーと値のペアを保持します。オブジェクトとの最大の違いは、キーにオブジェクトそのものを含む「あらゆるデータ型」を使用できること、そして要素の挿入順序が保持されることです。また、`size` プロパティで要素数を即座に取得できる点も優れています。大量のデータを頻繁に追加・削除する場合、`Map` の方がオブジェクトよりもパフォーマンスが優れている場合があります。
- **Set**: 重複のない値の集合を保持します。配列から重複を取り除きたい場合、`new Set(array)` と記述するだけで簡単に重複除去が可能です。

最後に、`WeakMap` と `WeakSet` についても触れておきます。これらはキー（WeakMapの場合）や値（WeakSetの場合）を弱参照（weak reference）として保持するため、オブジェクトが他の場所で参照されなくなると自動的にガベージコレクションの対象となります。これはメモリリークを防ぐための高度な最適化手法として非常に有用です。

### データ構造の選び方

どのコレクションを使うべきかは、そのデータの性質に依存します。

1. **データが順序を持つか？** -> 配列
2. **キーを元にデータにアクセスするか？** -> オブジェクト または Map
3. **要素の重複を許さないか？** -> Set
4. **キーに文字列以外の型を使いたいか？** -> Map

これらのコレクションを正しく使い分けることで、データの効率的な管理が可能になります。データの性質を見極め、適切な構造を選択することが、パフォーマンスと保守性の両立に繋がります。
