# コレクション型：データの管理と構造化

プログラミングをしていると、たった一つの値（数値や文字列）だけを扱うことはほとんどありません。「ユーザーのリスト」「カートに入った商品の情報」「設定値のセット」など、関連する複数のデータをまとめて管理する必要があります。

JavaScript において、これらの「データの集まり」を扱うための仕組みを **コレクション (Collection)** と呼びます。主に「配列」と「オブジェクト」の 2 つが主役ですが、モダンな JavaScript では他にも強力な道具が用意されています。

## 1. 配列 (Array)：順序のあるリスト

同じ種類のデータを、順番に並べて管理するのに最適です。

```javascript
const fruits = ['apple', 'banana', 'orange'];
console.log(fruits[0]); // 'apple' (0から数える！)
```

配列の真の魅力は、その **「メソッド」** の豊富さにあります。

- **`push`**: 最後に追加。
- **`pop`**: 最後を削除。
- **`map`**: 全要素を加工して新しい配列を作る。
- **`filter`**: 条件に合う要素だけを取り出す。

```javascript
// 例：数値を2倍にする
const numbers = [1, 2, 3];
const doubled = numbers.map(n => n * 2); // [2, 4, 6]
```

## 2. オブジェクト (Object)：キーと値のペア

「名前はアリス、年齢は 25 歳」といった、ラベル（名前）を付けてデータを管理するのに最適です。

```javascript
const user = {
  id: 1,
  name: 'Alice',
  age: 25,
  isPremium: true
};

console.log(user.name); // 'Alice'
```

オブジェクトの操作で最近欠かせないのが、**スプレッド構文 (`...`)** です。元のデータを壊さずに、一部だけを書き換えた新しいオブジェクトを簡単に作れます。

```javascript
// age だけ 26 に更新した新しいオブジェクトを作る
const updatedUser = { ...user, age: 26 };
```

## 3. Map と Set：特殊なコレクション

ES6 (2015年) から追加された、より専門的なコレクションです。

- **`Map`**: オブジェクトに似ていますが、**「キーにどんな型でも（オブジェクト自体でも！）使える」** という特徴があります。また、要素の順番が保証されるのもメリットです。
- **`Set`**: **「重複を許さない」** 値の集まりです。同じ値を 2 回入れようとしても、無視されます。
  ```javascript
  const tags = new Set(['js', 'ts', 'js']);
  console.log(tags.size); // 2 (js は重複しているので削られる)
  ```

## 低レイヤーの視点：メモリ上の配置と計算量

- **配列**: メモリ上に「連続して」データが並んでいます。そのため、「3 番目のデータを見せて」という指示には、先頭から計算して一瞬でアクセスできます（これを **O(1)** と呼びます）。
- **オブジェクト (ハッシュマップ)**: データがメモリのあちこちに散らばっています。「どの名前がどこにあるか」を記録した地図（ハッシュテーブル）を使ってアクセスします。

もし、1 万件のデータの中から「特定の ID を持つ人」を探したいとき、配列だと端から順番に探す（最悪 1 万回チェックする）必要がありますが、オブジェクトや `Map` なら、一撃で（1 回のチェックで）見つけ出すことができます。
「データの量」と「どうやって取り出したいか」によって、最適なコレクションを選ぶのがエンジニアの腕の見せ所です。

## 私の視点：イミュータビリティ（不変性）の追求

私がコードを書くときに最も気をつけているのが、**「元の配列やオブジェクトを直接書き換えない」** ことです。

```javascript
// 悪い例
const list = [1, 2, 3];
list.push(4); // 元の list が変わってしまう

// 良い例
const list = [1, 2, 3];
const newList = [...list, 4]; // 新しい配列を作る
```

元のデータを書き換えてしまうと、「いつ、どこでデータが変わったのか」を追いかけるのが不可能になり、バグの原因になります。
モダンな JavaScript 開発（特に React など）では、この「新しいデータを作って、古いデータは残しておく」というスタイルが基本となります。

### まとめ

- **配列**: 順番が重要。`map` や `filter` で加工する。
- **オブジェクト**: ラベルで管理。スプレッド構文が便利。
- **Map / Set**: キーの自由度や、重複排除が必要なときに使う。
- データの検索が多いならオブジェクトや Map が高速。
- 元のデータを壊さない「不変性」を意識して操作しよう。

### 展望
