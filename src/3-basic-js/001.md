# JavaScriptとは

JavaScriptは、現代のWeb開発において不可欠なプログラミング言語です。もともとはブラウザ上で簡単な動的挙動を実現するためにNetscape社によって開発されましたが、今やWebブラウザを超え、サーバーサイド（Node.js）、デスクトップアプリ（Electron）、モバイルアプリ（React Native）など、あらゆる場所で動作する言語へと成長しました。

### プログラムは「データの変形」である
プログラミングを学ぶ際、初心者は「AをしてからBをする」という手順（手続き）に目を奪われがちです。しかし、JavaScriptの真髄は**データの変形**にあります。

例えば、ユーザーからの入力を受け取り、それをDOM（Document Object Model）として画面に描画するプロセスを考えましょう。これは、非同期のイベントストリーム（入力データ）を、DOMツリーという構造データへと変換し、ブラウザのレンダリングエンジンへ渡す計算処理です。

### 低レイヤの現実：V8エンジンとイベントループ
JavaScriptは「インタプリタ言語」と紹介されることが多いですが、現代のモダンなJavaScriptエンジン（Google ChromeのV8など）は、実行時にバイトコードへコンパイル（JIT: Just-In-Timeコンパイル）を行うため、非常に高速に動作します。

また、JavaScriptの最大の特徴は「シングルスレッド」であることです。これは、一度に一つの処理しか行えないことを意味します。では、なぜ複数のネットワークリクエストを同時に処理できるのでしょうか？

ここに「イベントループ」という概念があります。JavaScriptは、重いI/O処理（ファイル読み込みやネットワークリクエスト）を、ブラウザやOSの機能（Web APIやシステムコール）へ投げ込み、自分はメインスレッドで他の計算を続けます。その処理が終わったときに、イベントキューに登録されたコールバック関数がメインスレッドに戻り、処理が再開されます。

### なぜJavaScriptなのか
JavaScriptが存在する理由は、Webというプラットフォームとの結合にあります。HTMLが骨組み、CSSが装飾だとしたら、JavaScriptはその上に「意志」を与えます。
また、TypeScriptのような静的型付け言語を上位レイヤに持つことで、JavaScriptの持つ柔軟性を維持しながら、大規模なプロジェクトでの保守性を担保できる点も、この言語が選ばれる理由です。
