# propsとchildrenの扱い

React コンポーネントは「部品」です。部品である以上、外からデータを受け取って動きを変えたり、他の部品を中に組み込んだりできる必要があります。そのための仕組みが **props** と **children** です。

これらを使いこなすことで、あなたのコードは「コピペの繰り返し」から「洗練された再利用可能な設計」へと進化します。

## 1. props：親から子へのメッセージ

**props** は、関数の引数のようなものです。親コンポーネントが子コンポーネントを呼び出す際、属性のような形でデータを渡します。

```tsx
// 子コンポーネント
function UserProfile({ name, age }: { name: string, age: number }) {
  return (
    <div>
      <p>名前: {name}</p>
      <p>年齢: {age}</p>
    </div>
  );
}

// 親コンポーネント
function App() {
  return <UserProfile name="アリス" age={25} />;
}
```

React におけるデータの流れは **「単一方向（トップダウン）」** です。
水が高いところから低いところへ流れるように、データは常に親から子へと流れます。この制約があるおかげで、「このデータはどこから来たのか？」という原因究明が非常に楽になります。

## 2. children：コンポーネントの「中身」を包み込む

特定のタグで囲んだ中身をそのまま受け取りたいときに使うのが、特別な props である **`children`** です。

```tsx
function Card({ children }: { children: React.ReactNode }) {
  return (
    <div className="card-frame">
      {children}
    </div>
  );
}

// 呼び出し側
<Card>
  <h2>注目記事</h2>
  <p>この記事の内容は...</p>
</Card>
```

このパターンを **「コンポジション（合成）」** と呼びます。
`Card` コンポーネントは中身が何であるかを知る必要はありません。ただ「枠」を提供することに専念し、中身は使う側が自由に決める。この「責務の分離」こそが、美しい設計の第一歩です。

## 低レイヤーの視点：参照の比較（Shallow Compare）

React が「再レンダリング（画面の描き直し）」をするかどうかを判断するとき、props の中身をどう見ているのでしょうか。

実は React は、パフォーマンスを稼ぐために **「浅い比較 (Shallow Compare)」** という手法を使っています。

- `number` や `string` などの値が変われば、再レンダリングします。
- `object` や `array` の場合、中身が変わっていても **「メモリ上の住所（参照）」** が同じなら、React は「変わっていない」と判断して再描画をスキップすることがあります。

だからこそ、React では「元のオブジェクトを書き換える」のではなく、「新しいオブジェクトを作って渡す」ことが重要なのです。住所が変われば、React は確実に変更を検知できるからです。

## 私の視点：Props Drilling（バケツリレー）を避ける

アプリが大きくなると、一番上の親から、5 階層下の子まで、props をひたすら中継して渡す「バケツリレー」が発生しがちです。

```tsx
<Parent>
  <Child data={data}>
    <GrandChild data={data}>
      <TargetComponent data={data} />
    </GrandChild>
  </Child>
</Parent>
```

これはコードの保守性を著しく下げます。そんな時は、先ほど学んだ `children` を使って、親が直接目的のコンポーネントを配置する設計に変更したり、後で学ぶ **Context API** を検討しましょう。
「誰がどのデータを知るべきか」を整理することは、プログラミングにおいて最も知的なパズルの一つですよ。

### まとめ

- props は親から子へデータを渡すための仕組み（引数のようなもの）。
- `children` を使うと、中身を自由に流し込める「枠」を作れる（コンポジション）。
- React のデータフローは常に「上から下」の一方向。
- 再レンダリングの判定は「参照の比較」で行われるため、不変性が大事。
- バケツリレー（Props Drilling）が起きたら、設計を見直すサイン。

### 展望

props と children でコンポーネントの見た目を整えられるようになったら、次は React の核心である「思想」と「データフロー」のさらなる深層へ。なぜこの不便とも思える制約が、大規模開発を救うのかをお話しします。
