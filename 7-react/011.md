# 特別な関数たち

React や Next.js でアプリケーションを構築していると、通常の JavaScript の知識だけでは説明がつかない、特別な挙動をする関数や API に出会います。これらは、React が「レンダリングを最適化」したり、「サーバーとクライアントを橋渡し」したりするために用意した、エンジニアのための **「専門的な道具」** です。

これらを使いこなすことは、単にアプリを作るだけでなく、「品質の高い、プロのアプリ」を作るための分岐点になります。

## 1. 最適化の三銃士：memo, useMemo, useCallback

React はデフォルトで「親が変われば子も変わる」という動きをします。しかし、何もしないと不必要な再計算が走り、サイトが重くなる原因になります。

- **`React.memo`**: コンポーネントを包むことで、「props が変わらない限り、再描画をスキップしていいよ」と伝えます。
- **`useMemo`**: 重い計算（例：1万件のデータのフィルタリング）の結果を保存し、依存する値が変わらない限り再計算させません。
- **`useCallback`**: 関数の定義自体を保存します。

特に `useCallback` は重要です。JavaScript では `{}` や `() => {}` は、見た目が同じでも作るたびに **「別物（新しいメモリ住所）」** とみなされます。`useCallback` を使わないと、せっかく `memo` した子コンポーネントに「新しい関数が来た！」と勘違いさせて、再描画を誘発してしまいます。

## 2. Next.js の魔法：fetch の拡張

Next.js (App Router) では、標準の `fetch` 関数が大幅に強化されています。

```tsx
// この 1 行で、Next.js はデータを「キャッシュ（保存）」します
const res = await fetch('https://api.example.com/data', {
  next: { revalidate: 3600 } // 1時間はキャッシュを使い回す
});
```

かつての Web 開発では、Redis のような外部データベースを使って自分でキャッシュを管理していましたが、Next.js なら `fetch` のオプション一つで「サーバーのメモリに保存しておく」といった高度な制御が可能になります。

## 低レイヤーの視点：メモ化とメモリ・トレードオフ

「すべてを `useMemo` で囲めば速くなるんじゃない？」と考える人もいるかもしれません。
しかし、**「メモ化（保存）」は無料ではありません。**

- **時間**: 計算をスキップして速くなる。
- **空間 (メモリ)**: 前回の結果をずっとメモリに保持し続ける必要がある。

メモリ容量の少ないスマートフォンでは、あまりに多くの値を保存しすぎると、逆にアプリ全体の動作が不安定になる（GC が頻発する）こともあります。
「重い計算かどうか」をしっかり見極め、必要な場所にだけピンポイントで使う。それが、ハードウェアへのリスペクトを持ったエンジニアの姿です。

## 私の視点：まずは「最適化しない」勇気

プログラミングの世界には **「早すぎる最適化は諸悪の根源」** という有名な言葉があります。

コードを `useMemo` や `useCallback` で埋め尽くすと、依存配列の管理が複雑になり、逆にバグを生みやすくなります。
まずは素直なコードを書き、React DevTools の **Profiler** という機能を使って、本当に重い部分だけを後から直す。
「正しく動く、読みやすいコード」を一番に。速くするのは、その次でいいのですよ。

### まとめ

- `memo`, `useMemo`, `useCallback` は、不必要な再描画や再計算を防ぐための道具。
- JavaScript の「参照（住所）」の概念を理解することが、最適化の第一歩。
- Next.js は `fetch` を拡張し、サーバーレベルでのキャッシュ管理を実現している。
- メモ化には「メモリ消費」というコストが伴うことを忘れない。
- Profiler で計測し、ボトルネックを見極めてから最適化する。

### 展望
