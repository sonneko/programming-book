# 動的ルーティング

Web サイトを作っていると、URL の一部だけが異なるページを大量に作りたい場面があります。
たとえば、ブログの `/posts/1` や `/posts/2`、ユーザープロフィールの `/users/watashi` などです。

これらを 1 ページずつ手作業で作るのは不可能です。そこで登場するのが **「動的ルーティング (Dynamic Routing)」** です。Next.js では、フォルダ名にカッコ `[]` を使うという非常にユニークな方法でこれを実現します。

## 1. カッコ `[]` の魔法

Next.js の `app` ディレクトリの中に、以下のようなフォルダを作ってみましょう。

```text
app/
 └── posts/
      └── [id]/
           └── page.tsx
```

このフォルダ構造にするだけで、Next.js は `/posts/apple` でも `/posts/123` でも、どんな値が来てもこの `page.tsx` を表示してくれるようになります。

## 2. パラメータを受け取る

URL のカッコ `[id]` の部分に入力された値は、コンポーネントの `params` という引数（props）から取得できます。

```tsx
// app/posts/[id]/page.tsx
export default function PostPage({ params }: { params: { id: string } }) {
  // URL が /posts/123 なら、params.id は "123" になる
  return (
    <div>
      <h1>記事 ID: {params.id}</h1>
      <p>ここで ID を元に DB から記事を取得します。</p>
    </div>
  );
}
```

通常、ルーターは URL 文字列を **正規表現**（例：`^\/posts\/([^/]+)$`）に変換してマッチングを行います。
カッコ `([^/]+)` の部分にヒットした文字列を「変数」として抽出し、コンポーネントに渡します。
Next.js は、私たちがフォルダ名を `[id]` とつけるだけで、この複雑な正規表現の生成とパラメータの抽出を、ビルド時に自動でセットアップしてくれているのです。

## 私の視点：SEO と「綺麗な URL」

動的ルーティングを使う際は、ID だけでなく「スラッグ（意味のある単語）」を URL に含めることをお勧めします。

- 悪い例：`/posts/982347` (何の記事かわからない)
- 良い例：`/posts/how-to-use-react` (Google も人間も中身がわかる)

Next.js の動的ルーティングを使えば、こうした「人間に優しい URL」を簡単に生成できます。
技術的な仕組みを覚えるだけでなく、**「ユーザーにとって、どんな URL が最も使いやすいか？」** という視点を持つことが、一人前のエンジニアへの近道ですよ。

### まとめ

- フォルダ名を `[name]` とすることで、URL の一部を変数にできる。
- `params` プロパティを通じて、URL の変数部分を取得できる。
- ID だけでなく、スラッグ（単語）を URL に使うのが SEO 上も有利。
- 裏側では、URL 文字列の解析と正規表現によるマッチングが行われている。
- 大量のコンテンツ（ブログ、EC サイトなど）を作るには必須の技術。

### 展望

動的ルーティングまで使いこなせるようになったら、Next.js で作れないサイトはほとんどありません。
