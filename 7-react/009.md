# Next.jsとは

「React は学んだ。でも、本番で使える Web アプリを作るには、まだ足りない……」。
ルーティング、画像の最適化、SEO 対策、サーバーとの連携。これらをすべて自前で構築するのは、気の遠くなるような作業です。

そこで登場するのが、React をベースにした最強のフレームワーク **Next.js** です。
Next.js は、React だけでは手が届きにくかった「パフォーマンス」と「開発体験」の課題を、一挙に解決してくれます。現在、モダンなフロントエンド開発において、React を使うなら Next.js を使う、というのが事実上の標準になっています。

## 1. レンダリングの「フルコース」

Next.js の最大の特徴は、コンテンツに合わせて最適なレンダリング手法（画面の作り方）を選べることです。

- **SSR (Server Side Rendering)**: リクエストがあるたびに、サーバーで HTML を作りたてで送ります。最新の情報が必要なサイト（ニュースなど）に向いています。
- **SSG (Static Site Generation)**: ビルド時（アプリを公開する前）に、あらかじめ HTML をすべて作っておきます。爆速で表示でき、SEO にも最強です（ブログなど）。
- **ISR (Incremental Static Regeneration)**: SSG をしつつ、バックグラウンドで一定時間ごとに HTML を更新します。「速さ」と「新しさ」のいいとこ取りです。

「全部 JavaScript で作る (SPA)」という極端なやり方から、必要な部分だけをサーバーに任せる柔軟なやり方へ。Next.js は Web の歴史を総括したような進化を遂げています。

## 2. ゼロ・コンフィギュレーション（設定不要）

Next.js をインストールした瞬間から、多くの強力な機能が「当たり前」のように使えます。

- **ファイルベースルーティング**: `app` フォルダにファイルを置くだけで、それがそのまま URL（ルーティング）になります。
- **画像の自動最適化**: `<Image />` コンポーネントを使うだけで、ブラウザに合わせて画像をリサイズし、次世代フォーマット（WebP）に変換してくれます。
- **高速なビルド**: Rust で書かれた高速なコンパイラ（SWC）を内蔵しており、開発中の待ち時間が極限まで削られています。

## 低レイヤーの視点：ハイドレーション (Hydration)

サーバーで HTML を作って送る（SSR/SSG）場合、ブラウザに届いた時点ではただの「静的なドキュメント」です。ボタンを押しても反応しません。

ブラウザは HTML を表示した後、裏側で JavaScript を実行し、既存の HTML に対して React のイベントハンドラなどを「流し込み」ます。このプロセスを **ハイドレーション（水分補給）** と呼びます。
カラカラに乾いた静的な HTML に、JavaScript という「水」を与えて、インタラクティブなアプリとして復活させる。この緻密な連係プレーが、Next.js の爆速な体験を支えているのです。

## 私の視点：Next.js は「Web の正解」を教えてくれる

私が Next.js を使っていて感じるのは、「Next.js のルールに従って書くだけで、自然と『良い Web サイト』が出来上がってしまう」という感覚です。

SEO、アクセシビリティ、読み込み速度。これらはエンジニアが個別に努力すべきことでしたが、Next.js はそれらをフレームワークの機能として組み込んでしまいました。
「車輪の再発明」に時間を使うのではなく、あなたが本当に作りたい「価値（サービス）」に集中させてくれる。それこそが、Next.js が世界中で愛される理由ですよ。

### まとめ

- Next.js は React を本番環境で使うためのフルスタックフレームワーク。
- SSR, SSG, ISR など、最適なレンダリング手法を選択できる。
- ファイルを置くだけでルーティングができるなど、開発体験が非常に高い。
- ハイドレーションによって、高速な表示と高い操作性を両立している。
- パフォーマンス最適化（画像やフォント）が標準で組み込まれている。

### 展望

Next.js の全体像が見えたら、次は最新の Next.js 13 以降で導入された革新的な機能 **「App Router」** と **「Server Components」** について詳しく学びましょう。これまでの Web 開発の常識が再び塗り替えられます。
