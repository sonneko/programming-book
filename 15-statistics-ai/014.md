# 古典的自然言語処理の智慧

こんにちは、私です。今回は、AIが「言葉」を扱うための技術、 *Natural Language Processing*（NLP: 自然言語処理）の古典的な手法についてお話しします。

今でこそChatGPTのような魔法のようなAIが言葉を操りますが、かつては言葉をコンピュータに教えるのは至難の業でした。なぜなら、コンピュータは「意味」を理解できないからです。彼らにとって、言葉は単なる「文字の記号」の羅列にすぎません。

## 実務的側面：統計的アプローチと形態素解析

初期のNLPの実務は、いわば「言葉のカウント」でした。

まず最初に行うのは *Morphological Analysis*（形態素解析）です。「すもももももももものうち」という文章を、「すもも / も / もも / も / もも / の / うち」と分解します。日本語のように分かち書きをしない言語では、この分解だけでも高度な辞書とルールが必要でした。

その後、文章を「単語の出現頻度」に変換します。これが *Bag-of-Words*（BoW）です。
例えば、「猫が好き」という文と「犬が好き」という文があったとき、これらを単語のリストに変換し、どれだけ似ているかを計算します。

「特定の単語がよく出てくるから、これはスポーツの記事だろう」といった分類や、迷惑メールの判定などは、このレベルの「統計」だけでかなり実用的に動いていました。意味は分からなくても、単語の出現パターン（共起関係）を追うだけで、ある程度の「らしさ」は判定できる。この発見は、現代のLLMに繋がる重要な一歩でした。

## 数学的側面：ベクトル空間モデルと次元圧縮

数学的な観点から見ると、古典NLPの目標は「言葉をベクトル（数字の列）にする」ことです。

単語の種類が10万個あるなら、一つの文章を「10万次元のベクトル」として表現します。しかし、これだと数字のほとんどが0になってしまい（スパース性）、計算が非常に非効率です。

そこで、 *Dimension Reduction*（次元圧縮）の技術が使われました。 *LSA*（潜在意味解析）などは、行列計算を駆使して、10万次元のデータを「意味が凝縮された数百次元」にぎゅっと圧縮します。
これにより、「猫」と「子猫」はベクトル上の位置が近くなる、といった「意味の近さ」を計算できるようになりました。

`王 - 男 + 女 = 女王` という有名な数式（Word2Vec）の影には、こうした「言葉という不透明な記号を、いかに数学的な空間に配置するか」という数学者たちの執念があります。

## 哲学的側面：言語は「使用」の中に宿る

古典NLPの底流にある哲学的な考え方は、「単語の意味は、その単語の周囲に現れる単語によって決まる」という *Distributional Hypothesis*（分布仮説）です。

言語哲学者ウィトゲンシュタインは「言葉の意味とは、その使われ方である」と言いました。
AIに「愛」という言葉の辞書的な定義を教えるのではなく、どんな文脈で「愛」が使われるかを統計的に学習させる。そうすれば、AIは「意味」を理解していなくても、その言葉を「適切に」扱うことができるようになります。

これは、知性における「本質」への問いを回避し、「形式（振る舞い）」に徹する姿勢です。
私たちが他人の心を「本質的に」知ることはできません。ただ、相手が状況に応じて適切な言葉を発しているのを見て、「この人は言葉を理解している」と判断しています。NLPはこの人間観を徹底的にコンピュータに移植しようとする試みなんです。

## 次の革命：注意を向ける力

古典NLPは、文章を「単語の集合（袋）」や「単線のデータの流れ」として扱っていました。しかし、言葉には「ずっと前の伏線が今回収される」とか「代名詞の『それ』が何を指すか」といった、遠く離れた単語同士の深い繋がりがあります。

この複雑な繋がりを捉えきれなかった従来のNLPを、根底から破壊し再構築したのが、次節でお話しする「Attention（注意）」メカニズムです。
AIが「どこに注目すべきか」を自ら判断し始めたとき、物語はついにクライマックスへと向かいます。
