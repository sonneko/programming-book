# オブジェクト型：データの構造化とプロトタイプ継承

JavaScript を「ただのスクリプト言語」から「プロフェッショナルな開発ツール」へと昇華させているのが、この **オブジェクト (Object)** という概念です。

「コレクション型」の章では、オブジェクトを「キーと値のペアを保存する箱」として紹介しました。しかし、JavaScript におけるオブジェクトはそれだけではありません。データだけでなく、**「振る舞い（関数）」** をも自分の中に持つことができるのです。

## 1. データとメソッド：自分自身の状態を知る

オブジェクトの中に定義された関数のことを **メソッド (Method)** と呼びます。メソッドの中では、特別な変数 **`this`** を使うことで、自分自身のデータを参照できます。

```javascript
const player = {
  name: '勇者',
  hp: 100,
  maxHp: 100,
  // これがメソッド
  heal(amount) {
    this.hp = Math.min(this.hp + amount, this.maxHp);
    console.log(`${this.name}は回復した！残りHP: ${this.hp}`);
  }
};

player.heal(20);
```

この「データとその操作方法を一箇所にまとめる」という考え方は、大規模なプログラムを整理する **オブジェクト指向プログラミング (OOP)** の基礎となります。

## 2. 破壊的な操作と非破壊的な操作

オブジェクトのデータを更新するとき、初心者は直接書き換えがちです。

```javascript
const user = { name: 'Alice' };
user.name = 'Bob'; // 破壊的操作 (直接書き換え)
```

しかし、これまでに何度も触れてきた通り、モダンな開発では **「非破壊的な操作」** が好まれます。

```javascript
const user = { name: 'Alice', age: 25 };

// スプレッド構文を使って、名前だけ違う「新しい」オブジェクトを作る
const updatedUser = { ...user, name: 'Bob' };
```

なぜわざわざ新しいオブジェクトを作るのでしょうか？
それは、もし複数の場所で `user` を参照していた場合、片方で勝手に書き換えると、もう片方の処理が予期せぬ挙動を起こす（バグになる）からです。

JavaScript のオブジェクトには、**「プロトタイプ (Prototype)」** という、他の言語（Java や C++）にはないユニークな仕組みがあります。

例えば、あなたが作った適当なオブジェクトで、定義もしていない `.toString()` というメソッドが呼べるのはなぜでしょうか？
それは、すべてのオブジェクトが背後に「親オブジェクト（プロトタイプ）」を持っており、自分にない機能は親に聞きに行くからです。

この「親から機能を受け継ぐ」仕組みを **継承** と呼びます。現代の JavaScript では `class` という構文を使ってより綺麗に書けますが、その裏側では今もこのプロトタイプという仕組みが動いています。

## 低レイヤーの視点：隠しクラス (Hidden Classes)

JavaScript のオブジェクトは、実行中に自由にプロパティを増やしたり消したりできます。

```javascript
const obj = {};
obj.a = 1;
obj.b = 2;
```

これは便利ですが、コンピュータにとっては最悪の状況です。メモリ上のどこにどのデータがあるか予測できないからです。

V8 エンジンは、この問題を解決するために **「隠しクラス (Hidden Classes)」** という仕組みを導入しました。
内部的に、プロパティの並び順が同じオブジェクトを同じ「クラス」として分類し、メモリ上のオフセット（距離）を計算して固定します。

## 私の視点：厳格モード (Strict Mode)

JavaScript には、昔の「ゆるすぎる」挙動を制限し、ミスを防ぐための **`'use strict';` (厳格モード)** があります。

モダンな開発環境（ES Modules や各種フレームワーク）では、デフォルトでこのモードが有効になっています。
例えば、スペルミスで未定義の変数に代入しようとすると、昔の JavaScript は勝手にグローバル変数を作ってしまいましたが、厳格モードならエラーを投げてくれます。
「エラーは怖いものではなく、私たちを守ってくれるガードレールである」。この視点を持つと、開発がぐっと楽になりますよ。

### まとめ

- オブジェクトは「データ」と「メソッド（振る舞い）」を一つにまとめる。
- `this` を使うことで、オブジェクト内のデータを参照できる。
- `...`（スプレッド構文）を使った「非破壊的な更新」を心がける。
- JavaScript は「プロトタイプ継承」という独自の仕組みで動いている。
- ブラウザの高速化のために、オブジェクトの構造は一貫させる。
- 厳格モードを活用して、安全なコードを書く。

### 展望

オブジェクトの扱いをマスターしたら、いよいよ Web ブラウザならではの世界―― HTML 要素を自由自在に操る **「DOM 操作」** の本丸に攻め込みましょう。
