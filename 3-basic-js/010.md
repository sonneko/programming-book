# DOM操作：ブラウザ上の要素を操る

私たちが 2 章で学んだ HTML と CSS。それらは、ブラウザが表示する「静止画」のようなものです。そこに命を吹き込み、ボタンを押したら文字が変わる、スクロールしたら要素がふわっと現れる……といった「動き」を作るのが JavaScript の役割です。

そのために、JavaScript がブラウザの各要素にアクセスするための窓口が **DOM (Document Object Model)** です。

## 1. DOM は「木構造」の操作インターフェース

ブラウザは HTML を読み込むと、それをメモリ上に **「木（ツリー）」** のようなデータ構造として展開します。

```text
Document (根っこ)
 └── html
      ├── head
      └── body
           ├── h1 (枝)
           └── div (枝)
                └── p (葉っぱ)
```

JavaScript からこの「木」を操作するためのコマンドが、DOM 操作です。

```javascript
// 1. 要素を捕まえる (Selection)
const title = document.querySelector('h1');

// 2. 中身を書き換える (Manipulation)
title.textContent = 'JavaScriptで書き換えました！';

// 3. スタイルをいじる
title.style.color = 'red';
```

## 2. イベント駆動：何かが起きたら動く

DOM 操作の基本は、**「イベント」** です。「ユーザーがクリックしたとき」「キーボードを叩いたとき」といった出来事を待ち構えて、処理を実行します。

```javascript
const btn = document.querySelector('#my-button');

// クリックイベントを待ち受ける (Event Listener)
btn.addEventListener('click', () => {
  alert('ボタンが押されました！');
});
```

この `addEventListener` は、現代の Web アプリケーション開発における心臓部です。

## 3. 非同期処理：サーバーからデータを取ってくる

DOM 操作とセットでよく使われるのが、外部からデータを取ってくる処理です。
例えば、「ボタンを押したら最新のニュースを取得して画面に出す」といった処理です。

昔の JavaScript では `XMLHttpRequest` という非常に扱いづらい API を使っていましたが、現代では **`fetch` API** と **`async/await`** 構文によって、驚くほどシンプルに書けるようになりました。

```javascript
async function getNews() {
  // サーバーにデータをリクエスト (非同期処理)
  const response = await fetch('https://api.example.com/news');
  const data = await response.json();

  // 取得したデータを DOM に反映
  const list = document.querySelector('#news-list');
  data.forEach(item => {
    const li = document.createElement('li');
    li.textContent = item.title;
    list.appendChild(li);
  });
}
```

## 低レイヤーの視点：リフローとリペイントの代償

JavaScript で DOM を書き換えるのは楽しいですが、やりすぎには注意が必要です。

ブラウザが画面を 1 回描画するためには、膨大な計算が必要です。
1. **Reflow (Layout)**: 要素の大きさや位置を再計算する。
2. **Repaint**: ピクセルの色を塗り直す。

例えば、`for` ループの中で 100 回 `appendChild`（要素の追加）を行うと、最悪の場合ブラウザは 100 回リフローを繰り返します。これは非常に重い処理です。
プロは、一度「仮想の箱（DocumentFragment）」にすべての要素を入れてから、最後に 1 回だけ本番の DOM に追加する、といった工夫をします。

「JavaScript のコードが速い」ことと「画面の表示がスムーズである」ことは、全く別の問題なのです。

## 私の視点：フレームワークへの入り口

直接 DOM をいじる（Vanilla JS）のは、DOM の仕組みを学ぶには最適ですが、実際の開発でこれを何百箇所も書くのは無理があります。どこを書き換えたか、今の画面の状態がどうなっているか、管理できなくなるからです。

そこで登場するのが、**React** や **Vue** といったモダンなフレームワークです。
これらは、私たちが直接 DOM をいじる代わりに、「データの状態（State）」を管理すれば、自動で DOM を最適に更新してくれる魔法のようなツールです。
でも、その魔法の正体は、今回学んだ `document.querySelector` や `appendChild` の積み重ねに過ぎません。基礎を固めることが、魔法を使いこなす近道ですよ。

### まとめ

- DOM は HTML を木構造として扱うための API。
- `querySelector` で捕まえ、`textContent` や `style` で操作する。
- `addEventListener` でユーザーのアクションに反応する。
- `fetch` と `async/await` で、ページをリロードせずにデータを取得する。
- DOM 操作は重い処理（リフロー）を伴うため、回数を最小限にする工夫が必要。

### 展望

JavaScript でブラウザを自由に操れるようになったら、次はいよいよ「型の安全性」を手にれる旅、**TypeScript** の世界へ。動的型付けの自由奔放さに、知的な秩序（型）を与えていきましょう。
