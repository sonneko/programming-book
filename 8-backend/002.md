# express.jsを使ったバックエンド

Express.jsは、Node.jsで最も広く利用されている軽量なWebアプリケーションフレームワークです。

### シンプルなサーバー

```javascript
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World!');
});

app.listen(3000, () => console.log('Server running on port 3000'));
```

Expressは、ルーティング、ミドルウェア、リクエスト・レスポンスの処理を極めて直感的に記述できます。小規模なAPIサーバーから複雑なバックエンドまで、柔軟に構築可能です。

### 技術的な深み
Node.jsのイベントループ（libuv）を最大限に活用し、非同期I/Oをノンブロッキングで処理することで、単一スレッドでありながら高いスループットを実現します。

### 低レイヤーの現実
Expressのミドルウェアの連鎖はJSのコールスタックを消費します。深い呼び出しや複雑な処理はイベントループをブロックし、サーバー全体のレスポンスを低下させる可能性があるため、パフォーマンス最適化には注意が必要です。

### 設計思想
Expressは、抽象化によるコードの簡潔さを優先する宣言的アプローチを採用しています。これにより、ルーティングとビジネスロジックの関心事を分離し、コンポーネント指向的な開発を促進します。
