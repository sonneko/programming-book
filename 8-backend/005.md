# CORSポリシー

CORS (Cross-Origin Resource Sharing) は、異なるオリジン（ドメイン、ポート）からのブラウザ経由のアクセスを制御する仕組みです。

### なぜ必要か

ブラウザのセキュリティ上の制約として、Webページと同じドメインのサーバーにしかリクエストを送れない「同一オリジンポリシー」があります。APIサーバーを別ドメインで運用する場合、この制約によりブラウザからリクエストが拒否されます。

サーバー側で許可されたオリジンを設定することで、この制約を安全に緩和し、クロスドメイン通信を可能にします。

### 技術的な深み
CORSは、ブラウザが本リクエストの前に「プリフライトリクエスト（OPTIONSメソッド）」をサーバーへ送り、通信を許可するかどうかを問い合わせるメカニズムです。これにより、意図しないドメインからのアクセスを防ぎます。

### 低レイヤーの現実
HTTPレスポンスヘッダー（`Access-Control-Allow-Origin`等）を適切にセットしないと、ブラウザはレスポンスをJSに渡さず拒否します。ネットワークレベルではリクエストは届いていても、ブラウザのセキュリティ層がデータを捨てている点に注意が必要です。

### 設計思想
「最小権限の原則」を適用します。すべてのオリジンを許可（`*`）するのではなく、必要なドメインのみをホワイトリスト方式で許可することで、クロスサイトリクエストフォージェリ（CSRF）や攻撃者の不正アクセスを抑制します。
