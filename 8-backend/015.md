# サービスを実際に公開したい人へ

作成したAPIサーバーを公開するには、クラウドプラットフォームを利用するのが一般的です。

### 公開への道

- **Vercel / Render / Fly.io**: Node.jsアプリケーションを簡単にデプロイできるプラットフォーム。
- **Docker**: アプリをコンテナ化し、環境依存を排除する。
- **ドメインとSSL**: 独自ドメインを取得し、HTTPSで安全に公開する。

公開環境では、ローカル環境とは異なる設定（環境変数、セキュリティ設定）が必要になるため、段階的に学習していきましょう。

### 知識の拡充

- **技術的深み**: Dockerコンテナ化によるイミュータブル・インフラストラクチャは、環境差異によるバグを排除し、再現性の高いデプロイを実現します。
- **低レイヤーの現実**: TLS/SSLによる暗号化通信は、CPU負荷を増大させますが、現代のネットワーク環境において必須のセキュリティ対策です。
- **設計思想**: 運用の自動化（CI/CD）と監視を取り入れることで、サービスを継続的に改善・拡張できるエンジニアリング体制を築くことです。
