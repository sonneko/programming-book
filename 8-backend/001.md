# バックエンドとは

バックエンド（サーバーサイド）は、フロントエンドが送信したリクエストを受け取り、処理し、結果を返すシステムです。Webアプリケーションにおける「データ」の永続化と、計算処理の大部分を担います。

### ステートフルとステートレス
Webの基盤であるHTTPプロトコルは本来「ステートレス」です。つまり、リクエストごとに互いに独立しており、前のリクエストを覚えることができません。
しかし、Webアプリケーションには「ログイン中かどうか」という状態が必要です。この矛盾を解決するために、CookieやJWT（JSON Web Token）を使って「セッション」を管理します。

### データベースの本質
バックエンド開発において、データベースはアプリケーションの「記憶」です。SQLは、データを操作するための数学的な言語（関係代数）に基づいています。単なる保存場所としてではなく、データ間の整合性を保証するための強力なメカニズムとして機能します。

### 技術的な深み
リクエストはHTTP/1.1やHTTP/2といったプロトコル上をTCPソケットで流れます。サーバー側ではリクエストを処理するためにスレッドや非同期I/O（イベントループ）を利用し、高並行処理を実現しています。

### 低レイヤーの現実
TCPコネクションはOSレベルのリソースであり、適切にクローズされないとファイルディスクリプタを枯渇させる原因となります。Node.jsやGoのサーバーは、このリソース管理を抽象化して提供していますが、負荷試験でのチューニングは必須です。

### 設計思想
バックエンドは「ステートレス」に設計することで、水平スケーラビリティが確保されます。状態を分離（Redis等の外部メモリ管理）することで、サーバーの増減に柔軟に対応できます。
