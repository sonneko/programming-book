# OOPでも関数型！

ここまで「純粋関数」や「Haskell」、「カリー化」といった、少しアカデミックで、ある種「浮世離れ」した話をしてきました。もしかしたら皆さんは、「自分は仕事や趣味でJavaScriptやPython、Javaを書いているから、こういうのは関係ないかな」と思っているかもしれません。

でも、ちょっと待ってください。実は、現代のプログラミングにおける最大のトレンドは、**「オブジェクト指向（OOP）の中に、関数型のエッセンスをどんどん取り入れること」** なのです。

## ツールとしての関数型

今、あなたが使っている言語を思い出してみてください。

- **JavaScript / TypeScript**: `map`, `filter`, `reduce` を使っていませんか？ アロー関数 `() => {}` は？ これらはすべて関数型の機能です。
- **Python**: `lambda` 式やリスト内包表記、`itertools` など、関数型由来のツールが満載です。
- **Java**: Java 8から「Lambda」と「Stream API」が導入され、設計の仕方がガラリと変わりました。
- **Rust**: そもそも関数型言語の影響を強く受けて設計されたシステムプログラミング言語です。

これらの言語を「オブジェクト指向言語」と呼ぶのは、今や半分正解で半分間違いです。今の主流は、両方の良いとこ取りをする「マルチパラダイム」なスタイルなのです。

## 今日から実践できる「関数型」の知恵

特定の言語を使わなくても、私たちがコードを書くときに意識できる「関数型的なアプローチ」はたくさんあります。

### 1. ループ（for/while）を「高階関数」に置き換える

「配列の要素を2倍にする」という処理を、泥臭い `for` 文で書くのをやめてみましょう。

```javascript
// Before: 手続き型
const doubled = [];
for (let i = 0; i < numbers.length; i++) {
  doubled.push(numbers[i] * 2);
}

// After: 関数型（データ変換）
const doubled = numbers.map(n => n * 2);
```

「何をしているか（2倍にしてマップする）」が直接表現されている `map` の方が、意図が伝わりやすく、変数 `i` の管理ミスによるバグも防げます。

### 2. 不変（Immutable）であることをデフォルトにする

変数を作るとき、安易に `let` (JS) や `var` (Java/Python) を使わず、可能な限り `const` や `final` を使いましょう。

一度決めた値を変えないという制約を自分に課すだけで、プログラムの「状態」を追いかける苦労から解放されます。

### 3. ロジックを「純粋」に保ち、副作用を端に追いやる

プログラム全体を純粋にするのは難しいですが、「計算ロジック」と「外部とのやり取り（I/O）」を分けることはできます。

- ユーザーの入力を受け取る（不純）
- **入力を元に、結果を計算する（純粋）**
- 結果を画面に表示する（不純）

この真ん中の「計算」の部分を純粋な関数として切り出すだけで、その部分はテストが非常に楽になり、バグが入り込む隙がなくなります。

## 低レイヤーの視点：抽象化のコストとJITコンパイラ

「高階関数（mapなど）を使うと、普通のループより遅くなるんじゃない？」という心配をされる方もいるでしょう。確かに、関数を一つずつ呼び出すオーバーヘッドはゼロではありません。

しかし、V8（JavaScriptエンジン）やJVM（Java仮想マシン）に搭載されている **JIT（Just-In-Time）コンパイラ** は、非常に賢いです。

## 私の視点：境界線はどこにある？

「オブジェクト指向か、関数型か」という対立軸で考えるのは、もう古いのかもしれません。

大切なのは、「状態をどう管理するか」という課題に対して、その時々で最適な道具を選ぶことです。複雑なデータ構造を整理するときはオブジェクト指向の「カプセル化」が役に立ち、複雑なデータ変換を記述するときは関数型の「パイプライン」が役に立ちます。

私が目指しているのは、特定の教義（ドグマ）に縛られるのではなく、あらゆるパラダイムから知恵を盗み、最高のコードを書く「ハイブリッドなエンジニア」です。皆さんも、この本で学んだ関数型の視点を、ぜひ普段のプログラミングにこっそり忍び込ませてみてください。

### まとめ

- 現代の主要言語は、すべて関数型の機能を備えている。
- `map` / `filter` の活用、不変性の意識、副作用の分離が第一歩。
- パフォーマンスへの影響は、現代のコンパイラがかなりの部分をカバーしてくれる。

### 展望

これで「関数型言語」のセクションは終了です。次は、さらに具体的な技術の裏側――APIの使い方や、サーバーの選び方など、より実践的な「付録」の世界へ進んでいきましょう！
