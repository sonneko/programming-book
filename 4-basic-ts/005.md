# 使ってはいけない集：型の海に漂うリスク

TypeScript は、JavaScript という荒波に「型」という堤防を築く技術です。しかし、その堤防には、意図的に穴を開けることができる「禁断の道具」がいくつか存在します。

これらは、一時的には便利かもしれませんが、長期的にはあなたのプログラムを腐らせ、予期せぬエラー（ランタイムエラー）を引き起こす原因になります。
ここでは、TypeScript を書く上で「極力避けるべきこと」と、その理由をエンジニアの視点でお話しします。

## 1. 禁断の果実：`any` 型

TypeScript における最大の罪、それが **`any` 型** の乱用です。

```typescript
// すべての型チェックを放棄する「死の宣言」
let data: any = JSON.parse(input);
data.methodDoesNotExist(); // コンパイルは通るが、実行時にクラッシュする
```

`any` を使った瞬間、その変数は TypeScript の加護（型チェック）を失い、ただの JavaScript に戻ってしまいます。
「型定義を書くのが面倒だから」「外部ライブラリの型がわからないから」といった理由で `any` を使うのは、堤防を自ら壊して津波（バグ）を招き入れるようなものです。

**対策**: どうしても型がわからないときは、代わりに **`unknown`** 型を使いましょう。`unknown` は「何か入っているけどわからない」という型なので、使う前に必ず「これは文字列か？」といった型チェックを強制してくれます。

## 2. 時代遅れの遺物：`var` キーワード

JavaScript 入門で学んだ通り、`var` は絶対に使ってはいけません。

```typescript
if (true) {
  var x = 10;
}
console.log(x); // 10 が表示される。ブロックの外なのに！
```

`var` は **「関数スコープ」** であり、さらに **「巻き上げ（Hoisting）」** という非常に直感に反する挙動をします。
現代の V8 エンジンにおいて、`const` や `let` は非常に効率的に最適化されており、バグも防げます。

**対策**: 常に **`const`** をデフォルトにし、どうしても再代入が必要な場合のみ **`let`** を使う。これが現代の鉄則です。

## 3. 嘘の証言：型アサーション (`as`) の乱用

「本当は数値なんだけど、コンパイラが文字列だと思い込んでいるから、無理やり数値だと言い張る」。これが **型アサーション** です。

```typescript
const value = "123" as unknown as number; // 危険！
```

コンパイラは「あなたがそう言うなら信じるよ……」とエラーを引っ込めますが、実行時のデータが本当にその型である保証はどこにもありません。
これはコンパイラに対する「嘘」であり、後で必ずしっぺ返しを食らいます。

**対策**: `as` で嘘をつくのではなく、**「型ガード (Type Guard)」** を使って、実行時に正しい型であることを証明しましょう。

## 低レイヤーの視点：デ最適化の恐怖

JavaScript 実行エンジン（V8 など）にとって、型が頻繁に変わるコードは「最適化の敵」です。

もしあなたが `any` を多用して、ある時は数値、ある時はオブジェクト……といった具合に変数の型をコロコロ変えると、エンジンは「この変数は何が来るか予測不能だ」と判断し、高速な実行モードを諦めて低速な汎用モードに切り替えます（これを **デ最適化** と呼びます）。

型を厳格に保つことは、単にバグを防ぐだけでなく、ハードウェアの性能を 100% 引き出すための「協力」でもあるのです。

## 私の視点：自分自身を疑え

「このコードは絶対にバグらないはずだ」という自信があるときほど、人間は `any` や `as` を使いたくなります。
でも、忘れないでください。人間は間違える生き物です。数ヶ月後のあなた、あるいはあなたのコードを引き継ぐ同僚は、あなたが「絶対に大丈夫」だと思った前提を忘れてしまうかもしれません。

TypeScript コンパイラという「絶対に嘘をつかない、世界一厳格なレビュー担当者」を味方につける。そのために、あえて自由を捨て、型という制約の中に身を置く。それが、プロのエンジニアの矜持（プライド）です。

### まとめ

- **`any` 型** は TypeScript の存在意義を否定する。代わりに `unknown` を。
- **`var`** は挙動が予測不能。常に `const` と `let` を。
- **型アサーション (`as`)** はコンパイラへの嘘。ランタイムエラーの温床。
- 型の一貫性は、ブラウザの実行速度向上にも貢献する。
- リンター（ESLint）を導入して、これらの「悪習」を自動で禁止しよう。

### 展望

「やってはいけないこと」を学んだら、次は正しく強力な道具の使い方を学びましょう。TypeScript の表現力を一気に広げる「クラス」の表記法について詳しく見ていきます。
