# AmazonWebService

世界で最も利用されているクラウドサービス、それが **Amazon Web Services (AWS)** です。元々は Amazon.com の社内インフラとして構築されたものが外販されるようになり、今や世界のインターネットインフラの大部分を支える巨大帝国となりました。

「エンジニアとして就職したい」と考えているなら、AWS の知識はもはや必須と言っても過言ではありません。世の中の企業の多くが AWS 上でシステムを構築しているからです。

## AWS の思想：Everything is a Service
AWS の最大の特徴は、その圧倒的なサービスの多さです。コンピュート（計算）、ストレージ、データベース、ネットワーク、機械学習、ゲーム開発、衛星通信に至るまで、200 を超えるサービスが提供されています。

これらをレゴブロックのように組み合わせて、一つの巨大なシステムを作り上げるのが「ソリューションアーキテクト」と呼ばれるエンジニアの仕事です。

## 初心者が必ず通る「三種の神器」
あまりに多機能な AWS ですが、まずはこの 3 つを押さえましょう。

1. **Amazon EC2 (Elastic Compute Cloud)**: クラウド上の仮想サーバー。Google Cloud の GCE と同じく、自由な Linux 環境が手に入ります。
2. **Amazon S3 (Simple Storage Service)**: 非常に高い耐久性（99.999999999% = 11ナイン）を持つデータストレージ。「インターネットに公開する画像や動画」の保存先として、世界標準となっています。
3. **AWS Lambda**: コードを実行するだけで動くサーバーレスサービス。Cloud Run と似ていますが、より「関数（処理）を一つだけ動かす」という目的に特化しています。

## AWS が選ばれる理由
1. **業界標準（デファクトスタンダード）**: ユーザー数が多いため、困ったときにインターネットで検索すれば必ず解決策が見つかります。また、AWS のスキルはそのままエンジニアとしての市場価値に直結します。
2. **信頼性と実績**: 金融機関から政府機関まで、極めて高い信頼性が求められる組織が AWS を採用しています。
3. **進化の速さ**: 毎年、年次カンファレンス「re:Invent」で驚くような新機能が大量に発表されます。常に技術の最先端に触れ続けることができます。

## AWS を学ぶ上での注意点
Google Cloud 同様、AWS も初心者には少し厳しい面があります。

- **「マネジメントコンソール」の迷宮**: 管理画面が非常に多機能で、目的のメニューを探すだけでも一苦労します。
- **セキュリティ（IAM）の厳格さ**: AWS はデフォルトで「すべてを拒否」する設定になっています。適切に権限を与えないと、自分のプログラムから自分のストレージにアクセスすることすらできません。
- **課金の恐怖**: 従量課金制です。初心者がやりがちなのは、「実験で立ち上げた強力なデータベースを消し忘れて、数万円の請求が来る」というミスです。**AWS 予算（Budgets）** の設定は、アカウントを作ったら最初に行う儀式だと思ってください。

## 12ヶ月の無料利用枠
AWS には、アカウント作成から 12 ヶ月間適用される「無料利用枠」があります。EC2 の小型インスタンスや S3 の一定容量、Lambda の 100 万リクエスト分などが無料で利用できます。

「まずは AWS に触れてみる」という経験は、あなたのエンジニアとしての視座を大きく広げてくれます。普段使っているスマホアプリのアイコン画像が、実は裏側で S3 から配信されていることに気づく。そんな「裏側の繋がり」が見えてくるようになれば、あなたはもう初心者卒業です。

## まとめ
AWS は、プログラマにとっての「究極の遊び場」であり、プロのエンジニアにとっての「必須の道具箱」です。

規模が大きすぎて最初は足がすくむかもしれませんが、まずは S3 にファイルをアップロードしてみる、Lambda で「Hello World」を出力してみる、といった小さな成功体験を積み重ねていきましょう。その先には、世界中のユーザーにサービスを届けるための、無限の道が続いています。
