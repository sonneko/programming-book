# マークダウン言語

この本の原稿は、すべて **「Markdown（マークダウン）」** という言語で書かれています。
マークアップ言語（HTML など）が「コンピュータのための記述」であるのに対し、マークダウンは **「人間が書きやすく、読みやすく、かつ構造も守れる」** という、プログラマの理想を形にしたような言語です。

プログラマが日常的に書くドキュメント、メモ、ブログ、README ファイル... そのほとんどすべてに、今はマークダウンが使われています。私がこの本を執筆する際も、マークダウンのおかげで「文章の内容」に 100% 集中することができました。

## マークダウンの哲学：書きやすさこそ正義
HTML で「見出し」を書くには、`<h1>見出し</h1>` と書く必要があります。
これを一日中書いていると、指が疲れてしまいますし、閉じタグ（`</h1>`）を忘れてレイアウトが崩れるといったミスも起きます。マークダウンならこうです。

```markdown
# 見出し
```

これだけで、HTML の `<h1>` に相当する意味を持ちます。
「読みやすさ（Readability）」を最優先に設計されており、たとえ HTML に変換しなくても、生のテキストのままでも十分に意味が通じます。メールやチャットで、自然に `#` や `*` を使って箇条書きを書くような感覚で、構造化された文書が出来上がります。

## 基本的な文法：これだけ覚えれば十分
これだけ覚えれば、あなたも今日からマークダウンマスターです。

- **見出し**: `#` を頭につける（`#` の数が増えるほど小さい見出しになる）。
- **強調**: `**太字**` や `*斜体*`。
- **リスト**: `-` や `*` を頭につける。
- **リンク**: `[テキスト](URL)`。
- **画像**: `![代替テキスト](画像のURL)`。
- **コードブロック**: バッククォート 3 つ（ \`\`\` ）で囲む。これがプログラマには最も重宝されます。

```markdown
# 私の好きなもの
1. **プログラミング**
2. 読書
3. [GitHub](https://github.com) を眺めること
```

## なぜプログラマはマークダウンを愛するのか？
1. **マウスがいらない**: Word のように、文字を太字にするためにマウスで範囲選択してボタンをクリックする必要がありません。キーボードから手を離さずに、構造化された文章が書けます。これは、コーディングと同じリズムで文章が書けることを意味します。
2. **バージョン管理がしやすい**: ただのテキストファイルなので、Git で「どこを修正したか」を一行単位で管理できます。複数人でドキュメントを共同編集する際にも、衝突（コンフリクト）を簡単に解決できます。
3. **どこでも動く**: GitHub, Slack, Notion, Discord, VSCode, Obsidian... 現代のあらゆるエンジニアリングツールがマークダウンを標準サポートしています。一度覚えたスキルが、すべてのツールで使えます。
4. **美しい変換**: この本のように、マークダウンから HTML や PDF, 電子書籍（EPUB）へと簡単に美しく変換（レンダリング）できます。

## 軽量マークアップ言語の仲間
マークダウンは「軽量マークアップ言語」というジャンルに属します。他にも以下のような仲間がいます。
- **AsciiDoc**: より大規模で複雑な本の執筆などに適しており、非常に多機能です。
- **Re:VIEW**: 日本の技術系同人誌などでよく使われる、電子書籍（EPUB）制作に特化した言語です。
- **CommonMark**: マークダウンの方言を統一しようとする、標準化のプロジェクトです。

## 学習のアドバイス
マークダウンは、学ぶというより「慣れる」ものです。
まずは自分のメモ帳をマークダウン形式（拡張子 `.md`）にしてみることから始めてください。VSCode などのエディタを使えば、左側に編集画面、右側にプレビュー画面を表示しながら楽しく書くことができます。

また、GitHub で自分のプロジェクトを作ったら、必ず `README.md` を書いてみてください。そこにプロジェクトの概要や使い方を美しくまとめることは、他の開発者への最高の「おもてなし」になります。

## まとめ
マークダウンは、思考を妨げない言語です。
頭の中にあるアイデアを、一番速く、一番自然な形でアウトプットするための道具。それがマークダウンです。

この本を読み終え、あなたが自分のプロジェクトを GitHub に公開するとき、その顔となる `README.md` を最高にかっこいいマークダウンで書き上げてください。その時、あなたは真の意味でプログラマのコミュニティの一員になり、言葉とコードの両方で世界と繋がることができるのです。
