# マークアップ言語

これまで紹介してきたプログラミング言語は、コンピュータに「何かをさせる（命令や計算）」ためのものでした。
しかし、世の中には **「情報の構造を記述する」** ための言語があります。それが **「マークアップ言語（Markup Languages）」** です。

マークアップとは、文章の特定の部分を **「タグ」** と呼ばれる記号で囲む（マークをつける）ことで、「ここからここまでは見出しですよ」「ここはリンクですよ」といった意味（セマンティクス）を付与する行為を指します。

## 1. HTML (HyperText Markup Language)
Web の世界で最も有名なマークアップ言語です。

### 特徴：Web の骨組みを作る
2 章で詳しく学びましたが、あなたが今見ているこのページも、すべて HTML で書かれています。

```html
<!DOCTYPE html>
<html>
<head>
    <title>サンプルページ</title>
</head>
<body>
    <h1>これは大見出しです</h1>
    <p>これは段落（本文）です。</p>
    <a href="https://example.com">リンクはこちら</a>
</body>
</html>
```
このように、テキストを `<タグ名>` と `</タグ名>` で囲んでいきます。

## 2. XML (eXtensible Markup Language)
HTML をより一般的にし、誰でも独自のタグを作れるようにした言語です。

### 特徴：データの交換と保存
XML は、人間が見るためというよりは、**「コンピュータ同士でデータをやり取りする」** ために使われます。

```xml
<user>
    <id>12345</id>
    <name>Sonneko</name>
    <email>example@example.com</email>
</user>
```
このように、データの意味をタグで明確にできるため、設定ファイルやデータベースのバックアップなどで多用されてきました。

## 3. SVG (Scalable Vector Graphics)
「画像」をマークアップで記述するための言語です。

### 特徴：数式で描かれた画像
JPG や PNG といった画像が「色のついた点の集まり（ビットマップ）」であるのに対し、SVG は「ここからここへ線を引く」といった命令の集まり（ベクター）です。そのため、どれだけ拡大しても画像がボケないという素晴らしい特徴があります。

```svg
<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>
```
これも XML の一種であり、テキストエディタで画像を「書く」ことができるのです。

## プログラミング言語との違い
マークアップ言語には、基本的に「ループ」や「条件分岐」といった計算機能はありません。
（※ 最近の HTML5 や CSS3 には一部計算に近い機能もありますが、本質的には異なります。）

そのため、「マークアップ言語はプログラミング言語ではない」と厳格に区別されることもあります。しかし、現代のソフトウェア開発において、情報を正しく構造化するスキルは、アルゴリズムを書くスキルと同じくらい重要です。

## 構造化データの重要性
なぜタグをつけて囲む必要があるのでしょうか？
それは、コンピュータには「ただのテキスト」の羅列では、どこが重要で、どこが何を表しているのかを判断できないからです。

タグをつけることで、検索エンジン（Google など）はあなたのサイトの内容を正しく理解し、読み上げソフトは目の不自由な方に適切な順番で文章を伝えることができます。

## まとめ
マークアップ言語は、カオスな情報の海に「意味」という秩序を与えるための道具です。
プログラミング言語のような派手なロジックはありませんが、そのシンプルさと美しさは、Web という広大な宇宙を支える、目に見えない土台になっています。

2 章での HTML の学習を通じて、情報の「形」を整える楽しさをぜひ味わってください。
