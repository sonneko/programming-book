# GitHubAPI

プログラマにとっての聖地、**GitHub** もまた、非常に強力な API を提供しています。**GitHub API** を使いこなすことは、単にリポジトリを操作するだけでなく、開発ワークフローそのものをプログラミングで自動化することを意味します。

あなたが GitHub 上で行っている「リポジトリの作成」「コードのコミット」「Issue（課題）の作成」「Pull Request（プルリクエスト）の承認」などは、すべて API 経由で実行可能です。

## REST API と GraphQL API
GitHub は、伝統的な **REST API**（v3）と、よりモダンな **GraphQL API**（v4）の 2 種類を提供しています。

- **REST API**: 特定の URL（エンドポイント）に対してアクセスし、定義されたデータを取得します。使い方がシンプルで、ドキュメントも非常に充実しています。
- **GraphQL API**: 1 つのエンドポイントに対して「自分が必要なデータ」をクエリ言語で指定して取得します。複数のリソースを一度に取得したい場合や、データの取得量を最小限に抑えたい場合に非常に強力です。

最初は REST API から使い始め、複雑なデータの取得が必要になったら GraphQL に挑戦してみるのが良いでしょう。

## 認証：Personal Access Token (PAT)
GitHub API を叩くには、自分が誰であるかを証明する **Personal Access Token (PAT)** が必要です。

1. GitHub の Settings > Developer settings > Personal access tokens から発行します。
2. そのトークンに「どの操作を許可するか（Scopes）」を設定します。例えば、リポジトリの読み書きだけなら `repo` スコープが必要です。
3. 発行されたトークンは、HTTP リクエストの `Authorization` ヘッダーに含めて使用します。

```bash
# curl を使ったリポジトリ情報の取得例
curl -H "Authorization: token YOUR_GITHUB_TOKEN" \
     https://api.github.com/user/repos
```

最近では、よりセキュリティが高い **Fine-grained personal access tokens** という、リポジトリ単位で詳細な権限設定ができるトークン形式が推奨されています。

## GitHub API でできる面白いこと
- **自動 Issue 作成**: ソースコード内の `TODO` コメントをスキャンして、自動的に GitHub の Issue として登録する。
- **リポジトリの統計分析**: 誰がどれくらいコードを書き、どの時間に活発に活動しているかを可視化する。
- **自動リリース**: 特定のタグが打たれたら、自動的にリリースノートを作成し、ビルドされたバイナリをアップロードする。
- **プロジェクト管理の自動化**: 新しく Issue が作られたら、自動的に特定のラベルを付与したり、担当者を割り当てたりする。

## Webhook との組み合わせ
LINE Messaging API の章でも触れましたが、GitHub にも **Webhook** の仕組みがあります。
「誰かがスターを押した」「プルリクエストが作成された」「コードがプッシュされた」といったイベントが発生した瞬間に、あなたのサーバーに通知を送らせることができます。

これを使えば、「誰かがリポジトリにスターを押してくれたら、自分の LINE に通知が届く」といった連携が簡単に作れます。自分の作ったツールに反応があるのは、開発者にとって最高にモチベーションが上がる瞬間ですよね。

## Rate Limit（回数制限）に注意
GitHub API には、トークンごとに 1 時間あたりのリクエスト上限があります。
- 認証済みリクエスト：1 時間あたり 5,000 リクエスト
- 未認証リクエスト：1 時間あたり 60 リクエスト

大規模なスクリプトを回す場合は、この上限に達しないように注意が必要です。リクエストの結果に含まれる `X-RateLimit-Remaining` ヘッダーを確認することで、あと何回リクエストが可能かを知ることができます。

## GitHub CLI (gh) の活用
API を直接叩くのも良いですが、GitHub 公式が提供しているコマンドラインツール **gh** も非常に便利です。内部的には GitHub API を呼び出していますが、私たちが使いやすいように高度に抽象化されています。

```bash
# コマンド一つでプルリクエストの一覧を表示
gh pr list
```

シェルスクリプトから GitHub を操作したい場合は、API を生で叩くよりも `gh` コマンドを使うほうがはるかに楽なことが多いです。

## まとめ
GitHub API は、単なるツールではなく、あなたの「開発体験（Developer Experience）」を自らの手でカスタマイズするための魔法の杖です。日々の開発の中で「あ、これ面倒だな」と思う作業があれば、まずは GitHub API のリファレンスを覗いてみてください。そこにはきっと、あなたの開発を加速させるヒントが隠されているはずです。
