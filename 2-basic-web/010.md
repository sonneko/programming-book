# グリッドシステム

Web サイトのレイアウトを組むことは、かつてはエンジニアにとって「暗黒時代」のような苦行でした。
10 年以上前は、透明な画像を使って隙間を作ったり、本来は表（データ）のための `<table>` タグを無理やりレイアウトに使ったり、画像を回り込ませるための `float` を駆使して「回り込み解除」という謎の呪文を書いたりしていました。

しかし、現代の私たちには **Flexbox** と **CSS Grid** という、最強の武器があります。これらを使えば、かつて数時間かかっていたレイアウトが、わずか数行で、しかも美しく完成します。

## 1. Flexbox：1 次元のレイアウト

要素を **「一列（横、または縦）」** に並べるのに最も適した道具です。

```css
.container {
  display: flex;
  /* 横に並べる */
  flex-direction: row;
  /* 要素を両端に寄せる */
  justify-content: space-between;
  /* 上下の中央に揃える */
  align-items: center;
}
```

「中央揃え（センタリング）」は、昔の Web 制作では世界で最も難しい操作の一つとされていましたが、Flexbox の `justify-content: center` と `align-items: center` を組み合わせるだけで、完璧に解決します。

## 2. CSS Grid：2 次元のレイアウト

要素を **「格子状（行と列）」** に並べるための道具です。Flexbox が「線」だとしたら、Grid は「面」を操ります。

```css
.container {
  display: grid;
  /* 3列のグリッドを作る（比率は 1:2:1） */
  grid-template-columns: 1fr 2fr 1fr;
  /* 要素間の隙間 */
  gap: 20px;
}
```

「この要素は 2 マス分使う」といった指定も簡単に行えるため、複雑なニュースサイトのトップページや、タイル状のギャラリーサイトなどを作るのに最適です。

## 3. レスポンシブデザインの救世主

グリッドシステムが真価を発揮するのは、**「PC では横に 3 つ並べたいけど、スマホでは縦に 1 つずつ並べたい」** という、画面サイズに応じたレイアウト変更をするときです。

```css
.card-list {
  display: grid;
  /* 画面幅に合わせて、最小 300px のカードを並べられるだけ並べる */
  grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
  gap: 16px;
}
```

このような「魔法の 1 行」を書くだけで、ブラウザが画面幅を自動で判断し、要素を折り返したり広げたりしてくれます。

## 低レイヤーの視点：計算の抽象化

なぜ `float` は難しくて、`flex` や `grid` は簡単なのでしょうか？

それは、**「ブラウザが計算を肩代わりしてくれているから」** です。
`float` を使っていた時代は、「要素 A の幅は 30%、余白は 5px だから、要素 B は……」と、人間がピクセル単位で緻密に計算して指示を出す必要がありました。

Flexbox や Grid は、人間が **「どう並べたいか（意図）」** を伝えると、ブラウザ内部のレンダリングエンジンが、画面の残り幅や要素の中身の大きさを考慮して、リアルタイムで最適なピクセル位置を逆算（*Solve Constraints*）してくれます。
私たちは「物理的な座標」ではなく「論理的な関係」を記述するようになったのです。

## 私の視点：Flexbox と Grid の使い分け

私はいつも、**「基本は Flexbox、構造は Grid」** というルールで使い分けています。

- ヘッダーのロゴとメニューを並べる、ボタンの中のアイコンと文字を並べる……といった「一方向の流れ」には Flexbox。
- ページ全体の大きな枠組みや、カレンダーのような「マスの目」には Grid。

最初は Flexbox だけでも十分戦えますが、Grid を覚えると「CSS でできないことはない」という全能感に包まれますよ。

### まとめ

- **Flexbox**: 1 次元（列）の配置に強い。中央揃えが簡単。
- **CSS Grid**: 2 次元（面）の配置に強い。複雑な分割が得意。
- 現代のレイアウトは、ブラウザに「計算を任せる」スタイル。
- `1fr` などの単位を使いこなして、柔軟な（レスポンシブな）設計を。
- まずは Flexbox をマスターし、必要に応じて Grid を取り入れよう。

### 展望

レイアウトの最強の武器を手に入れたところで、次はブラウザがどのようにしてこの複雑な計算を行い、画面を描き出しているのかという「レンダリングパイプライン」の深層に迫ります。これがわかると、脱・初心者です。
