# ブロック要素とインライン要素

CSS でデザインを始めると、必ずと言っていいほど「なぜこの要素は横に並ばないの？」「なぜこの要素には幅が指定できないの？」という壁にぶつかります。その謎を解く鍵が、すべての HTML 要素が持っている **「表示形式 (Display Type)」** という性質です。

古くからの Web のルールですが、これを理解しないとレイアウトを思い通りに操ることはできません。

## 1. ブロック要素 (display: block)

画面の **「かたまり（ブロック）」** として振る舞う要素です。

- **特徴**:
  - 何も指定しなくても、横幅いっぱいに広がります。
  - 前後に自動的に改行が入ります（次の要素は必ず下の行に行きます）。
  - 幅 (`width`) や高さ (`height`) を自由に指定できます。
- **代表的なタグ**:
  - `<div>`, `<h1>` 〜 `<h6>`, `<p>`, `<ul>`, `<li>`, `<header>`, `<main>`, `<footer>`

「箱を積み上げていく」イメージでレイアウトを組むときに使います。

## 2. インライン要素 (display: inline)

文章の **「中（インライン）」** に流れる要素です。

- **特徴**:
  - 中身のテキストや画像の分だけしか幅を取りません。
  - 横に空きがあれば、次の要素と横並びになります。
  - **幅 (`width`) や高さ (`height`) の指定が無視されます。**
  - 上下のマージン (`margin-top`, `margin-bottom`) が効きません。
- **代表的なタグ**:
  - `<span>`, `<a>`, `<strong>`, `<em>`

「文章の一部を装飾する」ための要素です。

## 性質は CSS で「上書き」できる

「`span` はインラインだから横にしか並ばない」と諦める必要はありません。CSS の **`display`** プロパティを使えば、その性質を自由に変更できます。

```css
span {
  /* span なのにブロック要素のように振る舞わせる */
  display: block;
  width: 100px;
  height: 100px;
  background: red;
}
```

また、両方の良いとこ取りをした **`display: inline-block`** という指定もよく使われます。「横に並ぶけれど、幅や高さも指定できる」という便利な性質です。

初期の Web は、大学の論文などの「文書」を共有するためのシステムでした。
論文には、段落（ブロック）があり、その中の特定の単語（インライン）を強調したりリンクにしたりする必要がありました。つまり、**「文書をコンピュータが正しく構造化して読み取るため」** の自然な区別だったのです。

現代の Web サイトは、もはや「文書」ではなく「アプリケーション」に近い見た目をしていますが、ブラウザのレンダリングエンジン（Blink など）の根底には、依然としてこの「テキストを流し込む（*Line Breaking*）」というロジックが深く刻まれています。

## 私の視点：CSS は「上書き」の歴史

最近の Web 制作では、`display: flex`（後述）の登場によって、ブロックとインラインの厳格な区別を意識する機会は少し減りました。
しかし、「なぜかデザインが崩れる」という原因の 8 割は、インライン要素に幅を指定しようとしていたり、意図しない改行（ブロック要素）が入っていたりすることにあります。

「このタグのデフォルトの display は何か？」を常に意識できるようになると、CSS のトラブルシューティング能力は飛躍的に向上しますよ。

### まとめ

- **ブロック要素**: 縦に積み重なる。幅・高さを指定できる。
- **インライン要素**: 横に流れる。幅・高さ・上下マージンが効かない。
- `display` プロパティで、タグ本来の性質を上書きできる。
- もともと Web は「論文」のための仕組みだったので、この区別が生まれた。
- 崩れたときはまず `display` の値を確認しよう。

### 展望

ブロックとインラインという「古典的な」並び方を理解したら、いよいよ現代の Web 制作における最強の武器、**「Flexbox」** と **「グリッドシステム」** に進みましょう。これを知れば、どんな複雑なレイアウトも自由自在です。
