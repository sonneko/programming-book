# よく使うhtmlタグ

HTML には、実は 100 種類以上のタグが存在します。全部覚える必要があるかというと、答えは **「NO」** です。プロのエンジニアでも、日常的に使うのはそのうちの 20 種類程度です。

大切なのは、たくさん覚えることではなく、**「それぞれのタグが持つ『意味』を正しく使い分けること」** です。ここでは、実際の開発現場で「これさえあればサイトが作れる」という主要なタグを厳選して紹介します。

## 1. テキストを形作るタグ

もっとも基本となる、文章のためのタグです。

- **`<h1>` 〜 `<h6>` (Heading)**: 見出しです。`h1` が最も大きく（重要）、`h6` に向かって小さくなります。
  - ルール：`h1` は 1 ページに原則 1 つだけにしましょう。見出しのレベルを飛ばして（`h1` の次にいきなり `h3` を使うなど）はいけません。
- **`<p>` (Paragraph)**: 段落です。文章のひとかたまりを囲みます。
- **`<a>` (Anchor)**: リンクです。`href` 属性で移動先の URL を指定します。
- **`<span>`**: 特別な意味を持たない「インライン」の箱です。文章の一部分だけに色をつけたいときなどに使います。

## 2. 構造を作り、グループ化するタグ

現代の Web 制作で最も重要なのが、要素をまとめるためのタグです。

- **`<div>` (Division)**: 特に意味を持たない「ブロック」の箱です。CSS でレイアウトを組むための「入れ物」として、最も多用されます。
- **`<ul>` (Unordered List) と `<li>` (List Item)**: 箇条書き（点付きリスト）です。ナビゲーションメニューなどにもよく使われます。
- **`<header>`, `<main>`, `<footer>`**: サイトの大きな区切りを表す、意味を持ったタグです。

## 3. メディアとインタラクション

- **`<img>` (Image)**: 画像を表示します。
  - `src`: 画像の場所。
  - `alt`: 画像が表示されなかったときに出る説明文。**アクセシビリティ（目の不自由な方への読み上げ）のために必須**です。
- **`<button>`**: クリックできるボタンです。

## セマンティック HTML：なぜタグ選びが重要なのか？

昔の Web サイトは、何でもかんでも `<div>` で作られていました。これを **「div 地獄 (div soup)」** と呼びます。

```html
<!-- 悪い例：全部 div なので、どこが重要か分からない -->
<div class="header">私のサイト</div>
<div class="content">こんにちは</div>
<div class="footer">Copyright 2024</div>
```

```html
<!-- 良い例：セマンティック（意味的）な記述 -->
<header>私のサイト</header>
<main>こんにちは</main>
<footer>Copyright 2024</footer>
</footer>
```

このように、タグ本来の意味を考えて使い分けることを **セマンティック HTML** と呼びます。なぜこれが必要なのでしょうか？

1. **SEO (検索エンジン最適化)**: Google などの検索エンジンは、HTML の構造を見て「どこが重要な内容か」を判断します。正しいタグを使うと、検索結果で上位に表示されやすくなります。
2. **アクセシビリティ**: 画面読み上げソフト（スクリーンリーダー）を使っているユーザーにとって、正しいタグ構成は「情報の地図」になります。
3. **開発効率**: 誰が見てもどこがヘッダーでどこがメインコンテンツか一目でわかるため、チーム開発でのミスが減ります。

しかし、例えば `<table>` タグを使って複雑なレイアウトを組むと、ブラウザはすべてのセルの中身が決まるまで全体の幅を確定できず、レンダリングが遅くなる（再レイアウトが重くなる）という性質があります。
逆に `<div>` や `セマンティックなタグ` を使ったシンプルな構成は、ブラウザにとって非常に計算しやすく、サクサク動くサイトに繋がります。

## 私の視点：「迷ったら div」でいい

最初は「どのタグを使うのが正解だろう……」と悩みすぎて手が止まってしまうことがあります。
そんな時は、**「まずは div で囲って、後から適切なタグに変える」** というスタンスで大丈夫です。
最初から完璧を目指すより、まずは動く（見える）ものを作ること。慣れてきたら、少しずつ `<section>` や `<article>` といった、より細かい「意味」を意識していきましょう。

### まとめ

- 頻繁に使うタグは 20 種類程度。
- `h1` 〜 `p` などのテキストタグと、`div` などの構造タグを使い分ける。
- セマンティック HTML は、Google や人間に「意味」を伝えるために重要。
- `img` タグの `alt` 属性は忘れずに。

### 展望

タグがわかれば、次はそれらをピンポイントで狙い撃ちして装飾するための「CSS セレクター」をマスターしましょう。特定の要素だけを自由自在に操る魔法の杖です。
