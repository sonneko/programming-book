# htmlの文法

HTML (**HyperText Markup Language**) は、Webページの「骨組み」を作るための言語です。プログラミング言語（Pythonなど）とは異なり、論理的な計算を行うのではなく、文章に対して「ここは見出し」「ここは段落」といった **意味付け（Markup）** をしていくのがその役割です。

私が初めてHTMLを触ったとき、「たったこれだけでブラウザに文字が出るんだ！」と感動した一方で、「タグを閉じるのを忘れて画面が崩れる」という洗礼も浴びました。基本をしっかり押さえていきましょう。

## 基本構造：タグと要素

HTMLは **タグ (Tag)** と呼ばれる特別な記号を使って記述します。

```html
<tagname>中身の内容</tagname>
```

- **`<tagname>`**: 開始タグ。ここからこの要素が始まりますよ、という合図です。
- **`</tagname>`**: 終了タグ。スラッシュ `/` を入れることで、ここで終わりですよ、と伝えます。
- **要素 (Element)**: 開始タグから終了タグまでの全体を指します。

例えば、段落を作りたいときは `<p>`（*Paragraph* の略）タグを使います。

```html
<p>これはHTMLの段落です。意味を持った文章になります。</p>
```

## 属性：タグに詳細な情報を与える

タグには、さらなる情報（設定値）を追加することができます。これを **属性 (Attribute)** と呼びます。

```html
<a href="https://google.com" target="_blank">Googleへのリンク</a>
```

- **`href`**: 属性名。ここではリンク先のURLを指定しています。
- **`"https://google.com"`**: 属性値。必ずダブルクォーテーション `"` で囲みます。
- **`target="_blank"`**: 別の属性。新しいタブで開く、という指示です。

属性は、開始タグの中に `属性名="属性値"` という形式でスペースを空けて記述します。

## HTMLの本質：木構造（DOM）

HTMLの最も重要な特徴は、要素の中に別の要素を入れることができる **入れ子（Nesting）** 構造です。

```html
<body>
  <h1>私の秘密のノート</h1>
  <div>
    <p>今日は、<span>HTML</span>について学びました。</p>
    <img src="memo.png" alt="ノートの画像">
  </div>
</body>
```

このように、一番外側の箱（`<body>`）の中に、見出し（`<h1>`）や別の箱（`<div>`）が入っている……という構造は、コンピュータ科学の世界では **「木構造 (Tree Structure)」** と呼ばれます。

ブラウザは、このHTMLテキストを読み込むと、メモリの中にこの木構造を組み立てます。これを **DOM (Document Object Model)** と呼びます。
後で JavaScript を使って「ボタンを押したら文字を変える」という処理をするとき、実はこの「DOMという木の枝を付け替えたり、葉っぱの色を変えたりする」操作を行っているのです。

## HTML5の基本テンプレート

Webページを新しく作るときは、必ず以下の「おまじない」のような形から始まります。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>ページのタイトル</title>
</head>
<body>
  <!-- ユーザーに見えるコンテンツはここに書く -->
  <h1>Hello World!</h1>
</body>
</html>
```

1. **`<!DOCTYPE html>`**: 「これは最新の HTML5 という形式ですよ」という、ブラウザへの宣言です。
2. **`<html lang="ja">`**: ページ全体の言語を指定します。
3. **`<head>`**: **「メタ情報」** を書く場所です。ここには、文字コードの設定やサイトのタイトル、CSSの読み込みなどを書きますが、**ブラウザの画面自体には表示されません**。
4. **`<body>`**: 実際に私たちがブラウザで見ることになる、Webサイトの中身を書く場所です。

## 低レイヤーの視点：文字コードの重要性

`<meta charset="UTF-8">` という記述がありますが、これは非常に重要です。

コンピュータは、全ての文字を「0」と「1」の数字（バイナリデータ）として扱っています。「あ」という文字をどの数字に対応させるかというルールが「文字コード」です。
もし、サーバーが「UTF-8」で送り、ブラウザが「Shift-JIS」という別のルールで解釈しようとすると、有名な **「文字化け」** が発生します。現代のWebでは、世界中のあらゆる文字を扱える `UTF-8` を使うのが絶対的なルールとなっています。

## 私の視点：インデントは「心」の乱れ

HTMLを書くときは、入れ子構造に合わせて **インデント（行頭のスペース）** を正しく入れるようにしましょう。

Pythonのように「インデントを間違えると動かない」わけではありませんが、インデントがバラバラなHTMLは、構造を把握するのが非常に困難になり、閉じタグの忘れなどのバグを見逃しやすくなります。
「機械のためではなく、未来の自分（人間）のために綺麗に書く」。これがエンジニアの第一歩です。

### まとめ

- HTMLは「タグ」を使って文章に意味をつける（マークアップ）。
- 要素は開始タグ、中身、終了タグで構成される。
- 属性（Attribute）を使って詳細な設定を行う。
- HTMLの構造は「木構造（DOM）」としてメモリに展開される。
- `head` は設定、`body` は中身。

### 展望

骨組みができたら、次はそれを美しく飾り付ける「CSS」の世界へ進みましょう。色をつけたり、形を整えたり、一気に「Webサイトらしく」なっていきますよ！
