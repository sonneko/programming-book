# 演習：ターミナルと Git の実践

6 章では、コマンドライン (CLI) の基本操作と、バージョン管理システム Git の基礎を学びました。これらはエンジニアにとっての「読み・書き・そろばん」のような基本スキルです。

今回は、これまでの知識を総動員して、実際の開発現場で毎日行われる「一連の流れ」をシミュレートしてみましょう。単にコマンドを書き写すのではなく、今、自分の PC の中（メモリやディスク）で何が起きているのかを想像しながら取り組んでみてください。

## 演習ミッション

以下の 3 つのステップを完遂してください。

### ステップ 1：ディレクトリ迷宮の探索と創造
1. ターミナルを開き、自分のホームディレクトリに `practice-git` という名前のディレクトリを作成してください。
2. その中に入り、`README.md` という空のファイルを作成してください。
3. `ls -la` コマンドを使い、隠しファイルも含めて現在のディレクトリの中身を確認してください。このとき、パーミッション（`-rw-r--r--` などの表記）も観察してみましょう。

### ステップ 2：Git の「タイムマシン」を起動する
1. `git init` を実行し、このディレクトリを Git リポジトリとして初期化してください。
2. `ls -la` をもう一度実行し、`.git` という隠しディレクトリが作成されたことを確認してください。**ここがタイムマシンの「心臓部」です。**
3. `README.md` に自分の名前を書き込み、保存してください。
4. `git status` で変更を検知していることを確認し、`git add` ＞ `git commit` を行ってください。
5. `git log` で、あなたの最初のコミットが歴史に刻まれたことを確認してください。

### ステップ 3：パラレルワールド（ブランチ）の生成
1. `git branch dev` を実行し、新しい「開発用」の並行世界を作成してください。
2. `git checkout dev` でその世界に移動します。
3. 新しいファイル `feature.txt` を作成し、何か適当な文字を書いてコミットしてください。
4. `git checkout main` で元の世界に戻り、`feature.txt` が **消えている（存在しない）** ことを確認してください。
5. `git merge dev` を実行し、`dev` の世界の変更を `main` に統合してください。これで `feature.txt` が現れるはずです。

## 低レイヤーの視点：`.git` ディレクトリを覗く

コミットをしたとき、`.git` ディレクトリの中では何が起きているのでしょうか？

実は、Git は変更されたファイルの「差分」を保存しているのではなく、**「ファイル全体の圧縮スナップショット」** を保存しています。
1. `git add` すると、`.git/objects` の中にファイルの圧縮データが作られます。
2. `git commit` すると、その時のディレクトリ全体の構成（ツリー）を記録したファイルが作られます。

`.git` ディレクトリを削除すると、その瞬間にすべての履歴（過去）が消滅し、ただの「現在のファイル」だけが残ります。この `.git` こそが、Git の本体なのです。

## 私の視点：失敗を恐れないための Git

私が初心者の頃に最も怖かったのは、`rm -rf` でファイルを消してしまうことや、間違ったコードを書いて元に戻せなくなることでした。

でも、Git を使いこなせるようになると、世界の見え方が変わります。
「最悪、さっきコミットしたところまで戻ればいいや」と思えるからこそ、大胆なコードの書き換え（リファクタリング）や、新しい技術への挑戦ができるようになります。
Git は単なる管理ツールではなく、**あなたの挑戦を支える「安全装置」** なのです。

### まとめ

- ターミナルの基本操作 (`cd`, `mkdir`, `ls`) を「指」で覚えよう。
- `git init` でタイムマシンの心臓部 (`.git`) が作られる。
- `add` で準備し、`commit` で記録し、`log` で歴史を見る。
- ブランチを使えば、本番環境を壊さずに実験ができる。
- 失敗しても戻れるという安心感が、エンジニアとしての勇気をくれる。

### 展望

ターミナルと Git という「プロの道具」を使いこなせるようになったあなたは、もう立派な開発者の卵です。
次は、いよいよモダン Web 開発の王道、**React** の深い世界へと潜っていきましょう。今学んだ Git が、チーム開発でどのように真価を発揮するのかも体感することになります。
