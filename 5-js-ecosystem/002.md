# Three.js入門

Web ブラウザは、元々はテキストや画像を表示するためのツールでした。しかし、今やブラウザの中で本格的な 3D ゲームを動かしたり、仮想空間（メタバース）を構築したりすることが可能です。それを支えているのが **WebGL** という技術であり、その WebGL を誰でも簡単に扱えるようにしてくれる魔法の杖が **Three.js** です。

私が初めて Three.js で画面に立方体を表示させ、それをマウスでぐるぐると回せたとき、ブラウザの「向こう側」に無限の空間が広がったような感動を覚えました。

## 1. 3D の世界を作る「三種の神器」

Three.js で 3D 空間を作るには、現実世界の撮影と同じように、最低限 3 つの要素が必要です。

- **Scene (シーン)**: 全体の「舞台」です。3D オブジェクト、ライト、背景などをすべてここに配置します。
- **Camera (カメラ)**: シーンをどの視点から、どのくらいの画角で見るかを決めます。
- **Renderer (レンダラー)**: シーンとカメラの情報を元に、PC の GPU を動かして、ブラウザの Canvas 要素に絵を描き出します。

```javascript
// シーンの作成
const scene = new THREE.Scene();

// カメラの作成（画角, アスペクト比, 描画開始距離, 描画終了距離）
const camera = new THREE.PerspectiveCamera(75, window.innerWidth / window.innerHeight, 0.1, 1000);

// レンダラーの作成
const renderer = new THREE.WebGLRenderer();
renderer.setSize(window.innerWidth, window.innerHeight);
document.body.appendChild(renderer.domElement);
```

ここに、**Mesh (メッシュ)** と呼ばれる「物体」を追加していきます。メッシュは、**Geometry (形状)** と **Material (質感)** を組み合わせて作られます。

## 2. アニメーションループ

3D グラフィックスは、一瞬の静止画ではありません。1 秒間に 60 回（60FPS）、カメラの位置をずらし、物体を回転させ、再描画し続けることで「動き」が生まれます。

```javascript
function animate() {
  // ブラウザの再描画タイミングに合わせて実行
  requestAnimationFrame(animate);

  // 物体を少しだけ回転させる
  cube.rotation.x += 0.01;
  cube.rotation.y += 0.01;

  // 描画！
  renderer.render(scene, camera);
}
animate();
```

## 低レイヤーの視点：GPU とシェーダー

Three.js の裏側では、CPU ではなく **GPU（グラフィックス処理ユニット）** がフル稼働しています。

何万個もの三角形（ポリゴン）の頂点計算を CPU で行うと、ブラウザは即座にフリーズしてしまいます。
GPU は、単純な計算を数千個同時に並列実行することに特化した計算機です。
Three.js は、JavaScript の命令を **GLSL** というシェーダー言語に変換（またはラップ）して GPU に送り込み、ピクセルごとの色計算を超高速に行わせています。
3D プログラミングを学ぶことは、コンピュータにおける「並列計算」の極意に触れることでもあるのです。

Three.js を使っていると、「かっこいいモデルをたくさん出したのに、カクカクして動かない」という壁に必ずぶつかります。

- **Draw Call (ドローコール)**: GPU への描画命令の回数です。1000 個の違う立方体を出すより、1 個の巨大なメッシュ（あるいはインスタンス化されたメッシュ）を出す方が圧倒的に速いです。
- **テクスチャサイズ**: 大きすぎる画像は、GPU メモリを圧迫し、スマホでのクラッシュを招きます。

「ただ表示する」のは簡単ですが、「すべてのユーザーの環境でスムーズに動かす」ことこそが、Three.js 職人への道です。

### まとめ

- Three.js は WebGL を扱いやすくするライブラリ。
- `Scene`, `Camera`, `Renderer` が基本のセット。
- 物体は `Geometry` (形) と `Material` (質感) から成る。
- `requestAnimationFrame` によるループでアニメーションを実現する。
- 高速な描画の裏側には GPU による並列計算がある。
- パフォーマンス向上のためには「描画命令をまとめる」工夫が重要。

### 展望

ブラウザの中に 3D 空間を作れるようになったら、次はその 3D 空間も含めた Web アプリ全体を、効率よく管理するための強力な味方、**React** について学びましょう。
