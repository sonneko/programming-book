# React概要

現代の Web 開発において、**React** は避けて通れない存在です。Meta（旧 Facebook）社によって開発されたこのライブラリは、単なる「便利な道具」を超えて、私たちの Web アプリケーションの作り方そのものを根本から変えてしまいました。

なぜ、世界中のエンジニアが React に熱狂するのでしょうか？ それは、React が **「UI を『状態（データ）』の関数として定義する」** という、極めて美しく合理的な思想を持っているからです。

## 1. 宣言的 UI：DOM 操作からの解放

かつての JavaScript 開発（Vanilla JS）では、データが変わるたびに「どの HTML 要素を捕まえて、どこを書き換えるか」を、私たちが泥臭く指示していました。

```javascript
// Vanilla JS: 手続き的な指示
const count = 5;
const el = document.getElementById('counter');
el.textContent = `現在のカウント: ${count}`;
```

React では、こう書きます。

```javascript
// React: 宣言的な定義
function Counter({ count }) {
  return <div>現在のカウント: {count}</div>;
}
```

「こうやって書き換えろ」という **手続き** ではなく、「このデータならこの見た目になる」という **宣言** を書くだけ。データ（`count`）が変われば、React が勝手に判断して、画面の必要な部分だけを最小限のコストで書き換えてくれます。これが React の最大の武器です。

複雑な画面も、小さな部品の組み合わせに過ぎません。部品ごとにロジックと見た目を閉じ込める（カプセル化する）ことで、開発のスピードが上がり、バグの特定も容易になります。

## 3. Hooks：関数の魔法

モダンな React 開発を支えるのが **Hooks (フック)** です。
もともと React のコンポーネントは「ただの関数」でしたが、Hooks が登場したことで、関数の中に「状態（メモリ）」を持たせたり、「副作用（データの取得など）」を扱ったりできるようになりました。

```javascript
import { useState } from 'react';

function App() {
  // count という状態と、それを更新する関数を作る
  const [count, setCount] = useState(0);

  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

この Hooks の正体は、JavaScript の **クロージャ** を活用した極めて巧妙な仕組みです。

## 低レイヤーの視点：仮想 DOM とレンダリング最適化

React がなぜ速いのか。その秘密は **「仮想 DOM (Virtual DOM)」** にあります。

ブラウザの実際の DOM をいじるのは、非常にコストの高い（遅い）処理です。
React は、メモリの中に「本物の DOM を模した軽いコピー（仮想 DOM）」を持っており、データが変わると、まず仮想 DOM 同士を比較して「どこが変わったか」を計算します（**Diffing アルゴリズム**）。

そして、**「本当に変わった部分だけ」** を本物の DOM に反映させます。
さらに最新の React (Fiber) では、この計算を細切れにして、ユーザーの操作（スクロールなど）を邪魔しないよう優先順位をつけて実行する、という OS のスケジューラのような高度な制御まで行っています。

## 私の視点：React は「考え方」である

React を学ぶことは、単に新しい文法を覚えることではありません。「画面をデータの写像として捉える」という、新しい数学的な思考を手に入れることです。

一度 React の考え方を身につければ、Web だけでなくスマホアプリ（React Native）や 3D 空間（React Three Fiber）まで、同じロジックで構築できるようになります。
「Learn Once, Write Anywhere」。この一貫した哲学こそが、React がエコシステムの中で覇権を握り続けている理由なのです。

### まとめ

- React は UI を「データ（State）の結果」として宣言的に記述するライブラリ。
- 「コンポーネント」という部品を組み合わせて、大規模な UI を構築する。
- Hooks によって、関数の中に状態や副作用をスマートに閉じ込める。
- 仮想 DOM と Fiber アーキテクチャが、高速なレンダリングを支えている。
- React の知識は、Web 以外の様々なプラットフォームにも応用できる。

### 展望

React の概要を掴んだら、次はいよいよ具体的な使い方―― JSX の書き方やコンポーネントの設計、そして Next.js を使ったフルスタック開発の世界へと踏み込んでいきましょう。
