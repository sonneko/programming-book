# クォータニオンと3Dの回転

3D空間で物体を「回す」というのは、直感に反して、プログラミングにおいては非常に厄介なトピックです。初心者が最初に行き着くのは、X, Y, Z軸ごとの回転角度を指定する **「オイラー角」** ですが、これには致命的な欠陥があります。

それを解決し、現代のあらゆる3Dソフトやゲームエンジンの裏側で回転を支えているのが、**クォータニオン** (*Quaternion* / 四元数) です。

### オイラー角の悲劇：ジンバルロック

オイラー角（例えば `x=30°, y=45°, z=90°`）は人間には分かりやすいですが、計算機には不都合です。

最大の問題は、**ジンバルロック** (*Gimbal Lock*) です。
特定の角度（例えばX軸を90度回した状態）になると、他の二つの回転軸が重なってしまい、一つの自由度が失われる現象です。こうなると、特定の方向へスムーズに回転できなくなり、キャラクターの動きが不自然に跳ねたり、制御不能になったりします。

### クォータニオン：4次元からの救世主

クォータニオンは、数学的には4つの実数の組 `(x, y, z, w)` で構成されます。
その正体は複素数を拡張した4次元の数ですが、エンジニアとして理解すべきポイントは **「任意の回転を、軸と角度のセットで表現する」** という点です。

クォータニオンの利点は圧倒的です。
1. **ジンバルロックが起きない**: どのような回転状態であっても、すべての方向へ自由に回すことができます。
2. **メモリ効率が良い**: 行列は16個の数字を使いますが、クォータニオンは4個で済みます。
3. **滑らかな補間ができる**: これが最も重要です。

### アニメーションの要：slerp

ある回転状態Aから状態Bへと、滑らかにアニメーションさせたいとき、クォータニオンは **slerp** (*Spherical Linear Interpolation* / 球面線形補間) という魔法のような計算が可能です。

オイラー角で角度を単純に足し算すると、回転の軌跡が歪んだり速度が一定にならなかったりしますが、slerpを使えば、最短距離を通って完璧に滑らかに回転を繋ぐことができます。現代のキャラクターの滑らかな動き、カメラの自然な旋回は、すべてこのslerpのおかげです。

### 思考の切り替え：数式ではなく「道具」として

クォータニオンの内部の数学（四元数の掛け算など）を完全に理解しようとすると、多くの数学者が挫折するほどの深淵に足を踏み入れることになります。しかし、グラフィックスエンジニアの多くは、中身の数式そのものよりも **「どういう性質を持ち、どう使うか」** に集中しています。

- 回転を合成したいなら、クォータニオン同士を掛ける。
- 方向ベクトルを回転させたいなら、クォータニオンを適用する。
- 最終的にGPUに送るときは、クォータニオンを行列に変換する。

このように、優れた抽象化としてクォータニオンを使いこなすことが、実務における正解です。

### 低レイヤーの現実：正規化を忘れるな

クォータニオンを使って計算を繰り返していると、浮動小数点の累積誤差によって、少しずつ「長さ」が1からズレていくことがあります。

長さが1ではないクォータニオンは、回転だけでなく、勝手に物体を「拡大」や「縮小」させてしまう副作用を持ちます。そのため、計算の節目節目でベクトルのときと同じように **正規化** (Normalization) を行う必要があります。この小さなケアを忘れないことが、安定した3Dシステムを作る秘訣です。

### まとめ

- オイラー角は分かりやすいが、ジンバルロックという欠陥がある。
- クォータニオンは4つの数字で回転を表現し、ジンバルロックを防ぐ。
- slerpを使えば、二つの回転の間を完璧に滑らかに補間できる。
- 中身の数学に惑わされず、強力な「回転の道具」として使いこなす。

回転の問題を克服すれば、3D空間での物体の動きはあなたの思いのままです。次は、これらの数学的成果をGPUに効率よく届けるための「メモリ」の話、バッファとレイアウトについて見ていきましょう。
