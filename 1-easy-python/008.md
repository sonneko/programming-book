# 真偽値

今回は真偽値をより深掘りしていきます。

典型的に中学校の数学で習う論理演算をプログラミングの世界でも使うことができます。

例えば、`x`が10から20の範囲にあるかどうかを判定するにはどうすればいいでしょうか？

```python
x = 14
if (x > 10):
    if (x < 20):
        print("x is between 10 and 20.")
```

このようなこともできますが、以下のようなこともできます。

```python
x = 14
if (x > 10 and x < 20):
    print("x is between 10 and 20.")
```

数学のように`10 < x < 20`と書くこともできます。Pythonはこの書き方をサポートしている珍しい言語の一つです。

```python
x = 14
if 10 < x < 20:
    print("x is between 10 and 20.")
```

しかし、基本的には`and`を使って以下のように書くのが一般的です。なぜなら、他のほとんどのプログラミング言語では上の書き方はできないからです。

```python
x = 14
if x > 10 and x < 20:
    print("x is between 10 and 20.")
```

`x > 10`の結果と、`x < 20`の結果が`and`で結ばれることで、どちらも`True`である時のみ最終的な結果が`True`になるようにできます。

なぜ他の言語で`10 < x < 20`と書けないかは、前回説明した関数としての演算子の性質を思い出すことで理解できます。`10 < x`は真偽値を返すので、`(真偽値) < 20`となってしまい、関数としては型エラー的なものが発生しますね。

他にもどちらかが`True`ならばもう一方がどうであっても`True`になるという`or`も使うことができます。

```python
x = 32
output = x < 10 or x > 20
print(output) # False
```

この例では、10未満もしくは20より大きいことを確認しています。

Pythonでは、`and`や`or`という言葉をそのまま使います。他の多くの言語（C言語やJavaScriptなど）では、`and`の代わりに`&&`、`or`の代わりに`||`という記号を使いますが、Pythonではこれらは使えないので注意しましょう。

> ### コラム：短絡評価=*short circuit*
> `and` や `or` には、**「結果がわかった時点で計算をやめる」**という賢い性質があります。これをショートサーキットと呼びます。
>
> 例えば、`A and B` という計算において、もし `A` が `False` だったら、 `B` が何であっても全体の結果は必ず `False` になりますよね？ そのため、Pythonは `B` の部分を見ることなく、すぐに `False` を返します。
>
> これは単なる効率化だけでなく、エラーを避けるテクニックとしても使われます。
> ```python
> # データの数が1つ以上、かつ、最初のデータが 100 のとき
> if len(my_list) > 0 and my_list[0] == 100:
>     ...
> ```
> もしリストが空の場合、`len(my_list) > 0` が `False` になり、右側の `my_list[0]` は実行されません。もしショートサーキットがなければ、空のリストの要素にアクセスしようとしてエラーになってしまいます。
