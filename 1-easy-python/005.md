# 関数の使用

Pythonや他には`関数`=*function*という概念があります。プログラミングにおける関数は、最初は、数学における`関数`とは全く別のものとして考えておいてください。

## 関数の書き方

関数とは、入力=*input*を受け取って出力=*output*するようなものです。以下の例では `left`と`right`を受け取って`output`を返しています。

```python
def add(left, right):
    output = left + right
    return output

output = add(1, 2)
print(output) # 3が出力される
```

`def`は関数を定義するという記号で、その後に関数の名前がきます。この名前付けのルールは基本的には変数と同じです。変数の名前付けルールについては前回を参照してください。

その後、受け取るものリスト(パラメータ)がきます。リストはあなたが決めた変数名を`,`区切りで記入してください。

パラメータは、変数の定義と考えてください。ただし、初期化=*initialize*する必要はなくて、自動的に呼び出された時に与えられた値が代入されるイメージです。

例えば以下のコードは...

```python
def your_func(a):
    print(a)

your_func(10)
```

```python
def your_func(a = 10) {
    print(a)
}
```

になるイメージです。この代入されているという感覚はプログラムを直感的に理解するときに重要なので、体に染みつけておきましょう。

それを書くことができたら`:`で締めくくって、その後関数の中身を書いていきます。ここにこの関数が実行する命令を書いていきます。複数行になって構いません。この関数の中身を書く範囲は右端をタブ=`\t`もしくは半角の空白4つで埋める必要があります。これをインデントと言います。

命令が終わって何かを返す必要があるのであれば`return`でそのすぐ後に来たものを返し=*return*即座にその関数の処理を終了します。


>注意点として、`return`が実行されるとその後の処理は実行されません。
>```python
>def return_zero():
>    return 0
>    print("You can't find me.")
>```
> 例えばこの場合は`print("You can't find me.")`は実行されません。
> このような場合、`print関数`の呼び出し行は、デッドコード=*dead code*と呼ばれ、決して実行されないコードという意味です。

関数の記法：
```python
# 関数を定義する
def 関数の名前(引数リスト):
    処理..
    ...

# 関数を呼び出す
関数の名前(引数リスト)
```

## 変数の有効範囲について

#### 関数内で新たに作った変数はその外から見ることができない

以下の例はエラーとなってしまいます。

```python
def add(left, right):
    output = left + right
    return output

print(output)
```

「outputという変数はトップレベルで定義されていません。」というエラーが出るかと思います。関数内部で定義した変数はその関数の外から見ることができません。

この場合「output変数の有効範囲はadd関数の中身である」ということができます。

この変数の有効範囲のことをスコープ=*scope*と言います。また、関数の中の変数をローカル変数=*local variable*・関数の外の変数をグローバル変数=*global variable*と言います。

しかし、新たに変数を作って覆い隠すことは可能です。

```python
def do_nothing():
    a = 10

a = 1 # 再代入ではなく新しく定義している
print(a) # 1と出力される。同一スコープで定義されているので問題なく使える
```

#### 変数を「見る」時は、まずは狭いスコープで定義されていないかを確認し、定義されていなかったらより広い範囲のスコープで定義されていないかを確認する。
```python
a = 10
b = 1

def do_nothing():
    a = 1 # 再代入ではなく新しく定義している
    print(a) # aはdo_nothingで定義されているのでそちらが優先され1が出力される
    print(b) # do_nothingで定義されていないので外側のものを読み取るので1と出力される

print(a) # 10と出力される
```

Pythonプログラム全体で利用可能な変数のスコープを特別に`トップレベルスコープ`=*top level scope*と呼びます。

> ### コラム
>
> このローカル変数という考え方は数学にも登場します。もしかすると、こちらを知っていたらスッと理解できるかもしれません。
> 
> $$
> \int f(x) dx = \int f(t) dt
> $$
> $$
> \sum_{k} a_{k} = \sum_{l} a_{l}
> $$
> これらの数式でいう、x, t, k, lがローカル変数です。ローカル変数はただのプレイスホルダー=*place holder*なので、名前を変えても全く問題ありません。
>


#### なぜ関数を作るのか？
プログラミングには DRY (Don't Repeat Yourself)という原則があります。「同じことを二度書くな」という意味です。

> このような原則はこの後たくさん出てきますから、その都度覚えていきましょう。

何度も出てくる処理を関数にまとめておけば、修正が必要になったときにその関数一箇所を直すだけで済みます。

また、複雑なプログラムに名前（関数名）をつけて小分けにすることで、人間がコードを読んだ時に「あぁ、ここでは合計金額を計算しているんだな」と理解しやすくなります。

また、再帰などの複雑な実行順序をわかりやすく＝意味論的=*semantically*に記述することができます。
