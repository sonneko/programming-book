# リスト型

これまでは、一つの変数に一つのデータ（整数や文字列）を入れてきました。しかし、実際のプログラミングでは「クラス全員の名前」や「一週間分の気温データ」のように、関連する複数のデータをまとめて扱いたい場面が多々あります。

そこで登場するのが**リスト（List）**です。

## リストとは
リストは、複数のデータを一つの「入れ物」の中に順番に並べて管理するための型です。他の言語では「配列」と呼ばれることもありますが、Pythonのリストは非常に柔軟で、異なる型（整数と文字列など）を混ぜて入れることも可能です。[^note1]

[^note1]: 効率や安全性の観点から、実際の開発では同じ型のデータだけを並べることが一般的です。

```python
# リストの定義
fruits = ["apple", "banana", "cherry"]
numbers = [10, 20, 30, 40]
mixed = ["Alice", 100, True] # 混ぜることもできる

print(fruits)
```

## インデックス（添え字）によるアクセス
リストの中の特定のデータを取り出すには、**インデックス**と呼ばれる番号を使います。
[while文の章](./009.md)でも触れましたが、コンピュータの世界では番号は **0番から始まります**。


なぜ0から始まるのでしょうか？これはコンピュータの低レイヤな仕組みに関係しています。
リストはメモリ上で「連続した領域」にデータが並んでいます。`fruits[0]` というのは、「リストの先頭から0個分だけ進んだ場所」という意味になります。`fruits[1]` は「先頭から1個分進んだ場所」です。
コンピュータにとっては、先頭からの「ズレ（オフセット）」で場所を特定するのが最も自然なので、歴史的にそうなっています。


```python
fruits = ["apple", "banana", "cherry"]

print(fruits[0]) # apple
print(fruits[1]) # banana
print(fruits[2]) # cherry
```

もし存在しない番号（例えば3番など）を指定すると、`IndexError` というエラーが出てコンピュータに怒られます。「そんな場所には何もないよ！」という意味です。

## リストの操作
リストは後から中身を変えたり、追加したりできます。

```python
# 中身の書き換え
fruits[1] = "grape"
print(fruits) # ["apple", "grape", "cherry"]

# 要素の追加
fruits.append("orange")
print(fruits) # 末尾にorangeが追加される

# リストの長さを知る
print(len(fruits)) # 4
```

リストの長さは、私が直感的に思う長さであって、最後のインデックスの数ではありません。整理すると、長さnの配列は0から始まりn-1で終わるインデックスを持ちます。

## リストの操作
リストにデータを追加したり、中身を書き換えたりすることも簡単です。

```python
fruits = ["apple", "banana"]

# データの末尾への追加(push・appendと呼ばれる)
fruits.append("orange")
print(fruits) # ["apple", "banana", "orange"]

# データの書き換え(ランダムアクセス)
fruits[0] = "strawberry"
print(fruits) # ["strawberry", "banana", "orange"]

# リストの長さ（要素の数）を取得
print(len(fruits)) # 3
```

この`.append()`と言うような記法を、メソッド呼び出しと言います。要は、リスト型に便利な関数が紐づけられていて、その関数を呼び出すイメージです。

## なぜリストを使うのか？

リストを使う最大のメリットは、[for文](./010.md)と組み合わせることで、大量のデータを一気に処理できることです。

```python
prices = [100, 200, 300, 400, 500]
total = 0

for price in prices:
    total = total + price

print(f"合計金額は {total} 円です")
```

このように、リストは「データの集まり」を一つの単位として扱い、それを「計算処理」へと流し込むための非常に強力なツールなのです。

> ### コラム：リストがないと書けないプログラム
> 「変数をたくさん作るのは面倒だからリストを使う」というのは正しいですが、それだけではありません。
> 実は、リストのような仕組みがないと、**「そもそも書くことが不可能なプログラム」**が存在します。
>
> 例えば、「ユーザーが入力した数字をすべて合計し、最後に表示する。ただし、ユーザーが何個数字を入力するかは実行するまでわからない」というプログラムを考えてみてください。
> 変数（`val1`, `val2` ...）を使う方法では、あらかじめ変数の数を決めておかなければなりません。100個用意しても、ユーザーが101個入力したらお手上げです。
>
> リストを使えば、実行中に要素をいくらでも追加できるため、データの数が事前に決まっていない「動的な」状況にも対応できます。これは現代のソフトウェアにおいて、リスト（やそれに類するデータ構造）が不可欠である最大の理由の一つです。
