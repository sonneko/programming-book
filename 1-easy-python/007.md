# if文

プログラムは、ただ上から下へ流れるだけではありません。状況に応じて「右へ行くか、左へ行くか」を決める必要があります。これを 「条件分岐」=*conditional branch* と呼び、そのための最も基本的な道具が `if`文です。

## 基本的な使い方

`if`文の基本構造は以下の通りです。

```python
if 条件式:
    # 条件が満たされた（True）の時の処理
    print("条件を満たしました")
```

またもやインデントを用います。

## if, elif, else：複数の分岐

「もし〜ならA、そうでなくて〜ならB、それ以外ならC」という風に、複数の条件を組み合わせることもできます。

```python
score = 85

if score >= 90:
    print("大変素晴らしいです！")
elif score >= 70:
    print("合格です。")
elif score >= 40:
    print("もう少し頑張りましょう。")
else:
    print("不合格です。")
```

- `if`: 最初の条件。
- `elif`: "else if" の略。前の条件が外れた場合に、次の条件をチェックします。
- `else`: すべての条件に当てはまらなかった場合の「最終手段」です。

## 比較演算子：条件を作る道具

条件式には、2つの値を比べる「比較演算子」を使います。ただし、この比較演算子も数学で言う演算子と変わりありません。

例えば、`+`のような算術演算は前後にある二つの数字を加算し数字を排出するという意味ですが、同様に比較演算子は前後の2つの値について何らかの決まった処理を行いその結果をブール値で返すと言うものです。言うなれば、2変数関数です。

```python
def add(a, b):
    return a + b
```

「`if`文の中で使うもの」ではなく`+`などと同じただの演算子であるという風に一元的に理解しておきましょう。

| 記号 | 意味 | 例 |
|---|---|---|
| `==` | 等しい | `x == 10` |
| `!=` | 等しくない | `x != 0` |
| `<` | より小さい | `x < 5` |
| `>` | より大きい | `x > 10` |
| `<=` | 以下 | `x <= 8` |
| `>=` | 以上 | `x >= 0` |

ここで注意が必要なのは、「等しい」を表すのが `==`（イコール2つ）であることです。プログラミングにおいて `=` 1つは「代入」を意味するため、区別するために2つ重ねるというルールの言語が多いです。

## 論理演算子：複雑な条件を組み合わせる

「AかつB」「AまたはB」といった複雑な条件は、「論理演算子」を使って作ることができます。

同様にこの論理演算子は、ブール値を2つ受け取ってブール値を1つ返す関数のようなものです。

```python
age = 18
has_license = True

if age >= 18 and has_license:
    print("車を運転できます。")

if age < 18 or not has_license:
    print("車を運転できません。")
```

- `and`: 両方の条件が正しい時に `True`。
- `or`: どちらか片方でも正しければ `True`。
- `not`: 判定をひっくり返します（`True` なら `False`、`False` なら `True`）。

`not`演算子は、単項演算子と呼ばれます。これは、負の数を表現するのに用いる`-`と同じで、後続のブール値を一つだけ取って、その逆を返します。

## ネストを深くしすぎない

`if`文の中にさらに `if`文を書くことを「ネスト（入れ子）」と言います。より一般的には、`if`文でなくとも、何らかの構造をマトリョーシカのように何度も重ねていくことをネストと言います。

```python
if user_exists:
    if user_active:
        if user_has_permission:
            do_something()
```

ネストが深くなると、今自分がどの条件の中にいるのか分からなくなり、バグの温床になります。そんな時は、条件を反転させて先に `return` してしまう 「ガード節」というテクニックを使うのがお勧めです。

```python
if not user_exists:
    return
if not user_active:
    return
if not user_has_permission:
    return

do_something()
```

### まとめ

- `if`, `elif`, `else` でプログラムの流れを枝分かれさせる。
- 比較演算子（`==`, `!=`, `<`, `>`）で条件を作る。
- 論理演算子（`and`, `or`, `not`）で条件を組み合わせる。
- ネストを浅く保つことが、読みやすいコードへの近道。

### 展望

次は、この「条件」をさらに活用して、同じ処理を何度も繰り返す「ループ（繰り返し構文）」について学んでいきましょう。
