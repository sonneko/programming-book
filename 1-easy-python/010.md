# for文

前回学んだ `while`文を使えば、どんな繰り返し処理も書くことができます。しかし、実際にプログラムを書いていると「特定の回数だけ繰り返す」とか「リストの要素を一つずつ取り出す」といったパターンが非常に多いことに気づきます。というか、それ以外にはほとんど存在しません。大抵の場合は、リスト

このような「よくある繰り返し」を、より簡潔に、安全に書くための構文が **`for`文** です。

## 基本的な使い方：回数を指定する

Pythonの `for`文は、`while`文よりも高級な構文です。というのも、`while`のように「繰り返す」というアルゴリズムを書いているのではなく、「それぞれに対して〜する」というより意味論的な記述のみを書いているからです。

```python
for index in range(10):
    print(f"現在は {index} 回目です")
```

このプログラムは、0から9までの数字を順番に `index` に代入しながら、10回実行されます。

`while`文で同じことを書くとこうなります：
```python
index = 0
while index < 10:
    print(f"現在は {index} 回目です")
    index += 1
```

この場合は、`for`文の方が「何回繰り返すか」が一目で分かり、さらに `index += 1` のような更新処理を書き忘れて無限ループになる心配もありません。実際に繰り返し回数の決まっているループを少ないかもしれませんが、すなわち、`index += 1`や`index < 10`といった低級でミスしやすい場所に触れずに、「繰り返し」という概念をより簡潔に本質的に表しています。

## イテレータという考え方

ここで理解しておきたいのが、`range(10)` の正体です。実はこれ、単なる数字の列ではなく、「イテレータ=*Iterator*」と呼ばれるオブジェクトを生成する関数です。

イテレータとは、一言で言えば 「次の値を順番に出してくれる『もの』」です。

`for`文は、このイテレータに対して「次の値をちょうだい」と頼み続け、値がなくなるまで処理を繰り返します。

逆にいうと、「次の値をちょうだい」と言って実際に次の値をくれるものなら、何でも`for`の後におけます。

## 実践：データの集まりを処理する

`for`文が最も輝くのは、複数のデータが入った「リスト（配列）」を扱うときです。

```python
fruits = ["apple", "banana", "orange", "grape"]

print("お店の在庫一覧：")
for fruit in fruits:
    print(f"・{fruit}")
```

このコードでは、`range()` の代わりに `fruits` というリストを直接 `for`文に渡しています。Pythonはリストから要素を一つずつ取り出し、変数 `fruit` に入れてくれます。これができるのは、リストが「次の値をちょうだい」と言って実際に次の値をくれるものデアルという条件を満たしているからです。

## 私の視点：`enumerate` でインデックスも一緒に使う

`for fruit in fruits:` は便利ですが、たまに「今、何番目の要素を処理しているか（番号）」も知りたいときがあります。そんな時は `enumerate()` 関数を使うのも一考です。

```python
fruits = ["apple", "banana", "orange"]

for i, fruit in enumerate(fruits):
    print(f"{i + 1}番目のフルーツは {fruit} です")
```

要は`enumerate`とは、`fruits`という文字列をもたらす=*yield*イテレータを、数字と文字列のセットをもたらすイテレータに変換するための関数です。

### まとめ

- `for`文は「決まった回数」や「データの集まり」を処理するのに適している。
- `range(n)` は 0 から n-1 までの値を生成する。
- リストを直接渡すと、要素を一つずつ取り出せる。
- `range` は「遅延評価」によってメモリを節約している。
- インデックスが必要なときは `enumerate()` を使う。

### 展望

さて、ここまで「データの集まり」としてサラッと「リスト」を紹介してきましたが、リストはプログラミングにおいて非常に奥が深く、重要なデータ構造です。次は、このリスト型のモダンな使い方をより詳しく、徹底的にマスターしていきましょう。
