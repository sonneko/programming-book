# コレクション型

これまで、私たちは変数を用いて一つひとつの値を扱ってきました。しかし、現実世界のデータは一つだけではありません。例えば「ユーザーのリスト」「ある日の売上データ」「設定項目」など、関連する多くのデータをまとめて扱いたい場面が多々あります。

このとき、それぞれを別々の変数として管理するのは現実的ではありません。そこで登場するのが「コレクション型」です。コレクション型とは、複数のデータを一つの箱に詰め込んで管理するための型です。

## なぜコレクション型が必要なのか？

単純に複数の値を管理するだけでなく、コレクション型は「計算処理（データの変形）」を行う際の強力な道具となります。

たとえば、1000個の売上データから合計金額を計算する際、変数を1000個定義して足し合わせるコードを書くことはありませんよね。コレクション型を使うことで、これらのデータを一つのまとまりとして扱い、ループ処理（`for`文など）によって効率的に計算を適用できるのです。

「計算処理をデータに適用する」という視点で言えば、コレクション型は計算の舞台そのものだと言えます。

## 代表的なコレクション型

Pythonで特によく使うコレクション型は以下の3つです。

### 1. リスト (List)
順序を持つデータの集まりです。追加・削除が可能で、インデックス（0番目、1番目…）を指定して高速に値にアクセスできます。

```python
# リストの定義
prices = [100, 200, 300]
print(prices[0]) # 100
```

### 2. 辞書 (Dictionary)
「キー」と「値」のペアでデータを管理します。ある値に対して名前（キー）をつけて管理したい場合に非常に便利です。広い意味で、このキーもインデックスと呼ばれます。

```python
# 辞書の定義
user = {"name": "Alice", "age": 25}
print(user["name"]) # Alice
```

### 3. セット (Set)
重複を許さないデータの集まりです。データの「存在チェック」を高速に行いたい場合に用いられます。数学の集合演算（積集合、和集合など）もサポートしています。リストや辞書と違いそのデータへのインデックスという概念がありません。

```python
# 1. 初期化 (Initialization)
s1 = {1, 2, 3}
s2 = set([3, 4, 5])
empty = set()

# 2. 要素の追加・更新 (Add / Update)
s1.add(6)          # 要素を1つ追加 -> {1, 2, 3, 6}
s1.update([7, 8])  # 複数の要素をまとめて追加 -> {1, 2, 3, 6, 7, 8}

# 3. 要素の削除 (Remove)
s1.remove(8)       # 指定した要素を削除（なければ KeyError）
s1.discard(9)      # 指定した要素を削除（なくてもエラーにならない）
popped = s1.pop()  # 任意の要素を1つ取り出して削除
s1.clear()         # すべての要素を削除して空にする -> set()

```

```python
s1 = {1, 2, 3}

# 4. 存在確認・判定 (Check / Judge)
has_item = 2 in s1        # True (要素が含まれているか)
is_missing = 9 not in s1   # True (要素が含まれていないか)
size = len(s1)            # 3 (要素数)

# 5. 集合演算 (Set Operations)
# 【和集合】すべてを合わせる
union_op = s1 | s2                 # 演算子 -> {1, 2, 3, 4, 5}
union_mt = s1.union(s2)            # メソッド

# 【積集合】共通部分を抜き出す
inter_op = s1 & s2                 # 演算子 -> {3}
inter_mt = s1.intersection(s2)     # メソッド

# 【差集合】s1 から s2 にあるものを引く
diff_op = s1 - s2                  # 演算子 -> {1, 2}
diff_mt = s1.difference(s2)        # メソッド

# 【対称差】どちらか片方だけにしかないもの
sym_op = s1 ^ s2                   # 演算子 -> {1, 2, 4, 5}
sym_mt = s1.symmetric_difference(s2) # メソッド

# 6. 比較・関係の判定 (Comparison)
is_disjoint = s1.isdisjoint({4, 5})  # True (共通要素が1つもないか)
is_subset = {1, 2}.issubset(s1)      # True (子が親に含まれているか)
is_superset = s1.issuperset({1, 2})  # True (親が子を内包しているか)

# 7. 複製 (Copy)
s_copy = s1.copy()  # 浅いコピー（シャローコピー）の作成

```

## データの変形とパイプライン

コレクション型を活用すると、データの処理をパイプラインのように記述できます。

例えば、「売上データのリストから、特定の条件を満たすものだけを抽出し、加工して合計を出す」といった処理です。

```python
# 例：売上データのリストから500円以上のものだけを抽出して合計する
sales = [100, 600, 300, 800, 400]

# データの絞り込みと変形
high_sales = [price for price in sales if price >= 500]
total = sum(high_sales)

print(f"合計: {total}")
```

このコードでは、`sales`という入力データが`high_sales`という中間データへと変形され、最終的な`total`へと集約されています。手続きを羅列するのではなく、データがどのように変形されていくかという流れ（データフロー）を意識して書くことが、保守性の高いコードへの第一歩です。

リスト内包表現というものを途中で使っています。

`[(この配列の要素にするモノ) for (inの後のイテレータを受け取る変数定義) in (イテレータ) if (この配列の要素にする条件)]`

`if`の後ろは省略することができます。非常に便利です。イテレータに生えるメソッドのネイティブ構文版として理解することができます。
